# 이커머스 비즈니스 전문가 메모리

## 초기 상태 (2026-02-20)
- 수수료 모델 PRD에 정의됨 (10%/12%/15%)
- 적립금 전환율 정의됨 (100%/80%/60%)
- KPI 보정 완료 (월 수수료 25만원+ / 100만원+)
- 실제 검증 미실시

## 2026-03-10 사업 모델 재정의

- 파트너클래스의 핵심 사업 목적은 플랫폼 수수료 극대화가 아니라 커뮤니티 활성화와 재료/키트 판매 활성화다.
- 파트너클래스는 자사몰과 브랜드 종속력을 높이는 전략 서비스로 본다.
- 파트너 대시보드 핵심 KPI 기본 후보는 `오늘 예정 수업`, `준비할 키트/재료`, `이번 달 예상 정산액` 이다.
- 협회 레이어 KPI 기본 후보는 `협회별 유입`, `협회원 전용 상품 클릭/구매`, `협회 일정 -> 자사몰 전환` 이다.
- 방향 판단은 `docs/파트너클래스/README.md`와 `shared-service-identity.md`를 우선한다.

## 2026-03-10 S1-1 수익 엔진 메모

- 키트 Step 1은 선제작 번들보다 `클래스별 자사몰 상품 링크 연결`을 먼저 고정하는 방향으로 완료됐다.
- 수강생 상세 페이지에서 재료를 즉시 확인하고 장바구니에 담을 수 있어야 재료 매출 전환 데이터가 쌓인다.
- 파트너 입력 부담을 줄이기 위해 상품 검색 연동보다 `링크 또는 branduid 입력`을 먼저 표준 UX로 채택했다.
- 라이브 DB 제약은 프론트/워크플로우에서 이미 우회 중이므로, 이후 KPI 측정 로직은 canonical 읽기 응답 기준으로 설계해야 한다.

## 2026-03-10 S1-4 리텐션/재구매 메모

- 예약 종료 뒤 리텐션은 `후기 적립금`만으로 끝내지 않고 `같은 강사 다음 수업`과 `키트 재구매`까지 한 흐름으로 연결해야 한다.
- 마이페이지 completed 섹션은 재방문을 유도하는 CRM 표면이고, 이메일은 그 표면으로 다시 복귀시키는 트리거로 본다.
- 재료 재구매는 별도 추천 알고리즘보다 실제 수강 클래스의 `kit_items`를 다시 보여주는 것이 전환 의도가 가장 높다.

## 2026-03-10 S1-5 정산 운영 메모

- 현재 canonical 수수료 체계는 `BLOOM 25 / GARDEN 20 / ATELIER 15 / AMBASSADOR 10` 이고, 레거시 `SILVER/GOLD/PLATINUM` 은 alias로만 유지한다.
- 정산 관리자 화면은 `정산 대기 처리`와 `정산서 발송 이력`을 같이 보여줘야 운영자가 같은 화면에서 정산 흐름을 닫을 수 있다.
- 운영 SMTP credential이 현재 실패 상태이므로, 정산 배치는 발송 실패를 성공으로 포장하지 않고 실패 건수와 에러 메시지를 그대로 노출한다.
- 실제 수익 기준 문서와 운영 가이드는 `docs/파트너클래스/commission-policy.md`, `docs/파트너클래스/settlement-automation-guide.md` 이다.

## 2026-03-11 S1-6 전환 보조 메모

- 상세 FAQ 확장은 고객센터 비용 절감보다 `예약 직전 이탈 감소`에 더 큰 목적이 있다.
- FAQ 카테고리 중 `키트·배송`, `정산`은 수강생의 결제 불안을 줄이고 재료 구매 전환까지 이어지는 핵심 레이어다.
- 검색 결과가 없을 때 문의 채널을 바로 붙여두는 구조가 실제 예약 보류를 줄이는 데 유리하다.
- 실제 운영 기준 문서는 `docs/파트너클래스/faq-expansion-guide.md` 이다.

## 2026-03-11 S1-7 파트너 활성화 메모

- 온보딩 체크리스트의 목적은 단순 안내가 아니라 `첫 예약까지 걸리는 시간을 줄이는 것` 이다.
- 파트너 입장에서 가장 체감이 큰 구간은 `일정 추가`와 `키트 설정` 이므로, CTA가 해당 화면까지 바로 이어져야 한다.
- 프로필, 교육, 강의, 일정, 키트가 모두 끝나면 `운영 준비 완료` 상태로 보고 체크리스트를 종료한다.
- 완료 모달의 핵심 메시지는 `이제 첫 예약만 기다리면 된다` 여야 한다.
- 실제 운영 기준 문서는 `docs/파트너클래스/onboarding-checklist-guide.md` 이다.

## 2026-03-11 S1-8 운영 우선순위 메모

- 파트너 대시보드의 다음 핵심 KPI 후보 중 실제 행동 전환에 가장 가까운 것은 `오늘 수업 / 준비할 키트 / 미답변 후기` 다.
- 이 3개는 단순 요약이 아니라 곧바로 일정 관리, 예약 관리, 후기 관리로 보내는 운영 액션 레이어로 봐야 한다.
- 키트 준비 카드는 향후 키트 매출/배송 준비율과 연결되는 운영 지표의 시작점이 된다.
- 실제 운영 기준 문서는 `docs/파트너클래스/dashboard-action-board-guide.md` 이다.

## 2026-03-11 S1-9 수익 흐름 검증 메모

- 로컬 통합 테스트 기준으로 `키트 연결 -> 상세 장바구니 -> 마이페이지 재구매 -> 파트너 운영 액션 -> 관리자 정산 패널` 흐름이 한 번에 이어진다.
- 정산 패널에서 주문 `136,000원`, 수수료 `34,000원`, 적립금 `102,000원` 집계 표시가 확인됐다.
- 실제 발송만 SMTP credential 문제로 막혀 있고, 수익 계산과 실패 노출 방식 자체는 현재 기준으로 맞다.
- 실제 운영 기준 문서는 `docs/파트너클래스/phase3-1-integration-test.md` 이다.

## 2026-03-11 S2-1 파트너 전환 메모

- 파트너 신청 랜딩의 핵심 전환 포인트는 `수수료 조건` 이 아니라 `운영 피로 감소 + 홍보 확장 + 자사몰 연결` 이다.
- 공개 메시지는 `강의만 잘하시면 됩니다. 나머지는 PRESSCO21이 합니다.` 로 고정하고, 재료·키트·정산·협회 확장을 사업 가치로 풀어낸다.
- 성장 경로 `BLOOM -> GARDEN -> ATELIER -> AMBASSADOR` 는 금전 조건보다 브랜딩과 운영 확장 혜택을 중심으로 보여준다.
- 실제 운영 기준 문서는 `docs/파트너클래스/partner-apply-sales-landing-guide.md` 이다.

## 2026-03-11 S2-2 협회 영업 메모

- 협회 B2B 도구의 핵심은 `협회원 혜택 + 일정 홍보 + 자사몰 락인 + 인센티브` 를 한 화면에서 수치화하는 것이다.
- ROI 계산은 협회원 수, 월 예상 수강 인원, 평균 재료 구매액 3변수로 시작하는 것이 가장 설명하기 쉽다.
- 관리자 협회 탭에서 제안서 URL을 바로 생성할 수 있어야 대표가 협회 미팅 직전에 숫자만 바꿔 대응할 수 있다.
- 실제 운영 기준 문서는 `docs/파트너클래스/affiliation-b2b-proposal-tool-guide.md` 이다.

## 2026-03-11 S2-3 탐색 허브 수익 메모

- 전국 탐색 허브의 직접 목적은 예약 전환 자체보다 `오프라인 클래스 탐색 -> 수강 -> 재료 재구매 -> 협회원 혜택 유입` 흐름을 더 자주 만들게 하는 것이다.
- `협회·세미나`, `혜택·이벤트` 탭은 수강생 입장에서는 탐색 편의 기능이지만 사업적으로는 협회 락인과 자사몰 전환 장치다.
- 상세의 `content_type` 분기는 단순 디자인 변화가 아니라, 협회원 혜택/세미나 공지/일반 예약이라는 서로 다른 수익 흐름을 정확히 안내하기 위한 장치다.
- 실제 운영 기준 문서는 `docs/파트너클래스/nationwide-discovery-ia-guide.md` 이다.

## 2026-03-11 S2-6 리텐션 비즈니스 메모

- 수강 완료 뒤의 첫 리텐션 장치는 `할인` 보다 `다음 행동 제안` 이 더 중요하다.
- 월간 리포트와 streak badge 는 재방문 이유를 만들고, 후기 감사 메시지는 재료 재구매와 다음 클래스 탐색으로 다시 연결하는 표면 역할을 한다.
- 30일 휴면 자동화는 단기 매출보다 `커뮤니티 재활성화 -> 자사몰 재접속` 을 만드는 장치로 본다.
- 현재 레거시 완료 예약의 이메일 누락 때문에 발송 대상이 줄어드는 상태이므로, 리텐션 KPI 해석 시 `raw_count` 와 `actual_send_count` 를 구분해야 한다.
- 실제 운영 기준 문서는 `docs/파트너클래스/community-retention-guide.md` 이다.

## 2026-03-11 S3-3 구독 파일럿 수익 메모

- 구독 파일럿의 목적은 `재료 재구매`를 `월간 반복 매출`로 바꾸는 첫 운영 실험이다.
- 가격 정책 1차 기준은 `kit_items 합산가 대비 10% 구독가 할인` 으로 고정했다.
- 현재 자동 생성되는 `SUBORD_*` 는 매출 인식용 주문번호가 아니라 운영 fulfillment 용 내부 ref 로 본다.
- 즉, 이번 단계의 KPI 는 `구독 등록 수`, `월간 생성 수`, `해지율`, `구독가 절감 체감` 이고, 실결제 자동화는 다음 단계다.
- 실제 운영 기준 문서는 `docs/파트너클래스/subscription-pilot-guide.md` 이다.
