## 핵심 기술 제약 요약
- JS 내 `${variable}` → 반드시 `\${variable}` 이스케이프
- 가상 태그(`{$...}`, `<!-- -->`, `<makeshop:>`) 절대 수정/삭제 금지
- CSS는 반드시 고유 컨테이너로 스코핑 (전역 셀렉터 금지)
- JS는 IIFE 패턴 필수
- UTF-8 이모지 사용 금지
- 외부 라이브러리는 CDN `<script>` 태그로만

## 프로젝트 구조
- 메인페이지: `메인페이지/Index.html`, `css.css`, `js.js`
- 파일 네이밍: `Index.html`(HTML탭), `css.css`(CSS탭), `js.js`(JS탭)
- `*.fixed.*` = 개선/수정 버전

## 2026-03-10 파트너클래스 UX 기준

- 파트너클래스 UI의 1순위 설계 대상은 수강생이다.
- 목록과 상세는 예약 전환보다 먼저 신뢰 형성과 불안 해소를 책임져야 한다.
- 파트너신청 페이지는 신청 폼이 아니라 `강의만 잘하면 되게 해주는 서비스`를 파는 제휴 입점 페이지여야 한다.
- 협회 레이어는 같은 허브 안에서 탐색만 분리하고, `협회/세미나`와 `협회원 혜택`이 클래스 레이어와 자연스럽게 이어져야 한다.
- 용어는 목록, 상세, API, n8n, NocoDB에서 완전히 통일해야 한다.
- 현재 문서 진입점은 `docs/파트너클래스/README.md` 이다.

## 2026-03-10 S1-1 UI 메모

- 강의 등록 키트 UI는 `상품명 / 자사몰 상품 링크 / 예상 판매가 / 1인 기준 수량` 4필드로 통일한다.
- 링크 입력 UX는 검색창보다 먼저 `상품 링크 또는 branduid 입력` 흐름으로 단순화한다.
- 상세 페이지 키트 섹션 제목은 `이 수업에 사용되는 재료`를 사용한다.
- 상세 키트 카드는 `상품 보기`와 `장바구니 담기`, 상단 `재료 한번에 담기` CTA를 함께 제공한다.
- 이후 S1-2는 상단 신뢰 요약과 CTA 계층을 이 키트 영역과 끊기지 않게 연결해야 한다.
