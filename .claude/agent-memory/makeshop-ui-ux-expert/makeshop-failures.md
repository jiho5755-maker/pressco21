# 메이크샵 적용 실패 이력

이 파일은 메이크샵 편집기에서 코드 적용 시 발생한 실패 사례를 추적합니다.
에이전트는 매 작업 시작 시 이 파일을 참조하여 동일한 실패를 사전 차단합니다.

---

## 기록 형식

```
## FAIL-{번호}: {실패 제목} ({날짜})

- **증상**: {사용자가 보고한 증상}
- **발생 위치**: {HTML/CSS/JS 탭, 해당 섹션}
- **원인**: {분석된 원인}
- **해결**: {적용한 해결 방법}
- **재발 방지 규칙**: {앞으로 지켜야 할 규칙}
```

---

## 알려진 공통 위험 패턴 (사전 등록)

### RISK-001: JS 템플릿 리터럴 이스케이프 누락
- **증상**: "데이터 수정 실패" 에러
- **원인**: JS 코드 내 `${variable}`을 이스케이프하지 않아 메이크샵 엔진이 치환코드로 오인
- **예방**: 모든 JS 템플릿 리터럴에서 `\${variable}` 형태로 백슬래시 이스케이프 필수

### RISK-002: 전역 CSS 셀렉터 사용
- **증상**: 메이크샵 관리자 패널, 결제 창 등 기본 UI 스타일 깨짐
- **원인**: `body`, `*`, `a`, `div` 등 전역 셀렉터로 CSS 작성
- **예방**: 모든 CSS를 고유 컨테이너 ID/클래스 내로 스코핑

### RISK-003: 가상 태그 손상
- **증상**: 상품명, 가격, 옵션 등 동적 데이터 표시 불가
- **원인**: `{$...}`, `<!-- -->` 등 메이크샵 가상 태그를 수정/삭제
- **예방**: 가상 태그는 절대 수정/삭제 금지, 주변 HTML 구조 변경 시 렌더링 컨텍스트 확인

### RISK-004: UTF-8 이모지 인코딩 문제
- **증상**: 편집기 저장 실패 또는 문자 깨짐
- **원인**: HTML/CSS/JS에 UTF-8 이모지 직접 사용
- **예방**: SVG 아이콘 또는 CSS 디자인 요소로 대체
