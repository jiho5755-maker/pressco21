## 서류 작성 AI 프롬프트 템플릿

### 사업계획서 초안 생성 프롬프트
```
다음 정부지원사업에 신청할 사업계획서 초안을 작성해주세요.

[공고 정보]
- 사업명: {{공고명}}
- 주관기관: {{주관기관}}
- 지원 내용: {{지원내용}}
- 지원 대상: {{지원대상}}
- 제출 마감: {{신청마감일}}

[신청 기업 정보]
- 상호: Pressco21 (foreverlove.co.kr)
- 업종: 꽃, 생활소품 온라인 도소매
- 규모: 소기업, 직원 8인
- 소재지: 서울 송파구
- 주요상품: 압화, 드라이플라워, 프리저브드플라워, 레진공예, 하바리움

[작성 요청 항목]
1. 신청 배경 및 필요성 (200자)
2. 사업 목표 및 기대효과 (200자)
3. 추진 계획 (3개 항목, 각 100자)
4. 자금 사용 계획 (지원금 사용처 중심)

이 내용은 수정 가능한 초안으로, 담당자가 최종 검토 후 수정해서 사용할 것입니다.
각 항목을 명확히 구분하여 작성해주세요.
```

## 행정서류 자동화 워크플로우 설계

### 워크플로우 1: 신청 서류 체크리스트 자동 생성
```
[트리거] Airtable에서 처리상태='신청진행중' 으로 변경된 공고 감지
↓
[HTTP Request] 공고 원문 URL 크롤링 (또는 기존 저장된 공고 내용 조회)
↓
[Gemini AI] 필요 서류 목록 추출 (위 프롬프트 사용)
↓
[Code 노드] Pressco21 보유 서류 목록과 대조 → 미보유/확인필요 서류 필터링
↓
[Telegram] 체크리스트 알림 발송
  형식: "📋 {공고명} 신청 서류 체크리스트\n✅ 보유: ...\n⚠️ 준비필요: ..."
↓
[Airtable] 체크리스트 내용을 담당자메모에 저장
```

### 워크플로우 2: 사업계획서 초안 자동 생성
```
[트리거] 텔레그램 명령어 "/초안 {공고ID}" 또는 Airtable 상태 변경
↓
[Airtable] 해당 공고 ID로 공고 정보 조회
↓
[Gemini AI] 사업계획서 초안 생성 (위 프롬프트 사용)
↓
[Google Docs API 또는 텍스트 파일] 초안 문서 생성
↓
[Telegram] 초안 생성 완료 알림 + 내용 요약 전송
↓
[Airtable] 초안생성일시 필드 업데이트
```

## Airtable 스키마 확장 (서류 관리용)

기존 `정부지원공고` 테이블에 추가할 필드:

| 필드명 | 타입 | 설명 |
|--------|------|------|
| `신청의향` | Checkbox | 담당자가 신청하기로 결정 시 체크 |
| `서류체크리스트` | Long text | AI가 생성한 필요 서류 목록 |
| `사업계획서초안` | Long text | AI가 생성한 사업계획서 초안 |
| `초안생성일시` | Date | 초안 자동 생성 일시 |
| `신청일` | Date | 실제 신청 완료일 |
| `결과` | Single select | 선정/미선정/대기/취소 |
| `수령금액` | Currency | 실제 수령한 지원금액 |

## 서류 작성 보조 품질 기준

1. **초안은 항상 초안임을 명시**: 담당자가 반드시 검토/수정
2. **허위 정보 절대 금지**: 회사 실정에 맞는 내용만 생성
3. **수치 데이터는 빈칸 처리**: 정확한 매출액, 직원수 등은 담당자가 직접 입력
4. **형식 준수**: 각 공고의 서류 형식/양식이 있으면 반드시 해당 양식 사용 안내
5. **마감일 강조**: 서류 준비 완료까지 충분한 시간이 있는지 항상 확인
