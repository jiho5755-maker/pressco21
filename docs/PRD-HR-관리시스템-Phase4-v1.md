## 0. 한줄 요약

텔레그램 미니앱(직원) + Flora Next.js PC 어드민(관리자)으로 **8명 3유형 근무체계에 최적화된 HR 관리 시스템**을 구축하여, 정부지원금 증빙 자동화와 관리자 업무 제로화를 달성한다.

## 2. 플랫폼 결정

### 2-1. 팀미팅 2라운드 결론

**텔레그램 미니앱 유지 + 탈출 설계 (A+C 하이브리드)**

| 옵션 | 개발공수 | 채택률 | 유지보수 | 판정 |
|------|---------|--------|---------|------|
| A) 텔레그램 유지 | 0h (기존) | 100% | 낮음 | 채택 |
| B) 자체 앱 | 80~120h | 리스크 | 높음 | 기각 |
| C) 하이브리드 | 100~150h | 리스크 | 최고 | 기각 |

### 2-2. 결정 근거 (전원 합의)

1. **채택률 100% 최우선**: 서향자 실장(60대) 1명이라도 이탈 시 이중 운영
2. **미니앱 이미 9,437줄 완성**: HR탭/업무보드/캘린더/출고/파일첨부
3. **봇 메시지 = 푸시 알림**: 별도 FCM 불필요
4. **8명에 앱스토어 배포는 과잉**: 독립앱은 30명+ 시점에 재검토
5. **고객앱(쇼핑)과 내부앱(HR) 반드시 분리**: React Native 앱은 고객용으로만

### 2-3. 탈출 설계 원칙

- WebApp API 호출부(initData 인증, 봇 메시지)를 별도 모듈로 격리
- UI/비즈니스 로직은 순수 React — 떼면 바로 PWA 전환 가능
- 트리거: 직원 20명 초과 또는 텔레그램 정책 변경 시

## 3. 직원 3유형과 시스템 대응

| 유형 | 대상 | 근무제 | 시스템 동작 |
|------|------|--------|-----------|
| Track A: 선택근로 | 장준혁, 이재혁, 서향자, 이혜자 | 선택근무제 (근기법 52조) | **실제 출퇴근 시각 기록 필수**, 정산기간 내 주40h 검증 |
| Track B: 재량근로 | 장다경, 조승해, (송미) | 재량근무제 (근기법 58조 3항) | 간주시간 기록, "업무보고 접수" 라벨, **시간 계산 금지** |

- Track B Core Time(11~16시) → "협의 가능 시간대"로 명칭 변경, 미준수 제재 기능 **절대 금지**
- GPS/위치추적/화면캡처 → **절대 도입 불가** (개인정보보호법 + 재량근로 취지 위배)
- 초과근로 → 차단 말고 **경고만**, 기록은 반드시 남김
- 동의서에 법령 보관 의무(3년) 고지 문구 추가

## 5. 기술 아키텍처

### 5-1. 시스템 구성

```
[직원 텔레그램] → [미니앱 React+Vite] → [Flora API (Next.js)]
                                              ↕
[관리자 PC] → [/admin/hr (Next.js SSR)] → [PostgreSQL]
                                              ↕
[n8n WF] → [텔레그램 봇 알림] ←→ [MinIO 파일]
```

### 5-2. 신규 API (7개)

| API | 메서드 | 용도 |
|-----|--------|------|
| `/api/hr/leave` | POST | 휴가 신청 |
| `/api/hr/leave` | GET | 휴가 목록 (본인/전체) |
| `/api/hr/leave/[id]/approve` | POST | 휴가 승인/반려 |
| `/api/hr/dashboard` | GET | 전직원 현황 집계 |
| `/api/hr/report/monthly-all` | GET | 전직원 월간 리포트 (엑셀) |
| `/api/hr/employee-registry/[id]` | PATCH | 근로자명부 수정 |
| `/api/hr/work-logs/daily` | GET | 일자별 전직원 업무일지 |

### 5-3. DB 변경 (최소)

- `hr_leave_records`: `attachment_url` 컬럼 추가 (MinIO 경로)
- `hr_employee_registry`: `annual_leave_base` numeric 컬럼 추가 (연차 기본 일수)
- 신규 테이블: 없음

### 5-4. 기존 API 유지 (13개)

Phase 1~2.5에서 구축한 전체 API 그대로 활용. 변경 없음.
