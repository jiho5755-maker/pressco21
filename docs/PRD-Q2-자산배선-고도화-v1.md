# PRD: Q2 자산 배선 고도화

> **프로젝트명**: PRESSCO21 기존 자산 유기적 연결 및 운영 고도화
> **작성일**: 2026-04-02
> **버전**: 1.0
> **근거**: C-Suite 팀미팅 종합 분석 (CSO/CTO/COO/CMO/CFO/PM)
> **대표 확정 사항**: 강사 인센티브 불필요, 쿠팡 776 전체+확장, 원가 미정리로 CRITICAL 마진 무시, 리뷰 적립금 스마트스토어 동일

---

## 1. 프로젝트 개요

### 1.1 배경

PRESSCO21은 2026년 Q1까지 n8n WF 26개, 미니PC Company Vault, 파트너클래스 8페이지, 비즈니스 인텔리전스, 스킨 고도화 등 기술 자산을 구축했다. 그러나 **자산 간 연결이 부재**하여 자동화 효과가 개별 단위에 머물러 있다.

### 1.2 핵심 진단

> **"구축 완성률 80%, 연결 완성률 40%"** — CTO 진단

| 현상 | 원인 |
|------|------|
| RFM이 매주 분류되지만 아무 액션 없음 | 세그먼트→알림톡 연결 WF 부재 |
| 리뷰 자동화 WF 3개가 완성 후 비활성 | 카카오 검수 대기 + 활성화 미실행 |
| CHURN-DETECT가 이탈 감지 후 방치 | 후속 리텐션 액션 노드 부재 |
| 이미지 자동처리 후 수동 등록 | Vault→MakeShop 연결 없음 |
| 플로라 AI가 회사 데이터를 모름 | NocoDB 데이터 브릿지 부재 |

### 1.3 목표

**Q2 (4~6월) 동안 신규 개발 최소화, 기존 자산 배선에 집중하여 자동화 효과 3~4배 증폭**

### 1.4 성공 지표 (KPI)

| 지표 | 현재 | Q2 목표 | 측정 방법 |
|------|------|--------|----------|
| 리뷰 수집률 | 미측정 | 배송완료 대비 15% | NocoDB review_notification |
| D+30 재구매율 | 미측정 | 35% | RFM 데이터 추적 |
| 이재혁 과장 CS 전화 시간 | ~90분/일 | ~20분/일 | 직접 체감 피드백 |
| 파트너클래스 활성 강사 수 | 미측정 | 30명+ | NocoDB 강사공간 |
| 쿠팡 입점 SKU | 0 | 776+ | 쿠팡 셀러센터 |
| RFM 자동 캠페인 발송 수 | 0 | 주 200건+ | 알리고 발송 로그 |

## 3. Phase 1 — "켜기" (4/2~6)

> 이미 만들어진 것을 활성화하고, 블로커를 제거한다.

### P1-1. SMTP blocker 해결

- **문제**: Naver SMTP `535 Username and Password not accepted` — 파트너클래스 정산 알림/이메일 전체 차단
- **영향 범위**: FA-003(반려이메일), 정산 이메일, 파트너 안내 이메일
- **해결**: Naver 앱 비밀번호 재발급 또는 대체 SMTP(SendGrid/Mailgun) 전환
- **담당**: CTO (Claude 직접)
- **소요**: 반나절
- **선행 조건**: 없음

### P1-2. 리뷰 자동화 WF 활성화 준비

- **현재**: WF-A(`o59TWjdTiIPZtaxY`), WF-B(`Qa2rjrKoliSC4owj`), WF-C(`yZ0emAwAdFSvl9jJ`) 서버 import 완료, 비활성
- **준비**: 카카오 알림톡 템플릿 3종(UG_7020, UG_7026, UG_7027) 검수 대기
- **적립금 단가**: 스마트스토어 리뷰 적립금과 동일 금액 (확인 후 WF-C에 반영)
- **활성화 조건**: 카카오 검수 통과 (예상 ~4/7)
- **담당**: CTO + 대표 확인

### P1-3. Tailscale VPN Phase 1 설치

- **목적**: 미니PC 원격 접근 → 인프라 기반 완성
- **작업**:
  1. 맥북: `brew install tailscale` → `tailscale up`
  2. 미니PC: `curl -fsSL https://tailscale.com/install.sh | sh` → `sudo tailscale up`
  3. SSH config에 Tailscale IP(100.x.x.x) 추가
  4. F번 백업검증 스크립트 배포
- **담당**: CTO (Claude 직접)
- **소요**: 1시간

### P1-4. 직원 장치 ID 수집 요청

- **목적**: Company Vault Syncthing 디바이스 추가 + 플로라 Chat ID
- **작업**: 직원 카톡으로 장치 ID 스크린샷 요청
- **담당**: 대표
- **소요**: 10분

## 4. Phase 2 — "연결" (4/7~18)

> 분리된 자산 사이에 배선(Wiring)을 놓는다.

### P2-1. 리뷰 자동화 WF 3개 Publish + 테스트

- **선행**: P1-2 (카카오 검수 통과)
- **작업**:
  1. WF-A → WF-B → WF-C 순서로 Publish
  2. 실제 최근 주문으로 테스트 발송 1건
  3. 알림톡 수신 + 적립금 지급 확인
  4. 이상 없으면 운영 전환
- **담당**: CTO
- **소요**: 2시간

### P2-2. RFM 세그먼트 → 자동 캠페인 연결 (신규 WF)

- **선행**: S3A-002 (RFM 세그먼트 매주 월요일 10시 실행)
- **신규 WF**: `RFM-ACTION` (매주 화요일 09:00 자동 실행)
- **흐름**:

```
Schedule (화 09:00)
→ NocoDB customer_segments 조회
→ Switch (세그먼트별 분기)
  ├─ At Risk (3개월 미구매): 윈백 알림톡 "오랜만이에요 + 재방문 혜택"
  ├─ New Customer (첫 구매 D+10): 관련상품 추천 알림톡
  ├─ Champions (VIP): 신제품 사전안내 (월 1회만)
  └─ Hibernating (6개월+): 윈백 + 특별 혜택
→ 알리고 알림톡 발송
→ NocoDB 발송 기록 저장
```

- **필요**: n8n WF JSON 1개 + 알리고 알림톡 템플릿 3~4개 (카카오 검수 필요)
- **주의**: 카카오 알림톡 검수 3~5일 소요. 검수 대기 중 SMS 백업 발송 가능
- **담당**: CTO (n8n-workflow-builder)
- **소요**: 1~2일

### P2-3. CHURN-DETECT → 강사 리텐션 자동 액션

- **선행**: 없음 (기존 CHURN-DETECT WF 수정)
- **수정 내용**: CHURN-DETECT 출력에 후속 노드 추가

```
CHURN-DETECT 결과
→ Switch (이탈위험 / 정상)
  ├─ 이탈위험:
  │   ├─ 텔레그램 어드민 경보 (대표+원장님)
  │   └─ 강사 개인 알림톡 "안녕하세요, 혹시 도움이 필요하신가요?"
  └─ 정상: 패스
```

- **필요**: 기존 WF JSON 수정 + 알림톡 템플릿 1개
- **담당**: CTO
- **소요**: 반나절

### P2-4. 쿠팡 로켓배송 견적서 제출

- **현재**: 776 SKU 견적서 작성 완료, 이미지 매칭 609개 100%
- **작업**:
  1. 견적서 최종 검수 (대표)
  2. 라벨 776장 생성
  3. 쿠팡 셀러센터 제출
- **방향**: 776 전체 입점. 이후 MakeShop 판매 상품 정리하여 추가 확장
- **담당**: 대표 + CTO(라벨 자동화 지원)
- **소요**: 2~3일
- **후속**: 입점 후 쿠팡 주문 데이터 → weekly_snapshot 통합 (Phase 3)

### P2-5. 플로라 AI → NocoDB 데이터 브릿지

- **목적**: 플로라가 "이번 주 매출?", "이탈 위험 강사?" 질문에 답할 수 있게
- **구현 경로**: n8n WF에서 텔레그램 커맨드 감지 → NocoDB 조회 → 플로라에 컨텍스트 주입
- **연결 데이터**:
  - 주간 매출/주문: weekly_snapshot
  - 이탈 위험 강사: customer_segments
  - 파트너 현황: 강사공간 tbl_Partners
- **담당**: CTO
- **소요**: 2~3일

## 5. Phase 3 — "확장" (4/21~5/9)

> 연결된 파이프라인 위에서 캠페인 실행과 운영 자동화 확장.

### P3-1. 이재혁 과장 CS 자동화 3단계

| 단계 | 작업 | 효과 |
|------|------|------|
| a | 배송시작/완료 알림톡 활성화 (검수 통과 후) | 고객 선제 안내 → 문의 전화 감소 |
| b | AI 챗봇(F050) FAQ 상위 20건 강화 | 상품사용법/택배/재고 자동응답 |
| c | 반품/교환 접수 자동화 (채널톡 봇 + NocoDB) | 예외만 사람이 처리 |

- **목표**: 하루 90분 → 20분으로 단축
- **담당**: CTO (a, b), COO 프로세스 설계 (c)
- **소요**: 단계별 각 2~3일

### P3-2. 재고 임계값 알림 + 품절 자동처리 (신규 WF)

- **흐름**:

```
Schedule (매일 08:00)
→ MakeShop API: 전 상품 재고 조회
→ NocoDB: 재고 현황 업데이트
→ 임계값 이하(기본 5개) 추출
→ 텔레그램 알림 (이재혁 과장 + 대표)

[별도 트리거] 재고 0 감지
→ MakeShop API: 품절 자동처리
→ 재고 입고 시 (수동 트리거): 품절해제
```

- **API 여유**: 현재 ~22회/시간, 한도 500회. 재고 조회 추가해도 안전
- **담당**: CTO (n8n-workflow-builder)
- **소요**: 2일

### P3-3. 입문자 이메일 시퀀스 자동화 (신규 WF)

- **대상**: 신규 가입 회원
- **시퀀스** (n8n Schedule + 조건 분기):

| 타이밍 | 제목 | 내용 |
|--------|------|------|
| D+1 | 꽃 공예, 어디서 시작하지? | 입문 가이드 링크 |
| D+3 | 초보자 TOP 3 상품 | 인기 상품 추천 |
| D+7 | 유튜브 무료 강의 | 유튜브 채널 링크 |
| D+14 | 원데이 클래스 체험 | 파트너클래스 추천 |
| D+21 | 다음 도전 재료 추천 | 두 번째 구매 유도 |

- **발송**: SMTP (비용 0원)
- **담당**: CTO + CMO (카피 가이드)
- **소요**: 2일

### P3-4. Company Vault 직원 온보딩

- **선행**: P1-4 (장치 ID 수집)
- **작업**: Syncthing 디바이스 추가 + 폴더 공유 매핑
- **담당**: CTO
- **소요**: 직원당 30분

### P3-5. 강사 온보딩 캠페인 (마케팅)

- **대상**: 자사몰 강사회원 1,500명 중 파트너클래스 미등록자
- **인센티브**: 없음 (기존 강사 할인가가 충분한 유인)
- **메시지**: 이메일 + 알림톡 "선생님, 수강생 만날 준비 되셨나요?"
- **랜딩**: 파트너 신청 페이지 (2609, 이미 구축)
- **담당**: CMO (카피) + CTO (발송 WF)
- **소요**: 1일

## 6. Phase 4 — "측정+반복" (5/10~6/30)

> 실행 결과를 측정하고, 시즌 기획전과 파트너클래스 스케일을 진행한다.

### P4-1. 효과 측정 대시보드

- 리뷰 수집률, 알림톡 클릭률, 재구매율 → NocoDB 주간 스냅샷에 추가
- S3A-001 주간 리포트에 캠페인 성과 섹션 추가
- **담당**: CTO
- **소요**: 1일

### P4-2. 어버이날 기획전 (5/1~12)

- **주제**: "감사의 꽃 — 5월 어버이날 선물 만들기"
- **구성**: 카네이션 압화 키트 3종 + 유튜브 How-to + 파트너클래스 연결
- **채널**: 자사몰 기획전 + 알림톡 발송 (RFM-ACTION WF 활용)
- **담당**: CMO (기획) + CTO (기획전 페이지)

### P4-3. 파트너클래스 E3 스케일

- **선행**: P1-1 (SMTP 해결), P2-3 (CHURN 리텐션)
- **작업**:
  - 실제 파트너 1호 섭외 + 온보딩
  - 콘텐츠허브(8014) 활성화
  - 수강완료 → 자사몰 재구매 유도 WF
- **담당**: CSO (전략) + CTO (구현)

### P4-4. 쿠팡 주문 데이터 통합

- **선행**: P2-4 (쿠팡 입점 완료)
- **작업**: 쿠팡 주문/정산 데이터 → weekly_snapshot 통합
- **목적**: 채널별 ROI 비교, 전체 매출 통합 BI
- **담당**: CTO

### P4-5. UGC 작품 갤러리 + SNS 캠페인

- **선행**: P2-1 (리뷰 WF 가동 후 데이터 축적)
- **작업**: 포토리뷰 → 자동 태깅 → "이주의 작품" 인스타 피처링
- **담당**: CMO
