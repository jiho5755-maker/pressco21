> **프로젝트명**: PRESSCO21 AI 에이전트 조직 구축 + 업무 자동화 + L5 자율운영
> **작성일**: 2026-04-04
> **버전**: 2.0 (v1 전면 재작성)
> **근거**: 3관점 진단 + 커뮤니티 벤치마크 + 에이전트 품질 감사 + 대표 인터뷰
> **최종 목표**: AI 에이전트가 부족한 전문인력을 채우고, 자율적으로 협업하며, 대표는 판단과 승인에 집중

## 1. 왜 하는가

### 1.1 회사 현실

PRESSCO21은 8명이 연매출 11.6억을 만드는 꽃 공예 전문 회사다.

| 부서 | 인원 | 하는 일 |
|------|------|---------|
| 대표이사 | 이진선 원장님 | 교육, 대외협력, 해외 강의 |
| 생산관리 | 서향자 실장, 이혜자 대리 | 재료 가공·포장 (연세 있어 AI 활용 불가) |
| 물류 | 장준혁 사장, 이재혁 과장 | 주문·출고·배송 + **생산관리 대행 + CS 전화** |
| 디자인기획 | 장다경 팀장, 조승해 사원 | 웹디자인, 유튜브/SNS, 상세페이지 제작 |
| 경영기획 | 장지호 팀장 | **사업전략+IT+마케팅+CS+고객관리+정산+재무+... 전부** |

### 1.2 핵심 문제

**장지호 팀장 1명이 존재하지 않는 6개 부서를 혼자 감당한다:**

| 없는 부서 | 장지호가 대행 | 시간 잡아먹는 정도 |
|----------|-------------|-----------------|
| 마케팅팀 | 광고 기획, 카피, 콘텐츠 전략 | 매우 높음 (가장 약한 분야) |
| CS팀 | 1:1 게시판, 리뷰 답변, 전화 응대 | 높음 (수시 인터럽트) |
| 디자인 기획 | 기획안 작성 → 다경/승해에게 지시 | 높음 (기획 없으면 디자인팀 멈춤) |
| 재무팀 | 매출 확인, 채널별 정산, 세무사 연계 | 중간 |
| IT팀 | 메이크샵 개발, n8n 자동화, 서버 관리 | 높음 (Claude Code로 해결 중) |
| 영업팀 | 제휴, B2B, 쿠팡 입점 | 비정기 |

**이재혁 과장도 과부하**: 물류 + 생산관리 대행 + CS 전화 = 3중 부담

**디자인팀 병목**: "기획안이 안 와서 못 시작함" — AI 기획안 자동생성이 최우선 (회사 회의 확정)

### 1.3 업무 인입 현실

비정기 업무가 **모든 채널**에서 예고 없이 들어온다:
- 전화 (고객, 거래처)
- 메일 (거래처, 정부사업, 협회)
- 카카오톡 (원장님, 직원)
- 직접 발견 (데이터 보다가 이슈 감지)

→ **단일 허브**로 모아서 우선순위 자동 정렬이 필요

### 1.4 목표

```
[Before] 장지호가 8개 역할을 머릿속으로 관리, 직원에게 카톡으로 지시
[After]  AI 에이전트 28명이 6개 빈 부서를 채우고, Flora가 업무 허브 역할
         장지호는 판단과 승인에 집중, 직원은 자기 전문 분야에만 집중
```

## 2. AI 에이전트 조직도

## 3. 업무 관리 체계

### 3.1 문제: 업무가 모든 채널에서 예고 없이 들어온다

```
전화 ──┐
메일 ──┤
카톡 ──┼── 장지호 머릿속 ── ???
발견 ──┤
Claude─┘
```

### 3.2 해법: Flora를 단일 업무 허브로

```
전화 ── 처리 후 "메모해줘" ──┐
메일 ── 확인 후 "메모해줘" ──┤
카톡 ── 확인 후 "메모해줘" ──┼── Flora ── NocoDB ── 자동 정렬
발견 ── "메모해줘" ──────────┤     (태스크 보드)
Claude── PM이 자동 기록 ─────┘
                                    ↓
                              매일 아침 브리핑:
                              "오늘 최우선 3가지는..."
```

**핵심**: 모든 업무를 Flora에게 "메모해줘" 한마디로 던지면, 시스템이 알아서 분류하고 우선순위를 매긴다.

### 3.3 Flora 아침 브리핑 개선

현재 아침 브리핑에 **태스크 우선순위**를 추가:

```
매일 08:30 Flora → 장지호 텔레그램:

"좋은 아침이에요!"

🔴 긴급 (오늘 꼭)
 1. CS 미답변 3건 (24시간 초과)
 2. 쿠팡 정산 마감 D-2

🟡 중요
 3. 메인페이지 수정 (다경 디자인 완료됨 → 스킨 푸시만)
 4. 알리고 알림톡 검수 결과 확인

⏳ 이어서
 5. 파트너클래스 E3 (어제 70%)

📋 직원
 - 다경: 유튜브 썸네일 (어제 요청)
 - 이재혁: 출고 38건
 - 승해: 신상품 상세페이지 3개
```

### 3.4 Flora 점심 체크인 (신규)

```
매일 12:30 Flora → 장지호:

"오전에 #1, #3 처리하셨네요!
 오후 추천: 쿠팡 정산(D-2) → 파트너클래스 이어서"
```

### 3.5 비정기 업무 빠른 처리 패턴

업무가 갑자기 들어올 때, **머릿속에서 꺼내서 시스템에 넣는 데 10초**:

```
[전화 끊고]
장지호 → Flora: "메모해줘: OO업체에서 견적 요청함, 금요일까지"
→ Flora가 NocoDB에 태스크 생성 (priority: 중요, due: 금요일)
→ 내일 아침 브리핑에 자동 포함

[메일 확인 후]
장지호 → Flora: "메모해줘: 정부지원사업 서류 제출 마감 4/15"
→ Flora가 태스크 생성 + 마감 7일 전부터 리마인드 설정

[데이터 보다가 이슈 발견]
장지호 → Flora: "메모해줘: 레진 카테고리 매출 20% 하락, 원인 분석 필요"
→ Flora가 태스크 생성 + CFO 자동 태깅

[개발 중 인터럽트 시]
장지호 → Flora: "메모해줘: 파트너클래스 결제 로직 50%, 다음은 환불 처리"
→ 컨텍스트 보존 → 돌아왔을 때 "환불 처리부터 이어가세요"
```

### 3.6 사내 메신저 전략

**결론: 텔레그램 (업무 자동화 주력) + 카카오톡 (일상 소통 보조)**

7개 메신저 비교 분석 결과, 텔레그램이 PRESSCO21에 최적:

| 기준 | 텔레그램 | 슬랙 | 카카오워크 | 카톡 (현재) |
|------|---------|------|----------|-----------|
| AI 봇 연동 | **Flora 운영 중** | 앱 10개 제한 | 단방향만 | 불가 |
| n8n 연동 | **네이티브 노드** | 네이티브 | HTTP 우회 | 불가 |
| 비용 | **무료** | Pro $58/월 | Free 가능 | 무료 |
| 직원 장벽 | 중간 (설치 필요) | 높음 | 낮음 (카톡 유사) | **최저** |
| 메시지 보존 | **무제한** | 90일 | 무제한 | 무제한 |
| 업무/개인 분리 | 가능 | 가능 | 가능 | **불가** |

**이미 투자된 자산**: Flora 봇(C-Step2), n8n WF 12개+, 텔레그램 그룹 → 추가 비용 0

**운영 구조:**
```
[텔레그램] 업무 자동화
  Flora 봇: 업무 지시, 조회, 리마인드, 완료 추적
  n8n 알림: 주문, 재고, CS, 모니터링
  업무 그룹: 전체/팀별 채팅방

[카카오톡] 일상 소통 (현행 유지)
  간단한 소통, 급한 연락, 생산관리 직원
```

**직원 진입장벽 해결 (단계적)**

| 단계 | 대상 | 내용 |
|------|------|------|
| 1단계 (즉시) | 이재혁 파일럿 | 출고 리스트, 재고 알림 수신만 |
| 2단계 (1주) | 다경/승해 | 기획안 수신 + [완료] 버튼 터치 |
| 3단계 (2주) | 전체 직원 | 알림 수신 + 버튼 응답 (타이핑 불필요) |
| 4단계 (익숙 후) | 자연어 | "재고 확인해줘" 같은 Flora 자연어 대화 |
| **5단계** | **전체** | **텔레그램 미니앱으로 업무 도구 통합** |

### 3.6.1 텔레그램 미니앱 전략

텔레그램 미니앱 = **텔레그램 안에서 열리는 웹 앱**. 별도 앱 설치 없이 Flora 채팅창에서 버튼 하나로 진입.

**왜 미니앱인가:**
- Vanilla HTML/CSS/JS로 개발 (메이크샵과 동일 기술 스택, 빌드 도구 불필요)
- Flora 봇에 메뉴 버튼 추가 = BotFather 5분 설정
- n8n 웹훅 fetch() 호출로 데이터 연동
- NocoDB 데이터를 n8n 프록시 경유 표시
- 비용 0원 (기존 Oracle 서버에 Nginx 호스팅)
- 직원: "플로라 열고 버튼 누르세요" → 풀스크린 업무 화면

**미니앱 아키텍처:**
```
[직원 텔레그램]
  → Flora 봇 [업무도구] 메뉴 버튼
    → 미니앱 (mini.pressco21.com)
      → fetch() → n8n 웹훅 → NocoDB/MakeShop API
      → 화면에 데이터 표시
```

**미니앱 로드맵:**

| 순서 | 미니앱 | 대상 사용자 | 기능 |
|------|--------|-----------|------|
| 1 | **오늘의 출고** | 이재혁 | 출고 리스트 체크, 완료 터치, 재고 알림 |
| 2 | **태스크 보드** | 장지호+전체 | 할일/진행중/완료, 담당자 배정, 마감 리마인드 |
| 3 | **기획안 뷰어** | 다경/승해 | AI 기획안 열람, 코멘트, 완료 보고 |
| 4 | **재고 대시보드** | 이재혁+장지호 | 상품별 재고, 임계치 경고, 발주 버튼 |
| 5 | **주간 리포트** | 장지호 | 매출/CS/마케팅 주간 요약 (차트 포함) |

**연세 직원 UX 설계 원칙:**
- 큰 버튼 (44px 이상), 큰 글씨 (16px 이상)
- 화면당 선택지 3개 이하
- 타이핑 최소화 (버튼 터치로 모든 조작)
- 텔레그램 다크/라이트 테마 자동 대응

**호스팅:**
- 도메인: `mini.pressco21.com` (Nginx + Let's Encrypt)
- 서버: 플로라 서버(158.179.193.173) — Flora 봇과 같은 서버, 여유 자원 충분
- 정적 파일만 (HTML/CSS/JS), 서버 부하 거의 0

### 3.7 직원별 업무 배분 자동화

**장다경 (디자인기획) → Flora로 기획안 자동 전달**
```
장지호 (Claude Code): "신상품 A 상세페이지 기획해줘"
  → 콘텐츠기획관 (opus): 기획안 자동 생성
  → Flora → 다경 텔레그램: "기획안 왔어요 [링크]. 디자인 시작해주세요"
  → 다경 완료 → Flora → 장지호: "완료됐어요"
```

**이재혁 (물류+생산) → Flora로 정리된 업무만 수신**
```
매일 08:00 Flora → 이재혁:
  "출고 42건, 재고 부족 1건 [발주?], CS 확인 2건"
  → 이재혁: 버튼 터치로 승인/확인
```

**조승해 (상세페이지) → Flora로 초안 자동 전달**
```
Claude Code → 콘텐츠기획관: 섹션 구성 + 카피 초안
  → Flora → 승해: "초안이에요. 이 기반으로 제작해주세요"
```

### 3.7 이재혁 과장 업무 경감

이재혁은 물류+생산관리+CS를 혼자 감당. AI가 **정리+자동처리** → 이재혁은 **판단+실행만**:

```
매일 08:00 Flora → 이재혁 텔레그램:

"출고 42건이에요. 특이사항:"
 ⚠️ 실리콘레진 100g 재고 8개 — 발주? [예/아니오]
 📞 CS 12건 자동 처리, 확인 필요 2건:
   1. 배송 지연 (#1234) — 택배사 확인
   2. 반품 요청 (#1235) — 사유 확인
```

| 현재 이재혁 | AI 경감 후 이재혁 |
|-----------|----------------|
| 주문 하나하나 확인 | 출고 리스트 자동 정리 → 확인만 |
| 재고 수동 체크 | 부족 시 자동 알림 → 승인만 |
| CS 전화 전부 받음 | 택배조회/재고문의 AI 자동 → 복잡한 것만 |
| 품절 수동 관리 | 재고 0 → 자동 품절 전환 |

### 3.8 디자인팀 기획안 자동 생성

회사 회의에서 확정된 #1 우선순위: "기획안이 안 와서 막힘" 해결

```
장지호 (Claude Code): "신상품 A 상세페이지 기획해줘"
  ↓
콘텐츠·브랜드 기획관 (opus):
  - 섹션 구성 (헤더→특징→사용법→스펙→추천)
  - 카피 초안 (브랜드 톤: 우아+따뜻+신뢰)
  - 이미지 배치 가이드
  - SEO 메타 태그
  ↓
Flora → 다경/승해 텔레그램: "새 기획안이에요 [링크]. 디자인 시작해주세요"
  ↓
다경/승해: 기획안 위에서 디자인만 집중
  ↓
완료 → Flora → 장지호: "상세페이지 완료됐어요"
```
