# ChatGPT Images 2.0 수동 이미지 자산 파이프라인

작성일: 2026-04-23  
담당 관점: 한지훈님, 정유나님, 최민석님, 윤하늘님

## 0. 결론

PRESSCO21에서 말하는 “덕테이프”는 공식 제품명이라기보다, 2026-04-21 공개된 **ChatGPT Images 2.0**의 강한 이미지 생성/한글 표현/편집 능력을 현장 업무에 빠르게 붙이는 내부 별칭으로 쓴다.

핵심 방향은 **API 자동화가 아니라 ChatGPT Pro 웹/앱 수동 생산 체계**다.

- 추가 API 비용: 0원 유지
- 계정 공유/자동 추출/프로그램 크롤링: 금지
- 생성: 사람이 ChatGPT 웹/앱에서 직접 수행
- 저장: 사람이 다운로드 후 파일명/메타데이터를 붙여 반입
- 검수: 브랜드/상품 사실성/한글 오탈자/모바일 크롭 확인 후 승인
- 활용: 상세페이지, 시즌 배너, 파트너클래스, 협회 제안, 교육자료에 우선 적용

## 2. 팀 회의 종합

### 한지훈님 결론

ChatGPT Images 2.0은 회사 전략의 핵심 엔진이 아니라, 실행 속도를 올리는 **덕테이프**로 써야 한다. 3개월은 “실험”이 아니라 운영 체계화 기간으로 보고, 파트너클래스 운영 지원과 재료/키트 판매 촉진에 먼저 붙인다.

### 정유나님 결론

이미지는 예쁜 장식이 아니라 전환율을 올리는 시각 영업 도구다. 상세페이지 히어로, 시즌 배너, 강사/협회 제안 자료부터 바꿔야 한다. 핵심은 한국어 가독성, 브랜드 일관성, AI 티 제거다.

### 최민석님 결론

자동 생성 시스템이 아니라 **수동 자산 공급망**으로 설계한다. 사람이 ChatGPT에서 생성하고, 검수한 파일만 저장소/프로젝트에 반입한다. 파일 1개마다 메타데이터 1개를 붙여 추적성을 유지한다.

### 윤하늘님 결론

첫 2주는 운영 기준, 프롬프트 라이브러리, 저장/검수 SOP, 샘플 20~30건 파일럿으로 충분하다. 완료 기준은 API 비용 0원, 승인율 70% 이상, 요청부터 승인까지 평균 2영업일 이내다.

## 4. 수동 생산 파이프라인

```text
요청 접수
  → queue.csv 등록
  → 프롬프트 템플릿 선택
  → ChatGPT Pro 웹/앱에서 사람이 생성
  → 후보 1~3안 저장
  → 파일명/메타데이터 작성
  → 1차 검수
  → 필요 시 ChatGPT 이미지 편집 또는 수동 보정
  → 승인본만 프로젝트 자산으로 반입
  → 사용 위치와 성과 기록
```

### 금지선

- OpenAI API로 이미지 생성하지 않는다.
- ChatGPT 계정 공유, 대여, 재판매, 제3자 서비스 구동에 쓰지 않는다.
- 브라우저 자동화로 대량 생성/자동 추출하지 않는다.
- 고객 개인정보, 계약조건, 내부 단가, 협상 내용은 프롬프트에 넣지 않는다.
- 실제 상품 구성/색상/크기와 다른 이미지를 상세페이지의 사실 이미지처럼 쓰지 않는다.
- 협회/강사/공방을 허위 추천·공식 인증처럼 보이게 하지 않는다.

## 6. 파일명 규칙

```text
{project}_{usecase}_{theme}_{YYYYMMDD}_vNN.{png|webp|jpg}
```

예시:

- `partnerclass_banner_spring-course_20260423_v01.webp`
- `makeshopskin_hero_pressedflower-kit_20260423_v02.png`
- `homepage_brandstory_microfleur_20260423_v01.jpg`

규칙:

- 영문 소문자, 숫자, 하이픈만 사용한다.
- 한글 파일명은 메이크샵 업로드 안정성을 위해 피한다.
- 버전은 `v01`, `v02`로 올린다.
- 승인 전 파일은 `draft` 상태로만 관리하고 게시하지 않는다.

## 8. 품질 검수 기준

### A급 — 바로 사용

- 한글 문구 오탈자 없음
- 모바일 썸네일에서도 핵심 메시지 보임
- 실제 상품/수업 정보와 충돌 없음
- 브랜드 톤: 우아, 따뜻, 신뢰, 자연, 포용
- AI 특유의 손/꽃/도구 왜곡 없음
- 이미지 용량과 비율이 사용처에 적합

### B급 — 보정 후 사용

- 크롭/색감/문구 1~2곳만 수정하면 사용 가능
- 배경은 좋으나 CTA 위치 조정 필요
- 텍스트는 HTML/CSS로 분리하면 사용 가능

### C급 — 보류

- 상품 구성 오해 가능
- 브랜드 톤 불안정
- 모바일에서 핵심 요소 잘림
- 한글 문구가 어색함

### D급 — 폐기

- 허위/과장 표현
- 실제 상품과 다른 품질 암시
- 인물/손/제품 왜곡 심함
- 저작권/상표/초상권 리스크가 큼

6. 협회원 전용 혜택 카드

1. 압화 700종+ 전문성 키비주얼

### 메인페이지/브랜드스토리

1. 1982년부터 이어진 꽃공예 연구 헤리티지 이미지

3. 연구/교육/재료 공급을 연결하는 브랜드 다이어그램

### 간편 구매

1. 상품 상세 보조 배너
2. “이런 분께 추천” 카드
3. 옵션 선택 가이드 이미지
4. 배송/구성품 안내 카드

### 내부 교육/CRM

1. CS 응대 FAQ 카드
2. 포장/검수 순서 이미지
3. 신규 직원 교육 썸네일

## 11. 30일 캠페인

### 1주차 — 자산 설계

- 핵심 상품/클래스 10개 선정
- 상세페이지/배너/교육자료/제안서 용도별 템플릿 확정
- 한국어 카피 규칙 통일

### 2주차 — 매출형 소재 제작

- 상위 상품 5개 상세페이지 히어로 후보 제작
- 시즌 배너 3종 제작
- 강사 모집/협회 제안 이미지 2종 제작
- 비교표/FAQ/가이드 이미지 제작

### 3주차 — 실험 운영

- 감성형 vs 정보형 이미지 비교
- 클릭률/문의 전환이 높은 스타일 선별
- 인스타, 카카오, 상세페이지, 제안서에 재배포

### 4주차 — 표준화

- 통과율 높은 프롬프트만 라이브러리에 남김
- 프로젝트별 대표 이미지 규격 확정
- 다음 시즌 상품군으로 확장

## 14. 로컬 검증 도구

이미지 자산을 실제로 반입한 뒤에는 아래 명령으로 메타데이터 기본 규칙을 확인한다.

```bash
python3 _tools/p21-ai-image-asset-check.py assets/manual-ai-images
```

이미지 파일 없이 JSON 메타데이터 형식만 먼저 점검할 때는 아래처럼 실행한다.

```bash
python3 _tools/p21-ai-image-asset-check.py <metadata-folder> --metadata-only
```

검증 도구가 확인하는 것:

- 필수 필드 존재
- `assetId` 파일명 규칙

- 날짜 형식
- 승인본의 `reviewer`, `approvedAt`, `truthCheck` 존재
- 이미지 파일과 메타데이터 파일의 1:1 동봉 여부
