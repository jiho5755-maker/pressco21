# 브랜드페이지 덕테이프 이미지 리디자인 계획

작성일: 2026-04-23  
대상 URL: https://jiho5755-maker.github.io/brand-intro-page/%EB%B8%8C%EB%9E%9C%EB%93%9C%ED%8E%98%EC%9D%B4%EC%A7%80/  
로컬 프로젝트: `브랜드스토리/브랜드페이지/`  
목적: ChatGPT Pro 웹/앱의 ChatGPT Images 2.0을 API 없이 수동 활용해 브랜드스토리 페이지의 이미지 완성도와 신뢰 설득력을 높인다.

## 0. 팀미팅 결론

### 윤하늘님 종합

이번 작업은 바로 코드를 갈아엎는 것이 아니라, **이미지 역할 재설계 → 덕테이프 생성 큐 → 디자이너 수동 생성 → 로컬 반입 → 시각 QA** 순서로 진행한다. 첫 목표는 브랜드페이지 전체를 새로 만드는 것이 아니라, 반복 이미지와 톤 불일치를 줄여 “26년 업력과 기술 신뢰”가 더 빠르게 보이게 만드는 것이다.

### 한지훈님 전략 판단

브랜드스토리 이미지는 예쁜 장식이 아니라 **신뢰를 시각화하는 증거 설계**다. AI 무드컷은 분위기 보조로만 쓰고, 연혁·연구·교육·공공기관·국제교류처럼 사실 증명이 필요한 구간은 실물 사진 기반을 유지한다.

### 정유나님 브랜드 판단

현재 페이지는 감성은 있으나 `workshop.png` 반복 사용 때문에 섹션별 증거감이 약해진다. 히어로, 철학, 레거시는 서로 다른 시각 언어로 분리하고, 갤러리·저서·기술 섹션은 밝기/배경/크롭 규칙을 통일해야 한다.

### 최민석님 기술 판단

로컬 `브랜드스토리/브랜드페이지/`에는 현재 `images/` 폴더가 없고 코드만 `./images/...`를 참조한다. 먼저 안전한 이미지 폴더 구조와 manifest를 잡고, HTML/CSS 참조 교체는 섹션별로 단계 적용한다. 운영 반입 전까지는 상대경로로 로컬 QA, 운영 반입 시 CDN 절대경로 전환이 안전하다.

## 1. 현재 페이지 이미지 감사

Playwright로 운영 URL을 확인한 결과, 주요 이미지 구성은 아래와 같다.

| 위치 | 현재 이미지 | 관찰 | 리디자인 방향 |
|---|---|---|---|
| Hero CSS 배경 | `images/sections/workshop.png` | 히어로, 철학, 레거시에 반복됨 | 대표 상징 이미지로 분리 |
| Philosophy | `workshop.png` | 작업대 이미지가 메시지와 맞지만 히어로 반복으로 밀도 저하 | 손/재료/제작 순간 클로즈업으로 대체 |
| Innovation | `transparent-specimen.png` | 기술 신뢰를 전달하는 핵심 이미지 | 배경/반사/그림자 정리, 더 정밀하게 |
| Publications | `book-1~4.png` | 책 표지 나열 중심 | 책을 오브젝트처럼 촬영한 편집 컷으로 고급화 |
| Gallery | `gallery-1~10.png` | 작품/표본/전시/교육이 혼재 | 카테고리별 밝기·크롭·배경 통일 |
| Legacy | `workshop.png` | 마무리에도 동일 이미지 반복 | 인물성+대표작+축적된 시간의 엔딩 컷으로 교체 |

추가 확인:

- 원격 이미지들은 대부분 1024x1024 PNG다.
- CSS의 히어로 배경은 `css/heritage.css`와 `css.css`에서 `../images/hero/hero-main.png` 주석/참조 흐름이 있으나, 운영 URL에서는 `workshop.png`가 배경으로 잡혀 있다.
- 로컬에는 `브랜드스토리/브랜드페이지/images/`가 없어 실제 이미지 반입 단계가 필요하다.

## 2. 리디자인 원칙

### 2-1. 실물 사진 보존 구간

아래 섹션은 실제 사진/기록/작품을 기반으로 해야 신뢰가 유지된다.

- Timeline
- Achievements
- Innovation
- Education
- International
- Publications
- Gallery

허용 작업:

- 배경 정리
- 색감 통일
- 조명 개선
- 여백 확장
- 작은 먼지/잡음 제거
- 크롭 재구성

금지 작업:

- 실제 작품 형태 변경
- 없는 수상/기관/로고 생성
- 인물 얼굴 임의 생성
- 실제 전시·교육 현장처럼 보이는 가짜 장면 생성

### 2-2. AI 무드컷 허용 구간

아래 섹션은 사실 증명보다 분위기/서사 보조 역할이므로 AI 무드컷을 제한적으로 허용한다.

- Hero
- Philosophy 보조 배경
- Section transition background
- Legacy 보조 컷

단, 무드컷도 PRESSCO21의 실제 압화/꽃공예 세계관에서 벗어나면 안 된다.

## 4. 덕테이프 작업 루틴

### 4-1. 디자이너 실행 루틴

1. 원본 사진을 1장 선택한다.
2. 아래 프롬프트 템플릿 중 섹션에 맞는 것을 복사한다.
3. ChatGPT Pro 웹/앱에서 원본 사진을 업로드한다.
4. ChatGPT Images 2.0 또는 Images with thinking이 가능하면 Thinking/Pro 모델 흐름으로 3안을 만든다.
5. 3안 중 1개만 선택한다.
6. 선택한 1안에 대해 수정 요청은 최대 2회만 한다.
7. 결과 PNG를 저장하고 WebP 최적화본을 별도 만든다.
8. `images/_manifest.json`에 사용처와 원본/프롬프트를 기록한다.

### 4-2. 공통 안전 프롬프트

모든 사진 기반 작업 앞에 붙인다.

```text
첨부한 실제 사진을 기준으로 작업해 주세요.
실제 작품/제품/책/사진 속 대상의 형태, 색상, 질감, 크기, 꽃의 종류, 구성품, 글자, 로고는 절대 바꾸지 마세요.
바꿔도 되는 것은 배경, 조명, 그림자, 주변 소품, 여백, 전체 분위기뿐입니다.
이 이미지는 PRESSCO21 브랜드스토리 페이지에 사용할 실제 기반 이미지입니다.
실제 사실을 왜곡하거나 없는 수상, 기관, 로고, 인물을 만들지 마세요.
```

## 6. 로컬 파일 적용 계획

### 6-1. 권장 폴더 구조

```text
브랜드스토리/브랜드페이지/images/
  raw/
    20260423/
  optimized/
    png/
    webp/
  hero/
  sections/
  gallery/
  books/
  _manifest.json
```

### 6-2. 교체 원칙

- 원본 생성물은 `raw/`에 보관한다.
- 페이지에서 직접 참조하는 파일은 `hero/`, `sections/`, `gallery/`, `books/`에 둔다.
- 배포용은 WebP 우선, PNG fallback을 준비한다.
- CSS 배경 이미지는 먼저 PNG로 안정 검증 후 WebP 전환을 검토한다.
- 운영 반입 전에는 `./images/...` 상대경로로 QA한다.
- 메이크샵/운영 반입 시 CDN 절대경로로 변환한다.

### 6-3. 교체 대상 예시

| 코드 위치 | 현재 | 후보 |
|---|---|---|
| `css/heritage.css` hero background | `../images/hero/hero-main.png` 또는 운영상 `workshop.png` | `../images/hero/brandstory_hero_heritage-lab.webp` |
| `index.html` philosophy image | `./images/sections/workshop.png` | `./images/sections/brandstory_section_hands-pressedflower.webp` |
| `index.html` innovation image | `./images/sections/transparent-specimen.png` | `./images/sections/brandstory_section_transparent-specimen.webp` |
| `index.html` legacy image | `./images/sections/workshop.png` | `./images/sections/brandstory_legacy_archive-table.webp` |
| `index.html` gallery images | `gallery-1~10.png` | 같은 파일명 유지 또는 v2 파일명으로 단계 교체 |
