# 덕테이프(ChatGPT Images 2.0) 활용 1페이지 보고

작성일: 2026-04-23

## 방향

ChatGPT Images 2.0을 PRESSCO21 내부에서는 “덕테이프”처럼 쓴다. 핵심은 API 개발이 아니라, ChatGPT Pro 웹/앱에서 사람이 고품질 이미지를 만들고 검수해 비즈니스 자산으로 빠르게 붙이는 것이다.

## 원칙

1. 추가 API 비용 0원
2. ChatGPT 웹/앱에서 사람이 직접 생성·저장
3. 계정 공유, 자동 추출, 브라우저 자동화 금지
4. 이미지 1장마다 메타데이터 1개 기록
5. 승인 전 이미지는 운영 페이지에 올리지 않음
6. 실제 상품/수업과 다른 연출은 “연출 이미지”로만 사용

## 우선 적용

| 순위 | 영역 | 산출물 |
|---|---|---|
| 1 | 파트너클래스 | 상세 히어로, 수강생 불안 해소 카드, 강사 모집 배너 |
| 2 | 자사몰 재료/키트 | 시즌 기획전, 완성작 예시, 구성품 비교표 |
| 3 | 협회/파트너 | 협회원 혜택, 공동 홍보, 세미나 카드 |
| 4 | 내부 교육 | 작업 순서, CS FAQ, 신규 직원 교육 이미지 |

## 첫 2주 산출물

- 운영 원칙 1장
- 프롬프트 라이브러리 v1
- 파일명/메타데이터 규칙
- 검수 체크리스트
- 파일럿 샘플 20~30건
- 주간 운영 리포트 템플릿

## 30일 목표

- 상세페이지/배너/제안서 10곳 이상에 승인 이미지 적용
- 1차 검수 통과율 70% 이상
- 요청→승인 평균 2영업일 이내
- 추가 API 비용 0원 유지
- 승인된 프롬프트 재사용률 60% 이상

## 담당 관점별 결론

- 한지훈님: 전략 엔진이 아니라 실행 속도를 올리는 덕테이프다.
- 정유나님: 예쁜 이미지보다 전환율을 올리는 시각 영업 도구로 써야 한다.
- 최민석님: 자동화가 아니라 수동 자산 공급망으로 설계해야 한다.
- 윤하늘님: 첫 2주는 운영 기준과 파일럿 20~30건이면 충분하다.

## 바로 다음 행동

1. 파일럿 1순위를 파트너클래스로 확정한다.
2. 후보 이미지 25건을 queue.csv에 등록한다.
3. ChatGPT Pro 웹/앱에서 사람이 1차 후보를 생성한다.
4. review-checklist.md로 A/B/C/D 판정한다.
5. 승인본만 실제 프로젝트 자산으로 반입한다.
