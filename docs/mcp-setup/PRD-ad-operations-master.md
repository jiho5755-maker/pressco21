## 5. 채널별 전략 상세

### 5.1 네이버 SA (검색광고) — MCP 자동화

| 항목 | 내용 |
|------|------|
| MCP | naver-ads-mcp (자체구축, 33→54개 도구) |
| 현재 | 파워링크 1 + 쇼핑검색 7 (활성), 키워드 28개 |
| 예산 | 파워링크 10,000 + 쇼핑전략 15,000 + 벌크 3,000 = 28,000원/일 |
| 전략 | 벌크 자동 사이클 + 키워드 자동 발굴 + A/B 테스트 |
| 측정 | MCP stat API (완전 자동) |
| 목표 ROAS | 260% (3개월) |

**핵심 기능 (Phase 2)**:
- 키워드 자동 추출 + 검색량/경쟁도/CPC 데이터
- 입찰 시뮬레이터 (경쟁사 중간값 비교)
- 벌크 주간 자동 사이클 (수확→필터→등록→분석→가지치기)
- 광고 문구 데이터 기반 추천
- 제외 키워드 + 품질지수 + 소예산 방어

### 5.2 네이버 GFA (성과형 디스플레이) — 단계적 진입

| 항목 | 내용 |
|------|------|
| MCP | API 승인 대기 → 승인 후 naver-ads-mcp에 6개 도구 추가 |
| 현재 | 미시작 |
| 예산 | 카탈로그 3,000 + 전환(자사몰) 5,000 = 8,000원/일 |
| 전략 | ① 카탈로그(리타겟) 즉시 ② 전환추적 후 자사몰 배너 |
| 측정 | GFA 대시보드 + UTM + GA4 |

**진입 순서**:
1. 카탈로그 판매 (전환추적 불필요, 즉시 시작)
2. 전환추적 확인 후 → 웹사이트 전환 캠페인 (자사몰 랜딩)

**API 불가 확정 (2026-04-17)**: 베타 파트너 전용. MCP 통합 불가.
수동 운영 + GA4 MCP 간접 분석으로 효율 측정. 상세: `guide-gfa-adboost-manual.md`

**킬 스위치**: 7일 ROAS 100% 미만 → 즉시 OFF

### 5.3 Meta Ads (Facebook/Instagram) — 오픈소스 MCP

| 항목 | 내용 |
|------|------|
| MCP | pipeboard-co/meta-ads-mcp (Stars 772, Apache 2.0) |
| 현재 | 대행사가 운영 중이었으나 현황 불명 |
| 키 필요 | App ID + Secret + 장기 토큰(60일) + 광고계정 ID |
| 예산 | 기존 집행 현황 파악 후 결정 |
| 전략 | MCP 설치 → 기존 캠페인 현황 전수조사 → 비효율 정리 → 유지/축소/OFF 판단 |
| 측정 | meta-ads-mcp stat 도구 (자동) |

**첫 번째 할 일 (키 발급 후)**:
```
1. meta-ads-mcp 설치
2. "현재 메타 광고 전체 캠페인 보여줘" → 현황 파악
3. 캠페인별 ROAS 조회 → 적자 캠페인 식별
4. 적자 = OFF, 흑자 = 유지, 미측정 = 전환추적 설정 후 재판단
```

**MCP 주요 도구 (오픈소스 제공)**:
- 캠페인/광고세트/광고 CRUD
- 인사이트 (ROAS, CPC, CTR, 전환)
- 오디언스 관리
- 예산 조정

**주의**: Meta 토큰은 60일 만료 → 갱신 일정 관리 필수 (2026-06-16 알림)

### 5.4 Google Ads — 오픈소스 MCP (승인 대기)

| 항목 | 내용 |
|------|------|
| MCP | googleads/google-ads-mcp (Stars 378) |
| 현재 | 운영 여부 미확인 |
| 키 필요 | Developer Token (MCC) + OAuth + MCC ID |
| 예산 | 현황 파악 후 결정 |
| 전략 | Meta와 동일 — 먼저 현황 파악 → 비효율 정리 → 유지/축소 판단 |
| 측정 | google-ads-mcp 리포트 도구 |

**Developer Token 승인 절차**:
1. Google Ads MCC 계정에서 API 센터 접속
2. Basic Access 신청 (signup-checklist 1번 참고)
3. 승인 1~7일 대기
4. 승인 후 google-ads-mcp 설치 + 연동

**승인 전에도 할 수 있는 것**:
- Google Ads 웹 UI에서 현재 캠페인 현황 수동 확인
- 실행 중인 캠페인이 있다면 ROAS 수동 점검

### 5.5 GA4 (Google Analytics) — 분석 허브

| 항목 | 내용 |
|------|------|
| MCP | google-analytics-mcp (Stars 1.2k, 공식) |
| 역할 | 광고 채널이 아니라 **전 채널 측정 허브** |
| 키 필요 | 속성 ID + 서비스계정 JSON + OAuth JSON |
| 전략 | 모든 광고의 유입→전환을 GA4에서 통합 측정 |

**GA4가 중요한 이유**:
- 네이버 SA → MCP stat으로 자체 측정 가능
- GFA → 전환추적 + UTM으로 측정
- Meta/Google → 각 MCP로 자체 측정
- **하지만 "자사몰에서 실제로 구매까지 갔는가"는 GA4만 알 수 있음**

**GA4 MCP로 할 수 있는 질문**:
- "이번 달 자사몰 매출 중 네이버 SA 비중이 몇 %야?"
- "Meta에서 온 사람과 네이버에서 온 사람 중 누가 더 많이 사?"
- "파트너클래스 페이지 이탈률이 어때?"

### 채널별 관리 방식

| 채널 | 도구 | 자동화 수준 | 정유나 역할 |

| 네이버 SA | naver-ads-mcp | 높음 (MCP) | 전략 결정, 벌크 사이클 감독 |
| 네이버 GFA | MCP (승인 후) / 수동 (승인 전) | 중간 | 타겟팅/소재/예산, 킬 스위치 |
| Meta Ads | meta-ads-mcp | 높음 (MCP) | 캠페인 전략, ROAS 모니터링 |
| Google Ads | google-ads-mcp | 높음 (MCP) | 캠페인 전략, ROAS 모니터링 |
| GA4 | google-analytics-mcp | 읽기 전용 | 전 채널 통합 분석 |
| 애드부스트 | 수동 | 낮음 | 주 1회 확인 |
