## 2. Phase 2 목표

### 2.1 한 줄 요약

Phase 1의 "보고 바꾸는" 도구에 **"판단하는" 도구**를 추가하여, 비전공자 대표가 자연어로 광고를 최적화할 수 있게 한다.

### 2.2 해결할 문제

| 문제 | Phase 1 | Phase 2 |
|------|---------|---------|
| 키워드 발굴 | 불가 (수동 입력만) | 시드 키워드 → 연관 200개 자동 추출 |
| 입찰가 적정성 | 모름 | 시뮬레이션 + 경쟁사 중간값 비교 |
| 예산 낭비 | 감지 불가 | 제외키워드 자동관리 + 품질지수 감시 |
| 시장 트렌드 | 감으로 | 데이터랩 시계열 데이터 기반 |
| 광고 문구 | 직접 작성 | 데이터 기반 컨텍스트 → Claude 추천 |
| 벌크 전략 | 수동 뿌리기 | 자동 수확→필터→등록→분석→가지치기 사이클 |
| 소예산 방어 | 없음 | 시간대별 소진 감시 + 자동 일시정지 |

### 2.3 성공 지표

| 지표 | 현재 | 3개월 목표 | 측정 |
|------|------|-----------|------|
| 광고 관리 시간 | 주 8시간 | 주 2시간 | 지호님 체감 |
| 키워드 수 | 28개 | 80~150개 | keyword_list 카운트 |
| 예산 낭비율 | 추정 15~20% | 5% 이하 | 제외키워드 차단 + QI 개선 |
| ROAS 반응 속도 | 주 1회 수동 | 일 1회 자동 제안 | MCP 호출 로그 |
| 자사몰 비중 | 34% | 36~38% | 월간 매출 비중 |

판정 기준:
- QI 7~10: 효율 좋음, 유지
- QI 4~6: 개선 필요 (소재/랜딩페이지 점검 권고)
- QI 1~3: 비효율 (소재 교체 또는 키워드 제거 권고)

- 소진율 80% 이상: 경고 텔레그램 발송
- 소진율 95% 이상: 자동 일시정지 제안

## 5. 벌크 전략 자동 사이클

### 5.1 현재 (수동)

지호님이 직접 키워드를 넣고, 최저 입찰가(70원)로 등록, 성과를 엑셀로 확인.

### 5.2 Phase 2 이후 (자동)

```
매주 월  keyword_discover("압화","프리저브드","보존화","레진"...)
  ↓     → 연관 키워드 수백 개 수확
자동    bulk_keyword_filter(min_search=100, max_cpc=200)
  ↓     → 검색량 100+, CPC 200원 이하만 통과
확인 후  keyword_create_bulk(adgroup_id, keywords, bidAmt=70)
  ↓     → 벌크 캠페인에 최저가 등록 (dry_run → 승인)
매주 금  bulk_performance_audit(campaign="벌크")
  ↓     → 전환 키워드 하이라이트 + 비전환 키워드 목록
매월 1일 bulk_auto_prune(days=30, min_conversions=0)
  ↓     → 30일 전환 0 키워드 OFF (dry_run → 승인)
자동    bulk_bid_optimizer(convert_keywords, delta=+15)
         → 전환 키워드만 입찰가 +15원 인상
```

### 5.3 핵심 원칙

- SA는 클릭 전환(CPC) → 클릭 안 되면 비용 0원 → 많이 뿌릴수록 "어걸리는" 확률 증가
- 최저가(70원) 등록이므로 리스크 최소
- 주간 가지치기로 예산 누수 방지
- "어걸린" 키워드는 자동 강화 (입찰가 소폭 인상)

### 6.3 근거

- 지호님 방침 "예산 최소, 세팅만"과 정합
