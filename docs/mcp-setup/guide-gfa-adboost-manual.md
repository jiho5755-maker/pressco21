# GFA + 애드부스트 수동 운영 가이드

> **상태**: GFA API 불가 확정 (2026-04-17, 베타 파트너 전용). 수동 운영 확정.
> **대상**: 장지호 팀장 (직접 세팅) + CMO 정유나 (주간 모니터링)
> **원칙**: 측정 안 되면 돈 안 쓴다. 7일 단위로 판단, 적자면 즉시 OFF.

## Part 1: GFA (네이버 성과형 디스플레이 광고)

### 1. 운영 구조

API 없음 → **GFA 웹 대시보드 + UTM + GA4**로 수동 관리.

```
[세팅]  gfa.naver.com에서 직접 캠페인 생성
[측정]  GFA 대시보드 (노출/클릭/CPC) + GA4 (전환)
[판단]  주 1회 금요일 성과 체크
[조치]  ROAS 100% 미만 → OFF / 300%+ → 예산 증액
```

### 2. 카탈로그 판매 캠페인 (즉시 시작)

전환추적 없이 시작 가능. 스마트스토어 내부 측정.

**세팅:**
1. gfa.naver.com → 캠페인 만들기 → **카탈로그 판매**
2. 카탈로그 관리 → 스마트스토어 비즈채널 연결 → 카탈로그 생성
3. 일예산: **3,000원**
4. 타겟: **자동** (AI 다이나믹 리타겟팅)
5. 소재: **자동** (상품DB 연동, 직접 만들 필요 없음)
6. 저장 → 검수 (1영업일) → 자동 시작

**왜 카탈로그부터:**
- 소재 제작 불필요 (자동)
- 이미 내 상품을 본 사람에게 다시 노출 (리타겟) → 전환율 높음
- 전환추적 불필요 (스마트스토어 내부 자동)
- 업계 평균 ROAS 350~500%

### 3. 웹사이트 전환 캠페인 (전환추적 확인 후)

**시작 조건**: 메이크샵 네이버 전환추적(픽셀) 설치 확인됨.

**세팅:**
1. 캠페인 만들기 → **웹사이트 전환**
2. 일예산: **5,000원**
3. 과금: **CPC** (클릭당 과금)
4. 입찰: **자동** (AI 학습, 처음 2주)

**광고그룹 3개 테스트:**

| 그룹 | 타겟 | 소재 방향 | 랜딩 URL |
|------|------|----------|---------|
| A. 꽃공예 | 여성 25~44 / 관심사: 핸드메이드+꽃 | 인기 상품 이미지 | foreverlove.co.kr + UTM |
| B. 인테리어 | 전체 25~49 / 관심사: 인테리어 | 프리저브드 이미지 | foreverlove.co.kr + UTM |
| C. 교육 | 여성 30~54 / 관심사: 자격증 | 파트너클래스 이미지 | foreverlove.co.kr + UTM |

**UTM 형식** (반드시 붙이기):
```
그룹A: ?utm_source=naver_gfa&utm_medium=display&utm_campaign=flower_craft
그룹B: ?utm_source=naver_gfa&utm_medium=display&utm_campaign=interior
그룹C: ?utm_source=naver_gfa&utm_medium=display&utm_campaign=partnerclass
```

**소재 제작 원칙:**
- 그룹당 소재 **3개** (최소 비교 단위)
- 이미지: 실제 상품 사진 (스톡 금지), 1200x628
- 제목 15자: 상품명 + 브랜드
- 설명 45자: USP 1줄 (가격/할인/전문성)
- **2주마다 하위 소재 교체** (피로도 관리)

### 4. 측정 방법

**GFA 대시보드 (주 1회 금요일):**

gfa.naver.com → 캠페인 → 성과

| 보는 지표 | 위치 |
|----------|------|
| 노출수 | 대시보드 기본 |
| 클릭수 / CTR | 대시보드 기본 |
| CPC | 대시보드 기본 |
| 전환수 | 전환추적 설치 시 표시 |
| 비용 | 대시보드 기본 |

**GA4 (자사몰 전환 확인):**

GA4 → 보고서 → 획득 → 트래픽 획득 → 소스/매체 필터: `naver_gfa / display`

| 보는 지표 | 의미 |
|----------|------|
| 세션수 | GFA에서 자사몰에 온 사람 수 |
| 참여율 | 왔다가 바로 나간 비율 (낮으면 소재/랜딩 문제) |
| 전환수 | 구매 완료 수 |
| 수익 | 전환 매출 |

**ROAS 계산 (수동):**
```
ROAS = GA4 수익 ÷ GFA 대시보드 비용 × 100

예: GA4 수익 150,000원 / GFA 비용 35,000원 = ROAS 429%
```

### 5. 판단 기준 (7일 단위)

| 지표 | 유지 | 소재 교체 | 즉시 OFF |
|------|------|----------|---------|
| CTR | 0.3%+ | 0.1~0.3% | 0.1% 미만 |
| CPC | 300원 이하 | 300~500원 | 500원 초과 |
| 전환 | 1건+ | 클릭 50+ 전환 0 → 랜딩 점검 | 클릭 100+ 전환 0 |
| ROAS | 200%+ | 100~200% → 2주 관찰 | **100% 미만 → 즉시 OFF** |

**그룹별 판단 후 액션:**
```
그룹A ROAS 350% → 예산 +2,000원
그룹B 전환 0건  → OFF, 예산을 A에 이동
그룹C ROAS 180% → 소재 교체 후 2주 재관찰
```

### 6. 주간 체크 루틴 (매주 금요일, 15분)

```
□ gfa.naver.com 접속
□ 각 캠페인/그룹 7일 성과 확인
□ CTR 0.1% 미만 그룹 → OFF
□ CPC 500원 초과 그룹 → 타겟 축소 또는 OFF
□ GA4에서 naver_gfa 전환 확인
□ ROAS 계산 (GA4 수익 / GFA 비용)
□ ROAS 100% 미만 → 즉시 OFF
□ 하위 소재 삭제 → 새 소재 업로드 (2주마다)
□ 결과를 정유나 CMO에게 공유 (텔레그램 또는 미니앱)
```

## Part 2: 애드부스트

### 1. 운영 구조

API 없음. 스마트스토어센터 UI에서만 관리. AI 자동 최적화.

```
[세팅]  스마트스토어센터에서 ON + 상품 선택 + 예산
[측정]  스마트스토어센터 광고 리포트 + 커머스 MCP 간접
[판단]  월 1회 (데이터 축적 느림)
[조치]  ROAS 2.0 미만 → OFF
```

### 2. 시작 조건

SA + GFA가 먼저 안정화된 후. 급하지 않음.

### 3. 세팅

1. sell.smartstore.naver.com → 마케팅 → 광고 관리 → 애드부스트
2. **시작하기** 클릭
3. 일예산: **5,000원**
4. 광고 상품 선택 기준:

```
우선순위 (AND 조건):
1. 최근 30일 판매 상위 30개
2. 리뷰 10개 이상
3. 재고 100개 이상
4. 판매가 10,000원 이상
```

5. 전환 추적: **구매 전환** 선택
6. 게재 위치: **자동**
7. 저장

### 4. 측정 방법

**스마트스토어센터 리포트 (월 1회):**

마케팅 → 광고 관리 → 성과 리포트

| 지표 | 확인 |
|------|------|
| 노출수 | 주 10,000+ 정상 |
| 클릭수 | 주 100+ 정상 |
| 전환수 | 주 5+ 정상 |
| 광고비 | 예산 내 소진 확인 |
| ROAS | 리포트에 표시됨 |

**커머스 MCP 간접 분석 (선택):**

naver-ads-mcp의 `commerce_order_list`로 ON/OFF 전후 매출 비교:
```
1주 OFF → 매출 기록 (기준선)
1주 ON → 매출 기록
차이 = 애드부스트 기여 매출 (추정)
ROAS = 기여 매출 / 광고비
```

### 5. 판단 기준 (1개월 단위)

| 조건 | 액션 |
|------|------|
| ROAS 3.0+ | 유지, 일예산 10,000원 증액 검토 |
| ROAS 2.0~3.0 | 광고 상품 TOP 30 재선정 |
| ROAS 2.0 미만 | **OFF** |
| 자사몰 매출 감소 동반 | **주의** — 스마트스토어로만 유입 가속은 자사몰 목표와 역행 |

### 6. 월간 체크 루틴 (매월 1일, 10분)

```
□ 스마트스토어센터 → 광고 리포트 → 월간 요약
□ ROAS 확인
□ 2.0 미만이면 OFF
□ 광고 상품 판매 순위 변동 확인 → TOP 30 재선정
□ 자사몰 vs 스마트스토어 매출 비중 확인 (역행 체크)
```
