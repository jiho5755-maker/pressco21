## 1. 네이버 GFA (성과형 디스플레이 광고)

### 1.1 현황: API 있음, 파트너 승인 필요

GFA는 SA(검색광고)와 **별도 전용 API**가 존재합니다.

| 항목 | 내용 |
|------|------|
| API 문서 | https://naver-ad-api.github.io/openapi-guide/ |
| Base URL | `https://openapi.naver.com/v1/ad-api/{version}/adAccounts/{adAccountNo}/` |
| 상태 | Beta, **파트너사 전용** (일반 광고주 직접 접근 불가) |
| 기능 | 캠페인/광고그룹/소재 CRUD, 타겟팅, 성과 조회 |
| 인증 | 네이버 개발자센터 앱 + 파트너 승인 |

### 1.2 Plan A: MCP 통합 (권장)

**Step 1 — 파트너 승인 신청 (지호님 직접)**

1. https://gfa.naver.com (또는 https://saedu.naver.com → GFA) 접속
2. 광고 계정 로그인
3. 고객센터/문의에서 "API 파트너 등록 요청" 문의
   - 용도: "자사 광고 자동화 (내부 사용, 제3자 제공 아님)"
   - 회사명: PRESSCO21
   - 개발자센터 앱 ID: `pressco21-mcp` (이미 등록됨)
4. 승인 대기 (보통 1~2주)

**Step 2 — 승인 후 MCP 확장**

파트너 승인이 나오면 naver-ads-mcp에 GFA 도구를 추가합니다:

```
추가 도구 (예상 6개):
- gfa_campaign_list     — GFA 캠페인 조회
- gfa_campaign_create   — GFA 캠페인 생성
- gfa_adgroup_manage    — 광고그룹 + 타겟팅 설정
- gfa_creative_list     — 소재 조회 (생성은 API 미지원, 조회/수정만)
- gfa_stat_report       — 성과 리포트 조회
- gfa_budget_adjust     — 예산 조정 (dry_run 지원)
```

기존 naver-ads-mcp에 `clients/gfa.py` + `tools/gfa.py` 추가로 구현 가능.
인증은 네이버 개발자센터 Client ID/Secret 활용 (이미 발급 완료).

**Step 3 — 정유나 CMO가 관리**

MCP 통합 후 정유나님이 다음을 자동화:
- 파트너클래스 홍보 배너 → 자사몰 랜딩
- 타겟팅: "꽃꽂이", "공예", "핸드메이드" 관심사
- 일예산 5,000~15,000원에서 ROAS 3.0+ 목표
- 주간 성과 리포트 자동 수집

### 1.3 GFA 세팅 가이드 (파트너 승인 전, 수동 운영)

파트너 승인 대기 중에도 GFA 웹 UI로 운영을 시작할 수 있습니다.

**A. GFA 계정 확인**

1. https://gfa.naver.com 접속 (네이버 광고 계정으로 로그인)
2. 좌측 **광고 관리** → **캠페인 만들기**
3. 기존 캠페인이 없으면 신규 생성

**B. PRESSCO21 추천 GFA 세팅**

| 설정 | 추천값 | 이유 |
|------|--------|------|
| 캠페인 목적 | **웹사이트 트래픽** | 자사몰 유입이 목적 |
| 일예산 | **5,000원** (1단계 시작) | CFO 권고: GFA는 15만/월로 시작 |
| 과금 방식 | **CPC** (클릭당 과금) | 소예산에서는 CPC가 안전 |
| 입찰 전략 | **자동 입찰** | 네이버 AI가 CPC 최적화 |
| 타겟팅 - 성별 | 여성 중심 (7:3) | 꽃공예 주 수강생 |
| 타겟팅 - 연령 | 25~54세 | 핵심 구매층 |
| 타겟팅 - 관심사 | 핸드메이드/공예, 꽃/원예, 인테리어, 선물 | |
| 게재 위치 | 네이버 메인/카페/밴드 | 관심사 기반 노출 |
| 랜딩 URL | `https://foreverlove.co.kr` (자사몰) | 오픈마켓 아님! |

**C. 소재(배너) 제작 가이드**

| 소재 유형 | 크기 | 내용 제안 |
|----------|------|----------|
| 이미지 배너 | 1200x628 | 파트너클래스 수강 안내 + 자사몰 혜택 |
| 이미지 배너 | 1080x1080 (정사각) | 인기 상품 3종 모음 |
| 반응형 소재 | 자동 | 상품 이미지 + 짧은 카피 |

카피 예시:
- "꽃으로 노는 모든 방법 — 프레스코21 파트너클래스"
- "압화·프리저브드·레진 전문 재료, 강사 회원 최대 25% 할인"
- "30년 노하우 프레스코21에서 시작하세요"

### 1.4 GFA 효율 분석 방법 (MCP 통합 전)

MCP 통합 전에도 효율을 측정할 수 있습니다:

**수동 분석 (주 1회)**
1. GFA 관리 화면 → 캠페인 → **성과 다운로드 (CSV)**
2. 확인할 핵심 지표:

| 지표 | 목표 | 판단 |
|------|------|------|
| CTR (클릭률) | 0.3% 이상 | 0.3% 미만이면 소재 교체 |
| CPC (클릭당 비용) | 200원 이하 | 200원 초과면 타겟팅 조정 |
| 전환수 | 주 5건 이상 | 0건이면 랜딩 URL 점검 |
| ROAS | 3.0 이상 | 2.0 미만이면 예산 축소 검토 |

**GA4 연동 (추천)**
- GFA 캠페인 URL에 UTM 파라미터 추가:
  `?utm_source=naver_gfa&utm_medium=display&utm_campaign=partnerclass`
- GA4에서 유입→전환 추적

## 2. 네이버 애드부스트

### 2.1 현황: API 없음, 수동 관리

| 항목 | 내용 |
|------|------|
| API | **없음** (커머스 API에 광고 엔드포인트 미제공) |
| 관리 | 스마트스토어센터 UI에서만 설정 가능 |
| 특징 | 네이버 AI가 타겟/소재/게재위치를 자동 최적화 |
| 제어 범위 | ON/OFF, 일예산, 광고 상품 선택 정도만 |

### 2.2 애드부스트 세팅 가이드

**A. 애드부스트 활성화**

1. 스마트스토어센터 (https://sell.smartstore.naver.com) 로그인
2. 좌측 메뉴 **마케팅** → **광고 관리** → **애드부스트**
3. **시작하기** 클릭
4. 비즈채널 연결 확인 (스마트스토어가 자동 연결됨)

**B. PRESSCO21 추천 세팅**

| 설정 | 추천값 | 이유 |
|------|--------|------|
| 일예산 | **5,000원** | 최소 테스트 예산 |
| 광고 상품 | **인기 TOP 30개만** | 전체 1,117개 중 전환율 높은 상품 집중 |
| 전환 추적 | **구매 전환** | 장바구니 아닌 실구매 기준 |
| 게재 위치 | **자동 (AI 판단)** | 애드부스트의 핵심이 AI 자동 최적화 |

**C. 광고 상품 선택 기준 (TOP 30)**

naver-ads-mcp의 `commerce_product_list`로 조회한 1,117개 상품 중:

```
선택 우선순위:
1. 최근 30일 판매 상위 30개 (스마트스토어센터 → 상품관리 → 판매순 정렬)
2. 리뷰 10개 이상 (사회적 증거 → 전환율 높음)
3. 재고 100개 이상 (품절 방지)
4. 판매가 10,000원 이상 (광고비 대비 수익성)
```

### 2.3 애드부스트 효율 분석 방법

API가 없으므로 **커머스 API 주문 데이터 + 스마트스토어센터 리포트**를 조합합니다.

**방법 1: MCP로 간접 분석 (자동화 가능)**

naver-ads-mcp의 `commerce_order_list` 도구로:
1. 주문 목록 조회 (기간별)
2. "유입경로" 필드가 있으면 → 애드부스트 유입 주문 필터링
3. 없으면 → 애드부스트 ON/OFF 전후 매출 비교 (A/B 방식)

```
분석 스크립트 로직:
1. 애드부스트 OFF 1주간 매출 기록 (기준선)
2. 애드부스트 ON 1주간 매출 기록
3. 차이 = 애드부스트 기여 매출 (추정)
4. 기여 매출 / 광고비 = ROAS
```

**방법 2: 스마트스토어센터 수동 확인 (주 1회)**

1. 스마트스토어센터 → **마케팅** → **광고 관리** → **성과 리포트**
2. 확인할 핵심 지표:

| 지표 | 목표 | 판단 |
|------|------|------|
| 노출수 | 주 10,000+ | 미달이면 예산 증액 또는 상품 추가 |
| 클릭수 | 주 100+ | 미달이면 상품 이미지/가격 점검 |
| 전환수 | 주 5+ | 0이면 상품 선택 재검토 |
| 광고비 | 35,000원/주 | 일예산 5,000원 기준 |
| ROAS | 3.0+ | 2.0 미만이면 상품 교체 |

### 2.4 효율 판단 체크리스트 (1개월 후)

| 체크 | 기준 | 액션 |
|------|------|------|
| ROAS 3.0 이상? | 유지 | 일예산 10,000원으로 증액 검토 |
| ROAS 2.0~3.0? | 관찰 | 광고 상품 TOP 30 재선정 |
| ROAS 2.0 미만? | 축소 | 일예산 3,000원으로 축소 또는 OFF |
| 자사몰 매출에 영향? | 비교 | 스마트스토어 매출↑ but 자사몰↓이면 목적에 역행 |

마지막 체크가 중요합니다. 애드부스트는 **스마트스토어 매출을 올리는 도구**이지, 자사몰 유입 도구가 아닙니다. 자사몰 비중 확대 목표와 상충할 수 있으므로, 스마트스토어 전체 매출 대비 광고 의존도를 모니터링해야 합니다.

## 3. 종합 액션 플랜

### Plan A (최선): GFA를 MCP에 통합

```
지금     GFA 파트너 API 승인 신청 (지호님)
  ↓
1~2주    승인 대기 중 수동 GFA 운영 시작 (위 가이드대로)
  ↓
승인 후   naver-ads-mcp에 GFA 도구 6개 추가 (CTO)
  ↓
이후     정유나 CMO가 SA + GFA 통합 관리
```

### Plan B (현실): 애드부스트 수동 + MCP 간접 분석

```
지금     애드부스트 ON (위 세팅 가이드대로)
  ↓
1주 후   commerce_order_list로 매출 변화 분석
  ↓
1개월    ROAS 판단 → 유지/증액/축소 결정
```

### 정유나 CMO 관리 범위

| 광고 | 관리 방식 | 정유나 역할 |
|------|----------|-----------|
| SA 파워링크 | MCP 자동 | 키워드/입찰/소재 전략 결정 |
| 쇼핑검색광고 | MCP 자동 | 캠페인 구조/벌크 전략 |
| GFA | MCP 자동 (승인 후) / 수동 (승인 전) | 타겟팅/소재/예산 |
| 애드부스트 | 수동 + MCP 간접 분석 | 상품 선택/예산/ON-OFF 판단 |
