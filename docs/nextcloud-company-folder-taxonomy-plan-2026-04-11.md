# PRESSCO21 Nextcloud 회사 폴더 분류 기획안

작성일: 2026-04-11
상태: 운영 적용 완료

## 1. 목적

이 문서는 PRESSCO21 공유드라이브를 직원들이 헷갈리지 않고 쓰도록, 너무 많지 않으면서도 다시 찾기 쉬운 폴더 구조를 정리한 운영 기준안이다.

핵심 목표는 네 가지다.

1. 파일을 올릴 때 고민이 적을 것
2. 다시 찾을 때 경로가 직관적일 것
3. Nextcloud, Syncthing, MinIO 역할이 겹치지 않을 것
4. 관리자가 나중에 다시 분류하느라 일을 한 번 더 하지 않게 할 것

## 2. 시스템 역할 분리

### Nextcloud

- 회사 내부 공유드라이브
- 문서, 이미지, 브랜드 에셋, 디자인 결과물, 공용 사진 보관
- 직원이 직접 업로드하고 직접 다시 찾는 기본 입구

### Syncthing

- 대용량 영상 전용 작업선
- 장지호, 장다경만 사용
- `raw`, `project`, `export` 자동 동기화

### MinIO

- 외부 공개 배포용 최종 이미지 저장소
- 쇼핑몰, 이메일, 랜딩페이지, 광고에 실제로 붙는 URL용
- 내부 작업물 전체를 넣는 곳이 아님

## 3. 최종 추천 루트 구조

루트는 지금처럼 아래 6개만 유지한다.

```text
브랜드
디자인
사진
내부문서
영상자료
관리자문서함
```

이 구조를 유지하는 이유:

- 직원이 어디에 올릴지 바로 판단 가능
- 업무 목적별로 나뉘어 있어서 다시 찾기 쉬움
- 현재 Nextcloud 권한 구조와도 잘 맞음

## 4. 각 루트별 권장 하위 폴더

## 4-1. 브랜드

회사 정체성과 공용 에셋을 두는 곳이다.

```text
브랜드/
├── 로고
├── 브랜드가이드
├── 템플릿
└── 소개서-제안서
```

### 넣는 파일

- 로고 AI, SVG, PNG
- 브랜드 가이드북
- 공용 문서/슬라이드/배너 템플릿
- 회사소개서, 원페이저, 공용 제안서

### 넣지 않는 파일

- 특정 이벤트 전용 배너 완성본
- 상품 상세페이지 산출물
- 아직 수정 중인 개인 작업 중간본

### 운영 기준

- 회사 전체가 반복해서 쓰는 자산만 둔다.
- 개인 작업 중간본은 넣지 않는다.

## 4-2. 디자인

실제로 제작해서 쓰는 디자인 결과물을 두는 곳이다.

```text
디자인/
├── 진행중-공유본
├── 최종본
├── 배포대기
└── 캠페인별
```

### 넣는 파일

- 팀이 같이 확인해야 하는 시안
- 확정된 상세페이지, 배너, 카드뉴스, 썸네일
- 공유가 필요한 PSD, AI, PNG, JPG
- MinIO에 올리기 전 마지막 내부 검수본

### 넣지 않는 파일

- 회사 로고 원본
- 공용 템플릿 원본
- 대용량 영상 파일

### 운영 기준

- 디자인 작업물은 용도 기준으로 넣는다.
- 특정 프로젝트가 크면 `캠페인별/202604_봄기획전` 같은 식으로 하위 폴더를 만든다.
- 너무 잦은 임시 폴더 생성은 피한다.

## 4-3. 사진

촬영 이미지와 참고 이미지를 두는 곳이다.

```text
사진/
├── 상품
├── 수업-행사
├── 고객후기
└── 선별원본
```

### 넣는 파일

- 상품 촬영본
- 클래스 현장 사진
- 전시/행사 사진
- 고객 후기 사진
- 나중에 다시 쓸 가능성이 높은 선별 원본

### 넣지 않는 파일

- 최종 상세페이지 편집본
- 브랜드 로고/아이콘
- 계약서 스캔본

### 운영 기준

- 사진은 촬영 목적 기준으로 분류한다.
- 웹에 올릴 최종 편집본은 `디자인` 또는 MinIO 공개 배포용으로 따로 간다.

## 4-4. 내부문서

일반 직원도 봐야 하는 공용 문서를 두는 곳이다.

```text
내부문서/
├── 회사기본
├── 운영매뉴얼
├── 회의-결정
└── 양식-체크리스트
```

### 넣는 파일

- 회사 소개와 공용 참고 문서
- 업무 가이드와 SOP
- 회의록과 의사결정 메모
- 반복 사용하는 양식과 체크리스트

### 넣지 않는 파일

- 매출표
- 계약서 원본
- 인사 서류
- 대표/관리자만 봐야 하는 문서

### 운영 기준

- 직원이 봐도 되는 문서만 둔다.
- 민감하면 `관리자문서함`으로 보낸다.

## 4-5. 영상자료

Syncthing으로 모인 영상을 다시 찾아보는 열람용 폴더다.

```text
영상자료/
├── 장다경
│   ├── 촬영원본
│   ├── 편집프로젝트
│   └── 완성본
└── 장지호
    ├── 촬영원본
    ├── 편집프로젝트
    └── 완성본
```

### 넣는 파일

- Syncthing으로 자동 수집된 영상 원본
- 프로젝트 파일
- 최종본

### 넣지 않는 파일

- 일반 문서
- 이미지 자료
- 수동 업로드 파일

### 운영 기준

- 이 폴더는 기본적으로 열람/다운로드 기준으로 본다.
- 대용량 영상 업로드는 계속 Syncthing으로 한다.
- Nextcloud에서 영상자료를 직접 메인 업로드 라인으로 쓰지 않는다.

## 4-6. 관리자문서함

관리자만 보는 민감 문서 보관소다.

```text
관리자문서함/
├── 매출-재무
├── 계약-법무
└── 인사-기밀
```

### 넣는 파일

- 매출 자료
- 세무 자료
- 계약서
- NDA
- 급여, 인사, 신분증 사본, 내부 기밀 문서

### 넣지 않는 파일

- 전 직원이 같이 보는 문서
- 일반 디자인 파일
- 공용 브랜드 자료

### 운영 기준

- 관리자만 접근한다.
- 공용 자료와 섞지 않는다.

## 5. 직원이 실제로 판단하는 기준

직원 입장에서는 아래 기준만 기억하면 된다.

### 1. 공용 브랜드 자산인가

- 예: `브랜드`

### 2. 디자인 결과물인가

- 예: `디자인`

### 3. 촬영 사진/현장 사진인가

- 예: `사진`

### 4. 공용 문서인가

- 예: `내부문서`

### 5. 대용량 영상인가

- 예: Syncthing

### 6. 민감한 관리자 문서인가

- 예: `관리자문서함`

## 6. 폴더를 다시 찾기 쉽게 만드는 규칙

### 원칙 1. 사람 이름보다 용도로 분류한다

- `조승해 작업물` 같은 폴더는 만들지 않는다.
- 대신 `디자인/쇼핑몰/`, `디자인/SNS-광고/`처럼 용도로 찾는다.

### 원칙 2. 새 최상위 폴더는 만들지 않는다

- 애매해도 기존 6개 안에서 해결한다.

### 원칙 3. 프로젝트가 커질 때만 하위 폴더를 만든다

예:

```text
디자인/쇼핑몰/202604_봄기획전
사진/수업-행사/202604_파트너클래스촬영
내부문서/회의기록/202604
```

### 원칙 4. 파일명은 간단한 공용 규칙을 쓴다

- 내부 자료: `날짜_문서명_v01`
- 이미지 최종본: `날짜_프로젝트명_용도_v01`
- 공개 배포용: 영어 파일명

예:

- `20260411_회사소개서_v01.pdf`
- `20260411_봄기획전_메인배너_v02.png`
- `pressco21-spring-banner-v01.webp`

## 7. 지금 회사에 맞는 실제 운영안

### 장지호

- Nextcloud 전체 운영
- 관리자문서함 사용
- 대용량 영상은 Syncthing 사용

### 장다경

- 일반 자료는 Nextcloud
- 대용량 영상은 Syncthing

### 조승해

- Nextcloud만 사용
- 브랜드, 디자인, 사진 위주

### 이재혁

- Nextcloud만 사용
- 내부문서, 사진 위주

### 이진선

- 관리자 계정으로 Nextcloud 사용
- 필요 시 관리자문서함 접근

## 8. MinIO와의 연결 기준

Nextcloud와 MinIO는 섞지 않는다.

### Nextcloud에 두는 것

- 내부 공유용 원본
- 문서
- 사진 보관
- 디자인 작업 결과물

### MinIO로 보내는 것

- 쇼핑몰 배포용 최종 이미지
- 이메일에 삽입할 최종 이미지
- 랜딩페이지, 광고, SNS에 공개 URL로 써야 하는 이미지

즉:

- `보관과 협업`은 Nextcloud
- `외부 공개 배포`는 MinIO

## 9. 최종 추천 결론

지금 회사에는 현재 루트 6개를 유지하면서, 각 루트 안만 아래처럼 간단히 정리하는 것이 가장 적절하다.

```text
브랜드/
  로고
  브랜드가이드
  템플릿
  소개서-제안서

디자인/
  진행중-공유본
  최종본
  배포대기
  캠페인별

사진/
  상품
  수업-행사
  고객후기
  선별원본

내부문서/
  회사기본
  운영매뉴얼
  회의-결정
  양식-체크리스트

영상자료/
  장다경/촬영원본
  장다경/편집프로젝트
  장다경/완성본
  장지호/촬영원본
  장지호/편집프로젝트
  장지호/완성본

관리자문서함/
  매출-재무
  계약-법무
  인사-기밀
```

이 구조의 장점은 세 가지다.

1. 직원이 어디에 넣을지 빠르게 판단할 수 있다.
2. 다시 찾을 때 사람 이름이 아니라 용도 중심으로 찾을 수 있다.
3. 현재 Nextcloud, Syncthing, MinIO 역할 분리와 충돌하지 않는다.
