# PRESSCO21 자료 정리 및 파일명 가이드

작성일: 2026-04-15

## 1. 왜 하는가

회사 자료가 각자 PC, 카카오톡, 텔레그램, 다운로드 폴더, 외장하드에 흩어져 있으면 나중에 찾기 어렵다.

앞으로 회사 자료는 `backup.pressco21.com`의 공유드라이브에 모은다.

처음 목표는 완벽한 정리가 아니라, 흩어진 자료를 회사 저장소로 모으는 것이다.

## 2. 기본 원칙

자료를 올릴 때는 아래 기준으로 판단한다.

```text
어디에 넣을지 알겠다
-> 맞는 폴더에 직접 넣는다.

어디에 넣을지 애매하다
-> 00_INBOX/직원업로드에 넣는다.

민감자료다
-> 공용 폴더에 넣지 않고 관리자에게 전달한다.
```

## 3. 어디에 넣으면 되는가

| 폴더 | 넣는 자료 |
| --- | --- |
| `00_INBOX/직원업로드` | 어디에 넣을지 애매한 자료, 임시 수집 자료 |
| `브랜드` | 로고, 브랜드가이드, 회사소개서, 제안서, 템플릿 |
| `디자인` | 상세페이지, 배너, 썸네일, 디자인 시안, 최종 디자인 파일 |
| `사진` | 상품 사진, 수업/행사 사진, 고객후기 사진, 선별 원본 |
| `내부문서` | 매뉴얼, 회의록, 업무 양식, 체크리스트, 직원 공유용 문서 |
| `영상자료` | 촬영 원본, 편집 프로젝트, 완성 영상 |
| `업무가이드` | 직원이 읽는 기준 문서, 표준답변, 체크리스트 |

## 4. 파일명 최소 규칙

가능하면 파일명은 아래 형식으로 바꿔서 올린다.

```text
날짜_무슨자료인지_상태_v번호
```

예시:

```text
20260415_상품사진_프리저브드장미_원본_v01.jpg
20260415_상세페이지_레지너스오일_시안_v01.psd
20260415_회의록_오픈마켓운영_공유본_v01.docx
20260415_고객후기사진_하바리움클래스_원본_v01.jpg
```

날짜를 정확히 모르면 `정리한 날짜`를 쓴다.

## 5. 가능하면 더 좋은 파일명

조금 더 정확하게 적을 수 있으면 아래 형식을 쓴다.

```text
YYYYMMDD_분류_대상_내용_상태_v01_작성자.ext
```

예시:

```text
20260415_디자인_레지너스오일_상세페이지_시안_v01_승해.psd
20260415_사진_부케클래스_수업현장_원본_v01_다경.jpg
20260415_내부문서_오픈마켓_운영체크리스트_공유본_v01_지호.xlsx
20260415_브랜드_PRESSCO21_회사소개서_최종_v03_지호.pdf
```

이렇게 적어두면 나중에 AI가 자료를 자동으로 분류할 때 훨씬 정확해진다.

## 6. 상태 단어는 이렇게 통일한다

| 상태 | 뜻 |
| --- | --- |
| `원본` | 촬영 원본, 받은 원본, 수정 전 원본 |
| `시안` | 아직 확정 전 디자인/문서 |
| `검수중` | 확인이 필요한 자료 |
| `공유본` | 직원이 같이 봐도 되는 자료 |
| `최종` | 내부 기준 최종본 |
| `배포용` | 외부 업로드 예정 자료 |
| `교체됨` | 더 이상 최신이 아닌 자료 |
| `폐기대기` | 삭제 전 확인이 필요한 자료 |

피해야 하는 이름:

```text
최종최종
진짜최종
수정본
새버전
새파일
새 폴더
IMG_0001
KakaoTalk_파일명 그대로
```

## 7. 받은 파일은 어떻게 바꾸는가

외부에서 받은 파일은 이름이 이상한 경우가 많다.

예:

```text
KakaoTalk_20260415_123455.jpg
IMG_8831.JPG
수정본최종.pdf
제안서(3).pptx
```

이럴 때는 파일을 열어보고 대략적인 의미를 붙인다.

```text
20260415_사진_고객후기_카카오수신_원본_v01.jpg
20260415_사진_상품컷_촬영원본_IMG_8831.jpg
20260415_내부문서_거래처제안서_검수중_v01.pdf
20260415_브랜드_제안서_거래처명_검수중_v01.pptx
```

계약서처럼 원본성이 중요한 파일은 원래 파일명을 뒤에 남겨도 된다.

```text
20260415_민감_거래처명_계약서_원본_KakaoTalk_20260415_123455.pdf
```

## 8. 민감자료는 공용 폴더에 올리지 않는다

아래 자료는 직원 공용 폴더에 넣지 않는다.

```text
계약서
세무자료
매출자료
정산자료
급여자료
인사자료
고객 개인정보
계정 정보
비밀번호
API키
토큰
사업자등록증
통장 사본
카드 정보
법무/분쟁 자료
```

이런 자료는 관리자에게 전달한다.

## 9. 각자 먼저 정리할 위치

각자 PC에서 아래 위치를 먼저 확인한다.

1. 바탕화면
2. 다운로드 폴더
3. 문서 폴더
4. 카카오톡/텔레그램에서 받은 파일
5. 작업 프로그램 저장 폴더
6. 외장하드/USB

정리 기준:

```text
알겠는 자료
-> 맞는 Nextcloud 폴더로 이동

애매한 자료
-> 00_INBOX/직원업로드

민감자료
-> 관리자에게 전달
```

## 10. 직원에게 필요한 핵심 요약

```text
파일명은 가능하면 "날짜_무슨자료인지_상태_v번호"로 바꿔서 올려주세요.

어디에 넣을지 알겠으면 브랜드/디자인/사진/내부문서/영상자료에 넣고,
애매하면 00_INBOX/직원업로드에 넣으면 됩니다.

계약, 세무, 매출, 인사, 고객 개인정보, 계정 정보는 공용 폴더에 넣지 말고 관리자에게 전달해주세요.
```
