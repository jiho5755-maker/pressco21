# PRESSCO21 Nextcloud 파일명 및 자료 분류 규칙

> 작성일: 2026-04-15  
> 목적: 직원별로 흩어진 회사 자료를 Nextcloud에 모으고, 이후 AI/n8n이 일괄 분류할 수 있게 파일명과 분류 기준을 통일한다.

## 1. 기본 원칙

처음 목표는 완벽한 정리가 아니라 `회사 자료를 한곳으로 모으는 것`이다.

직원은 아래처럼 판단한다.

```text
어디에 넣을지 알겠다
-> 해당 폴더에 직접 넣는다.

어디에 넣을지 애매하다
-> 00_INBOX/직원업로드에 넣는다.

민감자료다
-> 공용 폴더에 넣지 않고 관리자에게 전달한다.
```

AI 자동분류는 당장 강하게 적용하지 않는다. 먼저 직원들이 자료를 모으고, 파일명/폴더 패턴이 쌓이면 그 자료를 기준으로 n8n/AI 분류를 붙인다.

## 2. 직원용 최상위 분류

| 폴더 | 넣는 자료 |
| --- | --- |
| `00_INBOX/직원업로드` | 어디에 넣을지 애매한 자료, 임시 수집 자료 |
| `브랜드` | 로고, 브랜드가이드, 회사소개서, 제안서, 템플릿 |
| `디자인` | 상세페이지, 배너, 썸네일, 디자인 시안, 최종 디자인 파일 |
| `사진` | 상품 사진, 수업/행사 사진, 고객후기 사진, 선별 원본 |
| `내부문서` | 매뉴얼, 회의록, 업무 양식, 체크리스트, 직원 공유용 문서 |
| `영상자료` | 촬영 원본, 편집 프로젝트, 완성 영상 |
| `업무가이드` | 직원이 읽는 기준 문서, 표준답변, 체크리스트 |
| `관리자문서함` | 관리자 전용 민감자료 |

## 3. 파일명 최소 규칙

직원이 반드시 기억할 최소 규칙은 하나다.

```text
날짜_무슨자료인지_상태_v번호
```

예시:

```text
20260415_상품사진_프리저브드장미_원본_v01.jpg
20260415_상세페이지_레지너스오일_시안_v01.psd
20260415_회의록_오픈마켓운영_공유본_v01.docx
20260415_고객후기사진_하바리움클래스_원본_v01.jpg
```

날짜를 모르면 파일을 정리한 날짜를 쓴다.

```text
20260415_자료정리_브랜드로고_원본_v01.ai
```

## 4. 권장 파일명 형식

가능하면 아래 형식을 쓴다.

```text
YYYYMMDD_분류_대상_내용_상태_v01_작성자.ext
```

필수는 아니지만, AI 일괄분류에는 이 형식이 가장 좋다.

| 요소 | 의미 | 예시 |
| --- | --- | --- |
| `YYYYMMDD` | 작성일, 촬영일, 수신일, 정리일 | `20260415` |
| `분류` | 자료 종류 | `브랜드`, `디자인`, `사진`, `내부문서`, `영상`, `민감`, `배포` |
| `대상` | 상품명, 프로젝트명, 고객명, 행사명 | `레지너스오일`, `부케클래스`, `스마트스토어` |
| `내용` | 실제 자료 내용 | `상세페이지`, `회의록`, `고객후기`, `썸네일` |
| `상태` | 원본/시안/검수중/최종/공유본/폐기대기 | `원본`, `시안`, `최종` |
| `v01` | 버전 | `v01`, `v02`, `v03` |
| `작성자` | 필요 시 작성자/소유자 | `다경`, `승해`, `재혁`, `지호` |

예시:

```text
20260415_디자인_레지너스오일_상세페이지_시안_v01_승해.psd
20260415_사진_부케클래스_수업현장_원본_v01_다경.jpg
20260415_내부문서_오픈마켓_운영체크리스트_공유본_v01_지호.xlsx
20260415_브랜드_PRESSCO21_회사소개서_최종_v03_지호.pdf
```

## 5. 상태 단어 표준

파일명에는 아래 상태 단어를 우선 사용한다.

| 상태 | 뜻 |
| --- | --- |
| `원본` | 촬영 원본, 받은 원본, 수정 전 원본 |
| `시안` | 아직 확정 전 디자인/문서 |
| `검수중` | 확인이 필요한 자료 |
| `공유본` | 직원이 같이 봐도 되는 자료 |
| `최종` | 내부 기준 최종본 |
| `배포용` | 외부 업로드 예정 자료 |
| `교체됨` | 더 이상 최신이 아닌 자료 |
| `폐기대기` | 삭제 전 확인이 필요한 자료 |

피해야 할 상태명:

```text
최종최종
진짜최종
수정본
새버전
새파일
정리
```

## 6. 폴더별 파일명 예시

### 6.1 00_INBOX

```text
20260415_INBOX_자료정리_출처불명_검수중_v01_다경.zip
20260415_INBOX_고객자료_분류필요_검수중_v01_재혁.pdf
20260415_INBOX_디자인참고_분류필요_검수중_v01_승해.png
```

`INBOX`에 넣을 때는 완벽한 이름이 아니어도 된다. 다만 `무슨 자료인지` 정도는 보이게 적는다.

### 6.2 브랜드

```text
20260415_브랜드_PRESSCO21_회사소개서_최종_v03.pdf
브랜드_PRESSCO21_로고가로형_원본_v01.ai
브랜드_PRESSCO21_브랜드가이드_최종_v02.pdf
브랜드_PRESSCO21_제안서템플릿_공유본_v01.pptx
```

### 6.3 디자인

```text
20260415_디자인_레지너스오일_상세페이지_시안_v01_승해.psd
20260415_디자인_봄캠페인_배너_검수중_v02_다경.png
20260415_디자인_스마트스토어_썸네일_배포용_v01_승해.jpg
```

웹에 올라갈 파일은 영어 파일명을 쓴다.

```text
pressco21-reginus-oil-detail-v01.jpg
pressco21-spring-campaign-banner-20260415-v02.jpg
```

### 6.4 사진

```text
20260415_사진_프리저브드장미_상품컷_원본_v01_다경.jpg
20260415_사진_부케클래스_수업현장_원본_v01_지호.jpg
20260415_사진_하바리움_고객후기_공유본_v01_재혁.jpg
```

카메라 원본 파일명이 `IMG_1234.jpg`라면 그대로 두지 말고, 가능하면 앞에 설명을 붙인다.

```text
20260415_사진_부케클래스_수업현장_원본_IMG_1234.jpg
```

### 6.5 내부문서

```text
20260415_내부문서_오픈마켓_운영매뉴얼_공유본_v01.docx
20260415_내부문서_주간회의_회의록_공유본_v01.docx
20260415_내부문서_배송업무_체크리스트_공유본_v02.xlsx
```

직원 공유용 내부문서에는 계약, 세무, 인사, 고객 개인정보를 넣지 않는다.

### 6.6 영상자료

```text
20260415_영상_부케클래스_촬영원본_원본_v01_다경.mp4
20260415_영상_유튜브_편집프로젝트_작업중_v01_지호.prproj
20260415_영상_릴스_완성본_배포용_v01_다경.mp4
```

영상 대용량 원본은 가능하면 Syncthing 기본 폴더를 사용한다.

### 6.7 업무가이드

```text
20260415_업무가이드_고객응대_표준답변_공유본_v01.md
20260415_업무가이드_디자인업로드_체크리스트_공유본_v01.md
20260415_업무가이드_오픈마켓_상품등록기준_공유본_v01.md
```

업무가이드는 직원이 읽는 결과물이다. 원본 AI Wiki는 관리자문서함 또는 내부 시스템 경로에서 관리한다.

### 6.8 관리자문서함

```text
20260415_민감_거래처명_계약서_원본_v01.pdf
20260415_민감_2026년1분기_세무증빙_원본_v01.zip
20260415_민감_고객명_개인정보_원본_v01.xlsx
20260415_민감_서비스명_계정권한_검수중_v01.xlsx
```

민감자료는 직원 공용 폴더에 넣지 않는다.

## 7. AI 일괄분류 기준

AI는 아래 순서로 자료를 분류한다.

1. 현재 폴더 위치
2. 파일명 안의 분류 단어
3. 파일명 안의 상태 단어
4. 확장자
5. 파일 내용 일부
6. 중복 파일 여부

### 7.1 분류 단어

| 단어 | 우선 추천 폴더 |
| --- | --- |
| `브랜드`, `로고`, `브랜드가이드`, `회사소개서`, `제안서`, `템플릿` | `브랜드` |
| `디자인`, `상세페이지`, `배너`, `썸네일`, `시안`, `PSD`, `AI` | `디자인` |
| `사진`, `상품컷`, `수업현장`, `행사`, `고객후기`, `촬영원본` | `사진` |
| `회의록`, `매뉴얼`, `체크리스트`, `양식`, `운영`, `SOP` | `내부문서` |
| `영상`, `릴스`, `유튜브`, `촬영원본`, `편집프로젝트`, `완성본` | `영상자료` |
| `계약`, `세무`, `매출`, `정산`, `급여`, `인사`, `개인정보`, `계정`, `비밀번호` | `관리자문서함` |
| `표준답변`, `업무가이드`, `FAQ`, `체크리스트`, `기준` | `업무가이드` |

### 7.2 확장자 힌트

| 확장자 | 힌트 |
| --- | --- |
| `.psd`, `.ai`, `.fig`, `.sketch` | 디자인 원본 가능성 높음 |
| `.jpg`, `.jpeg`, `.png`, `.heic`, `.raw` | 사진 또는 디자인 출력물 |
| `.mp4`, `.mov`, `.prproj`, `.aep` | 영상자료 |
| `.docx`, `.hwp`, `.hwpx`, `.pdf`, `.md` | 내부문서/브랜드/계약 가능성 |
| `.xlsx`, `.csv` | 매출/정산/고객자료/운영자료 가능성 |
| `.zip` | 묶음 자료라 검토 필요 |

## 8. 받은 파일 처리 규칙

외부에서 받은 파일은 원래 파일명이 이상한 경우가 많다.

예:

```text
KakaoTalk_20260415_123455.jpg
IMG_8831.JPG
수정본최종.pdf
제안서(3).pptx
```

이 경우 파일을 열어보고 대략적인 의미만 붙인다.

```text
20260415_사진_고객후기_카카오수신_원본_v01.jpg
20260415_사진_상품컷_촬영원본_IMG_8831.jpg
20260415_내부문서_거래처제안서_검수중_v01.pdf
20260415_브랜드_제안서_거래처명_검수중_v01.pptx
```

법적 증거성이나 수신 원본성이 중요한 파일은 원본 파일명을 완전히 지우지 말고 뒤에 남긴다.

```text
20260415_민감_거래처명_계약서_원본_KakaoTalk_20260415_123455.pdf
```

## 9. 직원별 1차 자료 수집 방법

각 직원은 자기 PC에서 회사 자료를 먼저 모은다.

권장 순서:

1. 바탕화면
2. 다운로드 폴더
3. 문서 폴더
4. 카카오톡/텔레그램에서 받은 파일
5. 작업 프로그램 저장 폴더
6. 외장하드/USB

정리 방식:

```text
알겠는 자료
-> 맞는 Nextcloud 폴더로 이동

애매한 자료
-> 00_INBOX/직원업로드

민감자료
-> 관리자에게 전달
```

## 10. 금지 규칙

아래는 피한다.

- `최종최종진짜최종`
- `새 폴더`
- `수정본(2)`
- `스크린샷 2026-04-15 오후 3.22.10`
- `IMG_0001`
- `KakaoTalk_...` 그대로 두기
- 계약/세무/개인정보를 공용 폴더에 넣기
- 같은 파일을 여러 폴더에 중복 업로드하기
- 확정 전 파일을 `배포대기`나 `업무가이드`에 넣기

## 11. 직원에게 안내할 짧은 버전

```text
파일명은 가능하면 "날짜_무슨자료인지_상태_v번호"로 바꿔서 올려주세요.

예:
20260415_디자인_레지너스오일_상세페이지_시안_v01.psd
20260415_사진_부케클래스_수업현장_원본_v01.jpg
20260415_내부문서_주간회의_회의록_공유본_v01.docx

어디에 넣을지 알겠으면 브랜드/디자인/사진/내부문서/영상자료에 넣고,
애매하면 00_INBOX/직원업로드에 넣으면 됩니다.

계약, 세무, 매출, 인사, 고객 개인정보, 계정 정보는 공용 폴더에 넣지 말고 관리자에게 전달해주세요.
```
