# PRESSCO21 mini PC Nextcloud 도입 계획

작성일: 2026-04-10
상태: 파일럿 배치 완료

## 목표

- File Browser보다 `공유드라이브`에 가까운 직원 UX 확보
- mini PC를 계속 원청 보관소로 유지
- 영상팀의 Syncthing 라인은 유지
- 웹디자이너와 일반 직원은 브라우저 기반 접근을 우선 적용

## 운영 원칙

- mini PC는 계속 `창고` 역할을 맡는다.
- Nextcloud는 `내부용 공유드라이브 포털`이다.
- 공개 인터넷 직노출은 하지 않는다.
- 접근은 LAN 또는 Tailscale로만 연다.
- File Browser는 바로 폐기하지 않고 관리자/비상용으로 남긴다.

## 1단계 파일럿

- `Nextcloud`를 별도 포트 `18080` 에 배치
- 기존 `File Browser` 는 `8090` 유지
- Nextcloud 앱/DB/캐시는 시스템 디스크(ext4) 사용
- 기존 SSD 공유 폴더는 읽기 전용 외부 경로로 컨테이너에 마운트
- 우선 적용 대상:
  - 웹디자이너
  - 대표
  - 일반 직원 공용자료 접근

## 왜 앱 데이터는 루트 디스크에 두는가

현재 외장 SSD는 `exfat`로 마운트되어 있다.

- 장점: 맥/윈도우와 호환성이 좋다
- 단점: 리눅스 서비스 관점에서는 권한, 소유권, 심볼릭 링크, 앱 데이터 운영에 불리하다

그래서:

- Nextcloud 자체 데이터베이스/앱 데이터는 루트 디스크(ext4)
- 기존 회사 자료 저장소는 외부 공유 경로

로 분리한다.

## 파일럿 성공 기준

1. mini PC에서 Nextcloud 컨테이너가 정상 실행
2. LAN/Tailscale 주소로 로그인 가능
3. 관리자 계정으로 초기 접속 가능
4. 웹디자이너용 공유 폴더 구조를 정의할 수 있음
5. File Browser와 충돌 없이 병행 운영 가능

## 2단계

- `files_external` 앱으로 기존 shared/company-hub 노출
- 개인 계정 + 그룹 권한 구조 전환
- 업로드/다운로드 가이드 문서화
- 필요 시 Tailscale Serve 기반 HTTPS 적용

## 역할 분리

- Syncthing:
  - 대표
  - 영상팀장
- Nextcloud:
  - 웹디자이너
  - 일반 직원
- File Browser:
  - 관리자 점검
  - 비상용 포털
  - 기존 운영 자동화 확인

## 예상 위험

- Nextcloud는 File Browser보다 무겁다.
- 루트 디스크 428GB를 쓰므로 대용량 영상 주저장소로는 부적합하다.
- 따라서 대용량 영상 원본은 계속 Syncthing + ACTIVE/ARCHIVE 흐름을 유지한다.
