## 1. 섹션 제목 (sectionTitles)

### 설계 원칙

섹션 제목은 **설명적이면서도 부드러운 톤**을 유지합니다. 딱딱한 정보 나열("커리큘럼", "일정") 대신, 자연스러운 안내 문장 또는 간결한 명사형을 사용합니다.

### A/B 방향

| 섹션 | A안 (채택) | B안 (대안) | 선택 이유 |
|------|-----------|-----------|----------|
| 강의 소개 | **강의 소개** | 이 수업은요 | A: 직관적, 스캔 가능. B: 친근하지만 다소 캐주얼 |
| 커리큘럼 | **이렇게 진행돼요** | 커리큘럼 | A: 따뜻한 톤, 초보자도 편안. B: 정확하지만 전문적이라 거리감 |
| 일정 및 예약 | **일정 및 예약** | 날짜를 골라보세요 | A: 명확한 목적 전달. B: 친근하지만 액션 섹션에는 명료함이 우선 |
| 강사/공방 프로필 | **강사 & 공방 소개** | 이 분과 함께해요 | A: 정보 구조 명확. B: 개인적 친밀감은 있으나 복수 강사 시 부자연스러움 |
| 재료 | **수강에 필요한 재료** | 준비물 | A: 구체적이고 정중함. B: 간결하지만 학교 느낌 |
| YouTube 영상 | **미리 만나는 수업 영상** | 관련 영상 | A: 기대감 유발, 브랜드 톤 적합. B: 너무 건조 |
| 수강 후기 | **수강 후기** | 함께 만든 이야기 | A: 보편적이고 검색 친화적. B: 감성적이지만 정보 탐색 시 혼란 |

## 2. CTA 버튼 텍스트 (cta)

### 설계 원칙

- **주 CTA (예약하기)**: 명확한 행동 유도. "수업"이라는 키워드를 포함해 맥락을 알려줌
- **모바일 하단 고정**: 화면 공간 제약으로 더 짧게. 금액 포함 버전도 준비
- **보조 CTA**: 부드러운 안내 톤

### 상세 설명

| CTA | 텍스트 | 톤 설명 |
|-----|-------|---------|
| 메인 예약 | **수업 예약하기** | "수업"을 포함해 무엇을 예약하는지 명확. "예약"은 클래스/체험 맥락에 자연스러움 |
| 모바일 고정 (짧은) | **예약하기** | 공간 절약. 페이지 맥락상 "수업"은 생략 가능 |
| 모바일 고정 (금액 포함) | **{price}원 예약하기** | 가격 인지 상태에서 바로 결제 유도. 전환율 향상 기대 |
| 파트너맵 | **파트너맵에서 공방 보기** | "공방"을 포함해 무엇을 보러 가는지 명시 |
| 재료 구매 | **재료 간편구매** | "간편"으로 허들 낮춤. 기존 '간편 구매' 프로젝트 네이밍과 통일 |
| 후기 더 보기 | **후기 더 보기** | 간결하고 직관적 |
| 재시도 | **다시 시도하기** | 에러 상황에서 부드러운 유도 |
| 목록 복귀 | **목록으로 돌아가기** | 에러/404 상황의 보조 탈출구 |

## 3. 상태 메시지 (seatStatus, materialsBanner, gracefulDegradation)

### 잔여석 상태

| 상태 | 메시지 | 디자인 연동 |
|------|-------|-----------|
| 여유 (4석 이상) | **여유 있어요 ({n}석 남음)** | 초록 텍스트, 차분한 톤 |
| 임박 (3석 이하) | **마감이 다가오고 있어요! ({n}석 남음)** | 오렌지/레드 텍스트, 살짝 긴급 |
| 마감 (0석) | **이 일정은 마감되었어요** | 회색 텍스트, 비활성 스타일 |

> **주의**: 목록 페이지(Task 211)와 동일한 톤을 유지합니다. "품절", "매진" 같은 상업적 표현 대신 "마감"을 사용합니다.
> 임계값(`almostFullThreshold`)은 3으로 설정. 정원이 큰 클래스(15명+)의 경우 UI단에서 비율 기반으로 조정 가능합니다.

### 재료 안내 배너

| 상태 | 메시지 | 톤 설명 |
|------|-------|---------|
| 포함 | **이 수업은 모든 재료가 포함되어 있어요. 편하게 오세요!** | 안심감 + 편의 강조 |
| 별도 | **수강에 필요한 재료를 미리 준비해주세요. 아래에서 간편하게 구매할 수 있어요.** | 안내 + 구매 유도 |

### Graceful Degradation 메시지

모든 섹션이 독립적으로 실패할 수 있으므로, 각 실패 상황에 맞는 부드러운 대체 메시지를 준비합니다.

| 상황 | 메시지 | 톤 |
|------|-------|------|
| 일반 섹션 준비 중 | **이 섹션은 현재 준비 중이에요. 곧 만나볼 수 있어요!** | 기대감 유지 |
| 재료 로딩 실패 | **재료 정보를 불러오는 중이에요. 잠시 후 다시 확인해주세요.** | 일시적 문제 암시 |
| 후기 없음 | **아직 첫 번째 후기가 없어요. 수업 후 소중한 경험을 나눠주세요!** | 참여 유도 |
| 파트너맵 미연동 | **파트너맵 연동을 준비하고 있어요.** | 간결한 상태 안내 |
| YouTube 미연동 | **영상을 불러올 수 없어요.** | 최소한의 안내 (섹션 자체를 숨길 예정) |

## 4. 정보 표시 레이블 (infoLabels)

### 수업 시간 표현

GAS API의 `duration_min` 값을 사람이 읽기 쉽게 변환합니다.

```
60분 미만  -> "{n}분 수업"        (예: "45분 수업")
60분 정각  -> "{h}시간 수업"      (예: "2시간 수업")
60분 초과  -> "{h}시간 {m}분 수업" (예: "1시간 30분 수업")
```

### 정원 표현

```
10명 이하  -> "최대 {n}명 소규모"   (예: "최대 8명 소규모")
11명 이상  -> "최대 {n}명"         (예: "최대 15명")
```

> "소규모"를 붙이는 기준(10명)은 `smallGroupThreshold`로 조절 가능합니다. 소규모 클래스의 매력을 부각하는 표현입니다.

### 레벨 배지

| API 값 | 표시 텍스트 | 보조 설명 |
|--------|-----------|----------|
| 입문 / BEGINNER | **입문** | 처음이어도 괜찮아요 |
| 중급 / INTERMEDIATE | **중급** | 기초를 다진 분께 추천해요 |
| 심화 / ADVANCED | **심화** | 한 단계 더 깊이 들어가요 |

> 보조 설명(`levelDescription`)은 배지 hover 시 또는 상세 정보 영역에 표시합니다.

### 파트너 등급 배지

| API 값 | 한국어 표시 | 파트너 프로그램 대응 |
|--------|-----------|-------------------|
| SILVER | **실버 파트너** | Bloom (입문) 등급 |
| GOLD | **골드 파트너** | Garden (성장) 등급 |
| PLATINUM | **플래티넘 파트너** | Atelier (마스터) 등급 |

> 파트너 프로그램의 Bloom/Garden/Atelier 네이밍은 파트너 내부용이며, 수강생에게는 실버/골드/플래티넘으로 표시합니다. 이는 수강생이 직관적으로 등급의 높낮이를 이해할 수 있도록 하기 위함입니다.

| 상황 | 주 메시지 | 보조 메시지 |
|------|---------|-----------|

## JS 코드 연동 방법

`js.js`에서 이 카피를 사용하는 권장 패턴:

```javascript
// js.js 상단에 카피 객체 선언
var COPY = {
    sectionTitles: { ... },
    cta: { ... },
    // ... class-detail-copy.json의 내용을 인라인으로 포함
};

// 사용 예시
function renderSeatStatus(remaining, maxStudents) {
    if (remaining <= 0) {
        return '<span class="seat-status seat-status--full">' + COPY.seatStatus.full + '</span>';
    }
    if (remaining <= COPY.seatStatus.almostFullThreshold) {
        return '<span class="seat-status seat-status--almost">'
            + COPY.seatStatus.almostFull.replace('{n}', remaining) + '</span>';
    }
    return '<span class="seat-status seat-status--available">'
        + COPY.seatStatus.available.replace('{n}', remaining) + '</span>';
}
```

> JSON 파일을 별도로 fetch하지 않고, js.js 상단에 var COPY = {...} 형태로 직접 포함합니다.
