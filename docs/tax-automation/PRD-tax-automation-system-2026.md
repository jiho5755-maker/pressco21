## 1. 제품명

PRESSCO21 세무 자료 수집 자동화 OS

## 2. 한 줄 정의

홈택스, 은행, 카드, 오픈마켓, PG, 4대보험, 정부24, UNI-PASS, 내부 CRM/매출 자료를 수집·분류·검증해 세무사에게 바로 전달 가능한 연도별·월별 증빙 패키지로 만드는 시스템이다.

## 3. 배경과 문제 정의

### 3.1 현재 문제

- 종합소득세 자료가 홈택스, 카드사, 은행, 오픈마켓, PG, 4대보험, 정부24, UNI-PASS, 사내 파일에 흩어져 있다.
- 비전공자는 어떤 자료가 필요한지, 어디에서 다운로드해야 하는지, 기간을 어떻게 잡아야 하는지 알기 어렵다.
- 세무사에게 자료를 넘긴 뒤 추가 요청이 반복된다.
- 파일명과 폴더 기준이 없어 전년도 자료 재사용, 월 누락, 중복 제출 위험이 있다.
- 플랫폼 매출은 발생일, 정산일, 입금일이 달라 금액 불일치가 자주 생긴다.
- 수입·관세 자료는 UNI-PASS와 내부 매입/재고 자료가 연결되지 않아 놓치기 쉽다.
- 자동화 작업이 API 단위로 흩어지면 “왜 자동화하는지”라는 큰 목표를 잃을 수 있다.

### 3.2 근본 원인

세무 자료 수집의 본질은 단일 API 연동이 아니라 `원천별 자료 위치`, `기간`, `식별자`, `원본 파일`, `금액 합계`, `세무사 확인 상태`를 한 곳에서 관리하는 문제다.

## 4. 제품 목표

### 4.1 단기 목표

- 2025년 귀속 종합소득세 자료를 비전공자도 가이드만 보고 수집할 수 있게 한다.
- 자료별 상태를 `미수집`, `수집 완료`, `확인 필요`, `누락 의심`, `해당 없음`으로 관리한다.
- 세무사 전달 폴더와 파일명 규칙을 고정한다.
- UNI-PASS 수입·관세 자동화는 전체 세무 자료 수집 OS의 한 모듈로 재배치한다.

### 4.2 중기 목표

- 수동 업로드만으로도 자료 누락 현황과 세무사 전달 패키지를 생성한다.
- 무료 API와 무료 조회 서비스를 연결해 반복 다운로드 업무를 줄인다.
- 월별 매출, 매입, 계좌, 카드, 통관, 급여/보험 자료의 누락·중복·금액 불일치 후보를 찾는다.

### 4.3 장기 목표

- 종합소득세, 부가세, 원천세, 4대보험, 관세 자료를 같은 원장과 파일 저장 구조에서 관리한다.
- 세무사 질문과 회사 답변 이력을 축적해 매년 반복 가능한 세무 운영 프로세스를 만든다.
- 자료 수집 자동화와 세무 판단을 분리해 법적·보안 리스크를 낮춘다.

## 5. 범위

### 5.1 포함 범위

- 종합소득세 자료 수집 체크리스트
- 비전공자용 다운로드 가이드
- 자료별 상태 관리
- 파일 업로드 및 파일명 표준화
- 원본 파일 보관
- 세무 증빙 메타데이터 추출
- 월별·연도별 누락 탐지
- 중복 파일 감지
- 금액 합계 추출과 대조 후보 생성
- UNI-PASS 통관·관세 자료 수집
- 세무사 전달 패키지 생성
- 무료 API·무료 조회 서비스 조사와 단계적 연결

### 5.2 제외 범위

- 종합소득세 신고서 자동 제출
- 세액 자동 확정
- 납부 실행 자동화
- 경비 인정 여부 최종 판단
- 부양가족 공제 가능 여부 최종 판단
- 성실신고확인 대상 여부 최종 판단
- 간편장부/복식부기/추계신고 선택 판단
- 공동인증서, 금융인증서, 포털 비밀번호 저장
- CAPTCHA, 2FA, 본인인증 우회

## 6. 사용자

| 사용자 | 목적 | 주요 기능 |
|---|---|---|
| 대표/관리자 | 자료 누락 없이 세무사에게 전달 | 전체 진행률, 누락 위험, 승인 필요 항목 확인 |
| 비전공자 직원 | 가이드에 따라 자료 다운로드·업로드 | 단계별 체크리스트, 파일명 규칙, 업로드 검증 |
| 세무사 | 정리된 원본과 요약자료 검토 | 전달 패키지, 누락 리포트, 설명 메모 |
| 자동화 시스템 | 반복 수집과 정규화 | API 호출, 파싱, 상태 변경, 알림, 대조 후보 생성 |

## 7. 핵심 사용자 시나리오

### 7.1 종합소득세 자료 수집

1. 직원이 시스템에서 `2025년 귀속 종합소득세` 체크리스트를 연다.
2. 각 항목의 다운로드 위치와 기간 기준을 확인한다.
3. 홈택스, 은행, 카드사, 오픈마켓, PG, 4대보험, 정부24에서 파일을 다운로드한다.
4. 파일명을 규칙대로 저장하거나 업로드한다.
5. 시스템이 파일명, 확장자, 기간, 중복 여부를 확인한다.
6. 누락된 월이나 불명확한 파일은 `확인 필요`로 표시된다.
7. 대표가 최종 확인 후 세무사 전달 패키지를 만든다.

### 7.2 UNI-PASS 통관·관세 자료 자동화

1. 시스템이 화물관리번호 또는 BL 번호 목록을 받는다.
2. UNI-PASS API001로 화물통관 진행정보를 조회한다.
3. 응답에서 수입신고번호 후보를 추출한다.
4. UNI-PASS API049로 수입제세 납부여부와 납부일자를 확인한다.
5. 통관 상태와 납부 상태를 `tax_evidence_items`와 관세 전용 테이블에 기록한다.
6. 월마감 패키지에 통관·관세 증빙을 포함한다.

### 7.3 세무사 전달

1. 대표가 `세무사 전달 패키지 생성`을 누른다.
2. 시스템은 수집 완료 파일, 확인 필요 목록, 누락 의심 목록을 분리한다.
3. Excel 요약표와 파일 manifest를 만든다.
4. PDF, XLSX, CSV 원본을 폴더 구조대로 묶는다.
5. 세무사 전달 후 질문·요청사항은 검토 큐에 기록한다.

|---|---|---|

| `COLLECTED` | 파일 수집 완료 | 검증 대기 |

## 10. 기능 요구사항

### 10.1 체크리스트

- 연도별 종합소득세 체크리스트를 생성할 수 있어야 한다.
- 항목마다 출처, 기간, 다운로드 방법, 필수 여부, 담당자, 상태를 가져야 한다.
- 해당 없는 항목은 사유를 남기고 제외할 수 있어야 한다.

### 10.2 파일 업로드와 검증

- PDF, XLSX, CSV, ZIP 파일을 받을 수 있어야 한다.
- 파일명 규칙을 검사해야 한다.
- 파일의 대상 연도/월/출처를 기록해야 한다.
- 같은 파일 또는 같은 기간의 중복 파일을 감지해야 한다.
- 원본 파일은 수정하지 않고 보관해야 한다.

### 10.3 메타데이터 추출

- 가능한 파일에서 날짜, 금액, 공급자, 사업자번호, 카드/계좌명, 합계 금액을 추출한다.
- 추출 신뢰도를 기록한다.
- 자동 추출 실패 시 수동 입력을 허용한다.

### 10.4 누락·중복·불일치 탐지

- 1월~12월 월별 자료 존재 여부를 확인한다.
- 전자세금계산서와 카드 결제 중복 가능성을 표시한다.
- PG 정산액과 은행 입금액 차이를 표시한다.
- 오픈마켓 매출액, 수수료, 정산액을 분리해 볼 수 있어야 한다.
- 통관 자료와 수입 관련 매입/관세 자료 연결 후보를 보여준다.

### 10.5 API 수집

- 무료 API는 먼저 `read-only` 또는 `dry-run`으로 검증한다.
- API 키는 환경변수 또는 서버 비밀 저장소에만 둔다.
- 응답 원본은 raw로 저장하고 정규화 테이블과 분리한다.
- API 실패가 기존 정상값을 0으로 덮어쓰지 않아야 한다.
- 수집 실행마다 run_id와 오류 로그를 남긴다.

### 10.6 세무사 전달 패키지

- 패키지 생성 시점의 manifest를 만들어야 한다.
- 패키지에는 요약 Excel, 원본 파일 폴더, 확인 필요 목록, 누락 의심 목록, 설명 메모를 포함한다.
- 패키지 생성 후 파일 목록과 해시를 기록한다.
- 세무사 전달 상태와 전달일을 기록한다.

## 13. 비기능 요구사항

### 13.1 보안

- 인증서, 비밀번호, API 키를 문서·코드·워크플로우 JSON에 저장하지 않는다.
- 민감파일 접근 권한은 대표/관리자/세무 담당자로 제한한다.
- 원본 파일 다운로드 로그와 업로드 로그를 남긴다.
- 외부 전송 전 manifest를 확인한다.

### 13.2 감사 가능성

- 모든 자동 수집은 raw 원본과 정규화 결과를 함께 남긴다.
- 파일명 변경 시 원본 파일명과 변경 파일명을 모두 기록한다.
- 전달 패키지는 생성 시점의 파일 목록과 해시를 남긴다.

### 13.3 안정성

- API 실패가 정상 데이터를 삭제하거나 0으로 덮어쓰지 않아야 한다.
- 수동 업로드 fallback이 항상 있어야 한다.
- 포털 화면 구조가 바뀌어도 핵심 자료 수집은 가이드로 계속 가능해야 한다.
