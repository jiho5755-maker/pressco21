---

## 1. 로드맵 원칙

1. 한 번에 자동 신고까지 가지 않는다.
2. 세무 판단과 자동 수집을 분리한다.
3. API보다 자료 지도와 원본 보관 체계를 먼저 만든다.
4. API 없는 포털은 “수동 다운로드 + 업로드 검증”으로 시작한다.
5. 무료 API와 무료 조회 서비스부터 연결한다.
6. 원본 파일, raw 응답, 정규화 원장을 분리한다.
7. 세무사 전달 패키지가 실제 업무 시간을 줄이는지를 매 Phase에서 확인한다.

## 2. 워크스트림

| 워크스트림 | 설명 |
|---|---|
| W1. 자료 지도 | 세무 자료 목록, 출처, 기간, 다운로드 방법, 필수 여부 정의 |
| W2. 수동 가이드 | 비전공자용 상세 다운로드·저장·전달 절차 |
| W3. 파일 보관 | Company Vault/Nextcloud 폴더 구조, 파일명, manifest, 해시 |
| W4. 수동 업로드 MVP | 체크리스트, 파일 업로드, 상태 관리, 누락 표시 |
| W5. 무료 API 연결 | UNI-PASS, 공공데이터, 오픈뱅킹 가능성, 오픈마켓/PG 후보 |
| W6. 정규화 원장 | raw, evidence file, evidence item, reconciliation queue |
| W7. 검증 자동화 | 월 누락, 중복, 금액 불일치, 설명 필요 거래 탐지 |
| W8. 세무사 협업 | 전달 패키지, 질문 이력, 수정본 재전달 관리 |

## 14. 검증 방법

- 문서 검증: 비전공자 1명이 가이드만 보고 자료 3종 이상 다운로드 가능해야 한다.
- 파일 검증: 파일명 규칙 위반과 중복 파일을 검출해야 한다.
- API 검증: dry-run과 운영 write를 분리해야 한다.
- 세무사 검증: 세무사가 추가 요청한 항목을 문서와 체크리스트에 반영해야 한다.
- 보안 검증: 인증키와 비밀번호가 문서/코드에 남지 않아야 한다.
