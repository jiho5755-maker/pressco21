# 세무 자동화 시스템 팀미팅 기록

> 일시: 2026-04-25  
> 안건: 종합소득세 자료 수집 가이드와 세무 자동화 시스템의 근본 PRD 재정립  
> 참석 관점: 박서연님(CFO), 최민석님(CTO), 윤하늘님(PM)  
> 상태: Draft v1

---

## 1. 회의 배경

최근 종합소득세 납부를 위한 자료 수집 과정이 여러 포털, 은행, 카드사, 오픈마켓, PG, 통관 자료에 흩어져 있어 시간이 많이 들고 누락 위험이 크다는 문제가 확인됐다.

이전 작업에서 UNI-PASS API 수집 초안까지 진행했지만, 이것은 전체 목표 중 `수입/관세 증빙 자동화`에 해당하는 일부다. 이번 회의의 목표는 더 큰 관점에서 “세무 자동화 시스템”의 PRD와 로드맵을 다시 잡는 것이다.

## 2. 제품 방향 합의

### 2.1 한 줄 정의

PRESSCO21 세무 자동화 시스템은 홈택스, 은행, 카드, 오픈마켓, PG, 4대보험, 정부24, UNI-PASS, 내부 CRM/매출 자료를 모아 세무사에게 바로 전달 가능한 증빙 패키지로 정리하는 자료 수집 OS다.

### 2.2 하지 않을 것

- 종합소득세 신고서를 자동 제출하지 않는다.
- 세액을 확정하거나 절세 판단을 자동으로 내리지 않는다.
- 대표 승인 없이 세금 납부, 민원 발급 신청, 신고 제출 버튼을 누르지 않는다.
- 공동인증서, 금융인증서, 민감 비밀번호를 코드나 워크플로우에 저장하지 않는다.

### 2.3 할 것

- 자료 목록을 체크리스트화한다.
- 비전공자가 따라 할 수 있는 다운로드 절차를 만든다.
- 파일명, 기간, 출처, 금액 합계를 기록한다.
- 누락 월, 중복 파일, 금액 불일치 후보를 찾는다.
- 세무사 전달 폴더와 ZIP 패키지를 만든다.
- 무료 API와 무료 조회 서비스를 우선 연결한다.

## 3. 박서연님(CFO) 관점

### 3.1 핵심 문제

- 종합소득세 자료는 “소득 자료”만 모으면 끝나지 않는다.
- 사업 매출, 매입, 카드, 계좌, 인건비, 4대보험, 부가세 신고 이력, 통관/관세, 공제 자료까지 같이 봐야 한다.
- 세무사는 최종 판단을 하지만, 회사 내부에서 빠뜨리지 않고 정리해 주면 추가 요청과 반복 커뮤니케이션이 크게 줄어든다.

### 3.2 필수 자료 묶음

1. 기본 정보
   - 사업자등록증명, 사업장 정보, 전년도 신고서, 환급 계좌
2. 소득 자료
   - 지급명세서, 근로·사업·기타소득, 금융·연금·임대·기타소득 여부
3. 매출 자료
   - 전자세금계산서 매출, 현금영수증 매출, 신용카드 매출, PG, 오픈마켓, 자사몰, 계좌이체 매출
4. 비용 자료
   - 전자세금계산서 매입, 현금영수증 지출증빙, 사업용 카드, 개인카드 사업 지출, 계좌 거래, 배송비, 광고비, 수수료, 재료비
5. 인건비·외주비
   - 급여대장, 원천세 신고, 4대보험 납부, 프리랜서 원천징수, 외주 계약·송금·세금계산서
6. 공제 자료
   - 연말정산 간소화 PDF, 보험료, 의료비, 교육비, 기부금, 연금저축, IRP, 부양가족 자료 제공 동의 여부
7. 신고·납부 이력
   - 부가세 신고서, 원천세 신고서, 국세/지방세 납부내역, 종합소득세 중간예납
8. 설명 메모
   - 큰 금액 입출금, 가족 간 이체, 보증금, 개인/사업 혼합 지출, 누락 가능 계좌/카드

### 3.3 CFO 원칙

- “세무사가 알아서 하겠지”가 아니라 “세무사가 판단할 수 있게 원천자료를 빠짐없이 준다”가 목표다.
- 파일은 임의로 삭제·가공하지 말고 원본을 보존한다.
- 개인/사업 혼합 거래는 숨기지 말고 메모로 구분한다.
- 공제 가능 여부는 자동 판단하지 말고 `세무사 확인 필요` 상태로 둔다.

## 4. 최민석님(CTO) 관점

### 4.1 추천 아키텍처

```text
[데이터 소스]
홈택스 / 은행 / 카드 / 오픈마켓 / PG / 4대보험 / 정부24 / UNI-PASS / CRM / 수동파일
        ↓
[n8n 수집·정규화 계층]
API 호출, 스케줄, 웹훅, 파서, 재시도 큐, 알림
        ↓
[PostgreSQL 원장 + NocoDB 운영 UI]
원천 raw, 정규화 거래, 수집 상태, 검토 큐
        ↓
[관리자 전용 파일 저장소]
원본 PDF·CSV·XLSX, 해시, manifest
        ↓
[월마감/세무사 전달 패키지]
요약 Excel, 원본 묶음, 누락·중복 리포트, ZIP
```

### 4.2 기술 경계

| 영역 | 자동화 범위 | 주의할 점 |
|---|---|---|
| n8n | 공식 API 호출, 스케줄, 파싱, NocoDB 저장, 알림 | 신고 제출·납부 실행 금지 |
| 브라우저 자동화 | 반복 다운로드 보조, 화면 변경 감지 | CAPTCHA/MFA 우회 금지, 인증정보 저장 금지 |
| 수동 가이드 | 인증 로그인, 최초 동의, 민원 발급, 세무사 확인 | 반복 업무는 추후 자동화 후보로 기록 |
| AI | 분류, 요약, 누락 후보, 설명 메모 초안 | 경비 인정·세액 확정 판단 금지 |

### 4.3 UNI-PASS 관련 결정

- 현재 작성된 UNI-PASS API001/API049 n8n 초안은 유지한다.
- API049는 수입신고번호 `impDclrNo` 단건 조회 구조이므로 월간 전체 납부자료 수집 API로 오해하면 안 된다.
- Phase 1은 `API001 화물통관 진행조회 → dclrNo 후보 확보 → API049 수입제세 납부여부 확인` 순서가 안전하다.
- 실제 운영 검증 전까지 n8n 배포, NocoDB 테이블 생성, 환경변수 주입은 명시 승인 후 진행한다.

## 5. 윤하늘님(PM) 관점

### 5.1 제품 목표

- 직원이 가이드만 보고 2025년 귀속 종합소득세 자료를 모을 수 있다.
- 대표는 어떤 자료가 아직 빠졌는지 한 화면에서 볼 수 있다.
- 세무사는 자료 요청보다 검토와 판단에 집중할 수 있다.
- API 자동화 전에도 수동 업로드만으로 세무사 전달 패키지를 만들 수 있다.

### 5.2 Phase 전략

1. Phase 0: 기준선 정의와 자료 지도 작성
2. Phase 1: 비전공자용 종합소득세 자료 수집 가이드
3. Phase 2: 수동 업로드 기반 MVP
4. Phase 3: 무료 API/무료 조회 서비스 자동화
5. Phase 4: 대조·검증 자동화
6. Phase 5: 세무사 협업 패키지
7. Phase 6: 반복 세무 업무 OS화

## 6. 결정 사항

| 결정 | 내용 |
|---|---|
| 제품 범위 | 자동 신고가 아니라 자료 수집·정리·누락 탐지·세무사 전달 |
| 우선 대상 | 2025년 귀속 종합소득세 자료 수집, 단 부가세/원천세/4대보험 자료도 참조 자료로 포함 |
| MVP 방식 | API 자동화보다 수동 업로드 기반 체크리스트·패키지 생성부터 구현 |
| UNI-PASS 위치 | 전체 세무 자동화 중 수입·관세 증빙 모듈 |
| 보안 원칙 | 인증키·인증서·비밀번호 저장 금지, 민감파일은 관리자 전용 저장소 |
| 세무 판단 | 세무사 확인 필요 상태로 분리 |

## 7. 열린 질문

아래 항목은 지금 문서 작성과 Phase 0 실행을 막지는 않지만, 세무사 또는 대표 확인이 필요하다.

4. 세무사 전달 파일 형식은 PDF, XLSX, CSV 중 무엇을 선호하는가?
