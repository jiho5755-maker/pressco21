## 1. Task 101/104 - 메인페이지

### 배포 대상

| 파일 | 편집기 위치 | 비고 |
|------|-----------|------|
| `메인페이지/Index.html` | 메이크샵 > 디자인 편집기 > 메인페이지 HTML | 원본 복원됨 (변경 없음) |
| `메인페이지/js.js` | 메이크샵 > 디자인 편집기 > 메인페이지 JS | 주요 변경 파일 |

### 핵심 변경사항

**Index.html**: 초기 커밋 원본 그대로입니다. 이전에 저장 오류가 발생한 Schema.org, ARIA 등의 HTML 인라인 수정을 모두 제거했습니다.

**js.js**: 아래 3가지 기능이 동적으로 적용됩니다 (HTML 수정 없이):

1. **`initTabAccessibility()`**: section02 New Arrival 탭에 ARIA 속성 동적 추가
2. **`initSchemaOrg()`**: Organization + WebSite JSON-LD를 `<head>`에 동적 삽입
3. **`applyLazyLoading()`**: section01~04 이미지에 `loading="lazy"` 추가 + 카테고리 아이콘 alt 텍스트 보완

### 배포 순서

1. `Index.html`을 편집기에서 열고 **기존 내용과 동일한지** 확인 (변경 없으면 저장 불필요)
2. `js.js`를 편집기에 붙여넣기 후 저장

### 테스트 체크리스트

| # | 확인 항목 | 확인 방법 | 통과 |
|---|----------|----------|------|
| 1 | 편집기 저장 성공 | js.js 저장 시 "데이터 수정 실패" 없는지 확인 | |
| 2 | 페이지 정상 로드 | F12 콘솔에 에러 없는지 확인 | |
| 3 | 기존 기능 정상 | 배너 슬라이드, 탭 전환, 더보기 버튼 동작 확인 | |
| 4 | Schema.org | F12 > Elements > `<head>` 하단에 `application/ld+json` 2개 확인 | |
| 5 | lazy loading | F12 > Elements에서 section02~04 이미지에 `loading="lazy"` 확인 | |

## 2. Task 106 - 비밀번호 토글 버튼 위치

### 배포 대상

| 파일 | 편집기 위치 |
|------|-----------|
| `1초 로그인(킵그로우)/회원 로그인/css.fixed.css` | 회원 로그인 페이지 CSS |
| `1초 로그인(킵그로우)/구매시 로그인/css.fixed.css` | 구매시 로그인 페이지 CSS |
| `1초 로그인(킵그로우)/주문 조회 로그인/css.fixed.css` | 주문 조회 로그인 페이지 CSS |

### 핵심 변경사항

비밀번호 토글(눈 아이콘) 버튼의 CSS 위치를 수정했습니다:

```css
/* 변경 전 (label 포함 li 전체 기준 → 버튼이 위로 올라감) */
#loginWrap .password-toggle {
    top: 50%;
    transform: translateY(-50%);
}

/* 변경 후 (input 56px 영역 기준 중앙) */
#loginWrap .password-toggle {
    bottom: 18px;
}
```

### 배포 순서

3개 CSS 파일 내용이 동일하므로, 하나씩 편집기에 붙여넣기 후 저장합니다.

### 테스트 체크리스트

| # | 확인 항목 | 확인 방법 | 통과 |
|---|----------|----------|------|
| 1 | 회원 로그인 | 비밀번호 입력란 우측 눈 아이콘이 input 영역 중앙에 위치 | |
| 2 | 구매시 로그인 | 동일 확인 | |
| 3 | 주문 조회 로그인 | 동일 확인 | |
| 4 | 토글 기능 | 눈 아이콘 클릭 시 비밀번호 표시/숨기기 정상 동작 | |
| 5 | 모바일 확인 | 모바일에서도 버튼 위치가 input 영역 중앙인지 확인 | |

## 4. Task 105 - SEO 스키마

### 상태: js.js 배포 후 자동 적용

- 메인페이지 Schema.org는 `js.js`의 `initSchemaOrg()`이 동적 삽입
- 간편 구매 페이지의 Product Schema는 `js.fixed.js`의 `initOGMeta()`가 동적 삽입
- **js.js / js.fixed.js 배포 후 자동으로 작동** → 별도 배포 없음
- 검증: [Google Rich Results Test](https://search.google.com/test/rich-results) 에서 실제 URL로 확인 (JS 렌더링 필요)
