# 메이크샵 페이지 협업 표준

이 문서는 PRESSCO21의 메이크샵 페이지를 웹디자이너, 운영자, Codex가 함께 수정할 때 공통으로 따르는 기준이다.

## 기본 원칙

- 메이크샵 편집기에 저장된 코드가 가장 최신 원본이다.
- 저장소는 메이크샵 최신 저장본을 따라간다.
- 각 페이지는 `HTML/CSS/JS` 3개를 한 세트로 관리한다.
- 백업 폴더와 최신 운영본 폴더를 혼용하지 않는다.

## 역할 분리

- 웹디자이너: 메이크샵 편집기에서 시각 디자인, 카피, 레이아웃 수정
- 운영자: 메이크샵 저장 후 저장소 파일 동기화, 배포 반영
- Codex: 최신 저장본 기준으로 기능 수정, 연동 수정, 런타임 오류 수정

## 최신 운영본 저장 규칙

각 페이지 폴더의 루트 `Index.html` 또는 `index.html`, `css.css`, `js.js`가 최신 운영본 저장 위치다.

예:

- `메인페이지/Index.html`
- `메인페이지/css.css`
- `메인페이지/js.js`

또는

- `간편 구매/index.html`
- `간편 구매/css.css`
- `간편 구매/js.js`

## 보관본 저장 규칙

각 페이지 폴더 아래 `저장본/` 폴더를 둔다.

보관 폴더명 형식:

```text
YYYY-MM-DD-작업명
```

예:

```text
저장본/2026-03-15-디자이너수정-1차/
```

항상 아래 3개를 같이 저장한다.

- HTML
- CSS
- JS

## 디자이너 수정 후 절차

1. 메이크샵 편집기에서 저장한다.
2. 저장 직후 메이크샵의 `HTML/CSS/JS` 탭 내용을 저장소의 최신 운영본 파일에 그대로 덮어쓴다.
3. 필요하면 같은 내용을 `저장본/YYYY-MM-DD-작업명/`에 복사해 보관한다.
4. 그 다음에만 Codex에게 수정 요청을 한다.

## Codex 수정 후 배포 절차

1. Codex가 수정한 파일 내용을 메이크샵 편집기에 붙여넣는다.
2. 메이크샵에서 저장한다.
3. 저장된 최종 `HTML/CSS/JS` 탭 내용을 다시 저장소 최신 운영본 파일에 덮어쓴다.
4. 필요하면 같은 내용을 `저장본/YYYY-MM-DD-작업명/`에 보관한다.

## 요청 템플릿

운영자는 Codex에게 아래 형식으로 요청한다.

```text
대상 페이지: 페이지명
최신 기준: 방금 메이크샵 저장본 sync 완료
수정 허용: 링크/연동/기능 수정만
수정 금지: 나머지 디자인, 카피, 버튼 모양
원하는 작업: ...
```

## 금지

- 메이크샵 저장 없이 저장소 파일만 최신본이라고 판단하지 않음
- HTML만 저장하고 CSS/JS를 빼먹지 않음
- 백업 폴더를 최신 운영본 위치로 사용하지 않음
- 같은 페이지를 동시에 두 사람이 메이크샵에서 수정하지 않음

## 한 줄 요약

항상 두 번 sync 한다.

1. 작업 전 sync
2. 배포 후 sync
