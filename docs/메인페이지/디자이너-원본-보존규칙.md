# 메인페이지 디자이너 원본 보존 규칙

## 목적

메인페이지는 웹디자이너와 사내 직원이 별도로 다듬은 디자인이 섞여 있으므로, 파트너클래스 기능을 붙일 때도 비파트너클래스 영역의 디자인을 유지하는 것을 우선 원칙으로 둔다.

## 소스 오브 트루스

다음 중 최신으로 확인된 것을 기준으로 작업한다.

1. 사용자가 직접 붙여준 HTML, CSS, JS
2. 사용자가 올린 운영 스크린샷
3. 메이크샵 현재 저장본
4. 저장소 파일

저장소 파일이 최신이라고 단정하지 않는다.

## 대상 파일

- `메인페이지/Index.html`
- `메인페이지/css.css`
- `메인페이지/js.js`
- `메인페이지/파트너클래스-홈개편/Index.html`
- `메인페이지/파트너클래스-홈개편/css.css`
- `메인페이지/파트너클래스-홈개편/js.js`

위 두 세트는 같이 본다. 한쪽만 고치고 다른 쪽을 방치하지 않는다.

## 변경 금지 기본 영역

명시 요청이 없으면 아래는 유지한다.

- 메인 배너의 시각 구성
- 카테고리 아이콘 레이아웃과 카피
- `New Arrival` 상품 카드 레이아웃
- `Weekly Best` 제목 위치와 상품 배치
- `Learn & Shop` 시각 구조
- `Partner Class Service` 카드 디자인과 카피
- 브랜드 소개 섹션의 시각 구성

## 더보기 버튼 보존 규칙

`New Arrival`과 상품 섹션의 `더보기` 버튼은 디자이너 원본을 유지한다.

- 대상 셀렉터: `.btn-wrap .btn-more`
- 아이콘 자산: `https://jewoo.img4.kr/2026/homepage/Icon/btn-more-icon.png`
- 의도된 형태:
  - 원형 버튼
  - 텍스트 숨김
  - hover 시 아이콘 90도 회전

확인 포인트:

- `width: 65px`
- `height: 65px`
- `border-radius: 50%`
- `font-size: 0`
- `:after` 또는 `::after`에 아이콘 이미지 적용
- `hover`에서 `transform: rotate(90deg)`

운영에서 가로형 pill 버튼으로 보이면 현재 메이크샵 CSS 탭이 이 기준과 동기화되지 않은 상태로 본다.

## 허용 수정 범위

아래는 메인페이지 디자인을 바꾸지 않는 선에서 수정 가능하다.

- 파트너클래스 링크 연결
- 파트너클래스 허브 섹션 연동
- 파트너클래스 API 호출
- 깨진 href 복구
- 저장 실패를 막는 문법 보정
- 라이브 오류를 막는 최소 수정

## 작업 순서

1. 현재 저장소 파일과 사용자 제공 코드를 비교한다.
2. 중복 CSS 셀렉터가 있으면 뒤에 나온 선언을 먼저 확인한다.
3. 비파트너클래스 디자인은 그대로 두고, 필요한 기능만 최소 패치한다.
4. 최종 보고 시 아래 두 항목을 항상 분리한다.
   - 변경한 부분
   - 그대로 유지한 부분

## 검증 순서

1. `python3 ~/.codex/skills/makeshop-d4-dev/scripts/check_makeshop_d4.py`로 메이크샵 제약 검사
2. 버튼 관련 확인
   - 링크 이동
   - 데스크톱 형태
   - 모바일 형태
   - hover 애니메이션
3. 사용자가 라이브 확인을 원하면 메이크샵 저장 후 운영 페이지에서 다시 검증
