# 메인페이지 메이크샵 동기화 체크리스트

비전공자 기준으로 메이크샵 메인페이지를 안전하게 관리하기 위한 절차다.

## 언제 쓰는가

- 웹디자이너가 메이크샵에서 메인페이지를 수정했을 때
- Codex가 메인페이지 코드를 수정한 뒤 다시 메이크샵에 반영할 때

## A. 디자이너 수정 후

### 1. 메이크샵 저장 확인

- HTML 탭 저장
- CSS 탭 저장
- JS 탭 저장

### 2. 현재 파일에 덮어쓰기

아래 3개를 메이크샵 탭 기준으로 그대로 덮어쓴다.

- `메인페이지/Index.html`
- `메인페이지/css.css`
- `메인페이지/js.js`

### 3. 보관본 만들기

필요하면 아래 형식으로 폴더를 만든다.

```text
메인페이지/저장본/YYYY-MM-DD-작업명/
```

예:

```text
메인페이지/저장본/2026-03-15-디자이너수정-1차/
```

그 안에 아래 3개를 같이 저장한다.

- `Index.html`
- `css.css`
- `js.js`

### 4. Codex에 요청할 때

아래 형식으로 보낸다.

```text
대상 페이지: 메인페이지
최신 기준: 방금 메이크샵 저장본 sync 완료
수정 허용: 파트너클래스 링크/연동만
수정 금지: 나머지 디자인, 카피, 버튼 모양
원하는 작업: ...
```

## B. Codex 수정 후 메이크샵 반영할 때

### 1. Codex가 준 파일 반영

- `Index.html` 내용을 메이크샵 HTML 탭에 붙여넣기
- `css.css` 내용을 메이크샵 CSS 탭에 붙여넣기
- `js.js` 내용을 메이크샵 JS 탭에 붙여넣기

### 2. 메이크샵 저장

- HTML 저장
- CSS 저장
- JS 저장

### 3. 최종 탭 내용을 다시 현재 파일에 덮어쓰기

메이크샵에 실제 저장된 최종 내용을 다시 아래 3개에 덮어쓴다.

- `메인페이지/Index.html`
- `메인페이지/css.css`
- `메인페이지/js.js`

이 단계가 빠지면 다음 작업 때 저장소 기준이 오래될 수 있다.

## C. 항상 지킬 것

- 메인페이지는 `HTML/CSS/JS` 3개를 한 세트로 본다
- 한 번에 한 사람만 메이크샵에서 수정한다
- 최신 기준이 메이크샵인지, 첨부 코드인지 먼저 정한다
- 비파트너클래스 디자인은 명시 요청 없이는 바꾸지 않는다

## D. 가장 자주 나는 실수

- 메이크샵에서 저장은 했는데 저장소 파일은 안 바꿈
- CSS만 최신이고 HTML/JS는 예전 파일 그대로 둠
- 디자이너가 수정한 뒤 바로 Codex에게 요청하지 않고 예전 파일 기준으로 수정 시작
- `기존 코드/` 폴더를 최신본 보관 위치로 착각

## E. 한 줄 요약
