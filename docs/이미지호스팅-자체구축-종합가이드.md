# PRESSCO21 자체 이미지 호스팅 서버 종합 가이드

> 작성일: 2026-03-19 | 버전: v1.0
> 기술 스택: MinIO (Community Edition) + Nginx + Let's Encrypt + Oracle Cloud Free Tier

---

## 1. 개요

### 1-1. 왜 자체 이미지 호스팅인가

PRESSCO21은 메이크샵 부가서비스인 이미지 호스팅(jewoo.img4.kr)을 사용해왔다.
이 서비스는 FTP 업로드, HTTP(비암호화), 폴더당 5,000개 제한 등 구식 방식으로 운영되며,
직원들의 업무 효율과 브랜드 통제력에 한계가 있었다.

자체 이미지 호스팅 서버를 구축하여 다음을 달성한다:

| 목표 | 설명 |
|------|------|
| 비용 절감 | 메이크샵 유료 부가서비스 → Oracle Free Tier 무료 |
| 보안 강화 | HTTP → HTTPS 전환 |
| 자동화 연동 | n8n S3 노드로 이미지 처리 파이프라인 구축 |
| 브랜드 통제 | 자사 도메인(img.pressco21.com) + 워터마크 + 일관된 이미지 관리 |
| 직원 편의 | FTP 프로그램 → 브라우저 드래그 앤 드롭 |

### 1-2. 기존 vs 신규 비교

| 항목 | 기존 (메이크샵) | 신규 (자체 MinIO) |
|------|---------------|------------------|
| URL | `http://jewoo.img4.kr/파일명` | `https://img.pressco21.com/images/경로/파일명` |
| 프로토콜 | HTTP (비암호화) | HTTPS (SSL 인증서) |
| 업로드 방식 | FTP 프로그램 필요 | 브라우저 드래그 앤 드롭 |
| 월 비용 | 유료 (부가서비스) | $0 (Free Tier) |
| 자동화 | 불가능 | n8n S3 노드 즉시 연동 |
| 폴더 제한 | 5,000개/폴더 | 무제한 |
| API | 없음 | S3 호환 API (수천 개 도구 호환) |
| 용량 | 4,071MB / 무제한 | Oracle Free Tier 디스크 여유분 |
| 트래픽 | 월 500GB | Oracle Free Tier 월 10TB |
| 만료 | 2027-12-20 | 영구 (Free Tier) |

## 2. 시스템 구성

### 2-1. 아키텍처

```
[직원 브라우저]
     |
     v
[minio.pressco21.com] ← 관리 콘솔 (파일 업로드/관리)
     |
     v
[MinIO 컨테이너] ← Docker, n8n_n8n-network
     |
     v
[/data/minio] ← 이미지 파일 저장 (서버 디스크)

     v

     |
     v
[고객 브라우저] ← HTTPS로 이미지 로딩
```

### 2-2. 접속 정보

| 항목 | 값 |
|------|-----|
| 이미지 서빙 URL | `https://img.pressco21.com/images/{폴더}/{파일명}` |
| 관리 콘솔 URL | `https://minio.pressco21.com` |
| 관리자 ID | `pressco21` |
| 관리자 PW | `/jang040300` |
| API 엔드포인트 (내부) | `http://minio:9000` |
| 콘솔 포트 (내부) | `http://minio:9001` |
| Docker 네트워크 | `n8n_n8n-network` |
| 데이터 볼륨 | `/data/minio` |

### 2-3. 버킷 및 폴더 구조

```
images/ (공개 버킷 - anonymous download)
  |
  +-- products/        상품 이미지

  +-- sns/             SNS 콘텐츠
  +-- internal/        내부 문서 (CS 사진, 서류 등)
```

## 3. 구축 과정 (비전공자용 상세 가이드)

> 이 섹션은 처음부터 다시 구축해야 할 때 참고하는 가이드입니다.

### 3-1. 사전 준비

필요한 것:
- Oracle Cloud 서버 SSH 접속 (키: `~/.ssh/oracle-n8n.key`)
- Namecheap 도메인 관리자 계정
- 서버에 Docker가 설치되어 있어야 함

### 3-2. MinIO Docker 컨테이너 실행

```bash
# 서버에 SSH 접속
ssh -i ~/.ssh/oracle-n8n.key ubuntu@158.180.77.201

# 데이터 저장 디렉토리 생성
sudo mkdir -p /data/minio
sudo chown 1000:1000 /data/minio

# MinIO 실행
docker run -d \
  --name minio \
  --restart always \
  -p 127.0.0.1:9000:9000 \
  -p 127.0.0.1:9001:9001 \
  -v /data/minio:/data \
  -e "MINIO_ROOT_USER=pressco21" \
  -e "MINIO_ROOT_PASSWORD=/jang040300" \
  --network n8n_n8n-network \
  quay.io/minio/minio:latest server /data --console-address ":9001"
```

각 옵션 설명:
- `--restart always`: 서버 재시작 시 자동 실행
- `-p 127.0.0.1:9000:9000`: API 포트 (내부 전용)
- `-p 127.0.0.1:9001:9001`: 관리 콘솔 포트 (내부 전용)
- `-v /data/minio:/data`: 이미지 파일 영구 저장 위치
- `--network n8n_n8n-network`: n8n과 같은 네트워크 (n8n에서 `http://minio:9000`으로 접근)

### 3-3. DNS 레코드 추가 (Namecheap)

1. https://www.namecheap.com 로그인
2. Domain List → `pressco21.com` → Manage → Advanced DNS
3. "ADD NEW RECORD" 클릭하여 2개 추가:

| Type | Host | Value | TTL |
|------|------|-------|-----|
| A Record | `img` | `158.180.77.201` | Automatic |
| A Record | `minio` | `158.180.77.201` | Automatic |

### 3-4. Nginx 리버스 프록시 설정

```bash
# 서버에서 실행
sudo nano /etc/nginx/sites-available/minio
```

아래 내용 입력:

```nginx
# 이미지 서빙 (공개)
server {
    listen 80;
    server_name img.pressco21.com;

    location / {
        proxy_pass http://127.0.0.1:9000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}

# 관리 콘솔 (내부 관리용)
server {
    listen 80;
    server_name minio.pressco21.com;

    location / {
        proxy_pass http://127.0.0.1:9001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

```bash
# 설정 활성화
sudo ln -sf /etc/nginx/sites-available/minio /etc/nginx/sites-enabled/minio

# 업로드 크기 제한 확대 (nginx.conf의 http 블록에 추가)
# client_max_body_size 50M;

# 설정 검증 및 적용
sudo nginx -t && sudo systemctl reload nginx
```

### 3-5. SSL 인증서 발급

```bash
sudo certbot --nginx -d img.pressco21.com --non-interactive --agree-tos --email admin@pressco21.com
sudo certbot --nginx -d minio.pressco21.com --non-interactive --agree-tos --email admin@pressco21.com
```

인증서는 자동 갱신됩니다 (certbot timer).

### 3-6. MinIO CLI 설치 및 버킷 생성

```bash
# MinIO 클라이언트(mc) 설치
curl -sL https://dl.min.io/client/mc/release/linux-arm64/mc -o /tmp/mc
chmod +x /tmp/mc
sudo mv /tmp/mc /usr/local/bin/mc

# MinIO 서버 등록
mc alias set pressco http://127.0.0.1:9000 pressco21 '/jang040300'

# 공개 이미지 버킷 생성
mc mb pressco/images

# 공개 읽기 정책 설정
mc anonymous set download pressco/images
```

### 3-7. 폴더 구조 생성

```bash
# .keep 파일로 폴더 생성 (MinIO는 빈 폴더를 지원하지 않음)
echo "" > /tmp/.keep
mc cp /tmp/.keep pressco/images/products/.keep
mc cp /tmp/.keep pressco/images/content/.keep
mc cp /tmp/.keep pressco/images/class/.keep
mc cp /tmp/.keep pressco/images/sns/.keep
mc cp /tmp/.keep pressco/images/internal/.keep
```

### 3-8. 비밀번호 변경 방법

```bash
# 1. 기존 컨테이너 중지 및 삭제 (데이터는 /data/minio에 안전)
docker stop minio
docker rm minio

# 2. 새 비밀번호로 재실행
docker run -d \
  --name minio \
  --restart always \
  -p 127.0.0.1:9000:9000 \
  -p 127.0.0.1:9001:9001 \
  -v /data/minio:/data \
  -e "MINIO_ROOT_USER=새유저명" \
  -e "MINIO_ROOT_PASSWORD=새비밀번호" \
  --network n8n_n8n-network \
  quay.io/minio/minio:latest server /data --console-address ":9001"

# 3. mc alias도 업데이트
mc alias set pressco http://127.0.0.1:9000 새유저명 '새비밀번호'
```

컨테이너를 삭제해도 `/data/minio`에 저장된 이미지 파일은 절대 사라지지 않습니다.

### 4-1. 절대 하지 말 것

| 항목 | 이유 |
|------|------|
| 기존 jewoo.img4.kr 이미지 일괄 삭제 | 기존 상품 상세페이지 이미지 깨짐 |
| MinIO 버전 무분별 업데이트 | 2025년 말부터 Community Edition Web UI 기능 축소 진행 중 |
| 모든 직원 동시 전환 | 혼란 발생, 1명씩 순차 전환 |
| 파일명에 한글/공백 사용 | URL 인코딩 문제 (%20 등) |
| FTP와 MinIO 동시 사용 | 어디에 원본이 있는지 혼란 |

### 4-2. 반드시 할 것

| 항목 | 이유 |
|------|------|
| 기존 이미지는 메이크샵에 그대로 유지 | 안전한 병행 운영 |
| 신규 이미지만 MinIO에 업로드 | 점진적 전환으로 리스크 최소화 |
| 파일명은 영어+숫자+하이픈만 사용 | URL 호환성 보장 |
| 업로드 후 URL 접근 테스트 | 실제 이미지 노출 확인 |
| .secrets 파일에 인증정보 기록 | 비밀번호 분실 방지 |

### 4-3. 파일명 규칙

```
[products/]  {상품코드}_{순서}.webp       예: P2024001_01.webp
[content/]   YYYY-MM-DD_{설명}.webp       예: 2026-03-19_spring-banner.webp
[sns/]       {채널}_{YYYYMMDD}_{순서}.webp 예: ig_20260319_01.webp
[class/]     {파트너ID}_{타입}.webp        예: P001_thumbnail.webp
[internal/]  YYYYMM/{주문번호}_{용도}.jpg  예: 202603/ORD-001_claim.jpg
```

### 4-4. URL 사용법

이미지를 업로드하면 아래 형식으로 접근 가능:

```
https://img.pressco21.com/images/products/P2024001_01.webp
https://img.pressco21.com/images/class/P001_thumbnail.webp
```

메이크샵 상품 상세, 게시판, 개별 페이지 등에서 이 URL을 이미지 경로로 사용하면 됩니다.

## 6. n8n 연동 가이드

### 6-1. n8n S3 Credential 설정

n8n 관리 콘솔에서 Credential 추가:

| 항목 | 값 |
|------|-----|
| Credential Type | S3 (AWS S3가 아닌 S3 Compatible) |
| Endpoint | `http://minio:9000` |
| Region | `us-east-1` |
| Access Key | (MinIO 콘솔에서 발급) |
| Secret Key | (MinIO 콘솔에서 발급) |
| Force Path Style | `true` (필수!) |

> Force Path Style을 빠뜨리면 `{bucket}.minio:9000` 형태로 DNS 조회하여 실패합니다.

### 6-2. 즉시 구축 가능한 워크플로우

**WF-IMG-01: 파트너클래스 썸네일 자동 처리**
```
WF-17 Approve 승인
  → 파트너 업로드 이미지 다운로드
  → 이미지 검증 (크기, 확장자, 해상도)
  → MinIO S3 API로 업로드 (/images/class/{class_id}/thumbnail.jpg)
  → NocoDB Classes 테이블 thumbnail_url 업데이트
  → 텔레그램 알림
```

**WF-IMG-02: SNS 콘텐츠 아카이브**
```
Instagram/YouTube API에서 최근 게시물 이미지 수집
  → MinIO /images/sns/{플랫폼}/{연월}/{post_id}.jpg 업로드
  → NocoDB 로그 테이블에 기록
```

**WF-IMG-03: 통합 백업**
```
매일 03:00 Cron
  → PostgreSQL dump → MinIO /images/internal/backups/db/
  → n8n 워크플로우 JSON 내보내기 → MinIO /images/internal/backups/n8n/
  → 보존 정책: DB 7일, n8n 30일, 이미지 영구
```

## 12. 트러블슈팅

### Q. 업로드 시 "File size too large" 에러
A. Nginx 업로드 제한 때문. 현재 50MB로 설정됨.
더 큰 파일이 필요하면 `/etc/nginx/nginx.conf`의 `client_max_body_size` 값 변경.

### Q. 이미지 URL 접근 시 "NoSuchKey" 에러
A. 파일명에 공백이 있으면 URL에서 `%20`으로 인코딩 필요.
파일명에 공백을 쓰지 않는 것을 권장.

### Q. 관리 콘솔에 접속이 안 됨
A. MinIO 컨테이너 상태 확인:
```bash
ssh -i ~/.ssh/oracle-n8n.key ubuntu@158.180.77.201
docker ps --filter name=minio
# 컨테이너가 없으면 재실행 (3-2 섹션 참조)
```

### Q. 비밀번호를 잊어버림
A. `/Users/jangjiho/Desktop/n8n-main/pressco21/.secrets` 파일에서 MINIO 관련 항목 확인.

### Q. 이미지가 삭제됨
A. MinIO 버전 관리를 활성화하면 삭제된 파일도 복구 가능:
```bash
mc version enable pressco/images
```

```
      ├── MinIO 이미지 호스팅 서버에 업로드 (웹 서빙용)
      ├── 회사 백업 보관고에 저장 (원본 보관용)
      ├── WebP 변환 + 리사이즈 + 워터마크 자동 처리
      └── 텔레그램 알림: "연출컷 3장 업로드 완료"
  → Google Drive 원본 삭제 (용량 무한 순환)
```
