## 흐름 1: 수강생이 강의를 찾아 예약하는 과정

```
수강생 행동                      시스템이 하는 일
──────────────────────────────────────────────────────────
1. 목록 페이지 접속
   (foreverlove.co.kr/shop/?id=2606)
                          →  [목록/js.js] n8n에 "강의 목록 주세요" 요청
                          →  [WF-01] NocoDB에서 강의 꺼내와 응답
                          →  화면에 강의 카드 표시

2. 마음에 드는 강의 클릭
   (상세 페이지 ?id=2607)
                          →  [상세/js.js] n8n에 "이 강의 상세정보 주세요" 요청
                          →  [WF-01] 강의 정보 + 강사 정보 + 후기 응답

3. "예약하기" 버튼 클릭
   (로그인 필요)
                          →  [상세/js.js] n8n에 "예약 기록해주세요" 요청
                          →  [WF-04] NocoDB에 예약 저장 (PENDING 상태)
                          →  메이크샵 결제 페이지로 이동

4. 결제 완료
                          →  메이크샵이 주문 기록
                          →  [WF-05] 15분마다 자동으로 새 주문 감지
                          →  NocoDB 정산 기록 생성
                          →  수강생에게 확인 이메일 발송
                          →  파트너에게 새 예약 알림 이메일 발송
```

## 흐름 2: 파트너가 강의를 등록하는 과정

```
파트너 행동                      시스템이 하는 일
──────────────────────────────────────────────────────────
1. 파트너 신청 페이지 접속
   (?id=2609)
                          →  이름, 공방명, 소개 등 입력 후 제출
                          →  [WF-07] NocoDB 신청 테이블에 저장
                          →  관리자에게 텔레그램 알림
                             "새 파트너 신청이 왔어요!"

2. 관리자가 NocoDB에서 승인 클릭
                          →  [WF-APPROVE] 자동 감지
                          →  파트너 계정 생성 (tbl_Partners)
                          →  메이크샵 회원 등급 "파트너"로 변경
                          →  파트너에게 승인 축하 이메일 발송

3. 강의 등록 페이지 접속
   (?id=2611)
                          →  강의명, 가격, 일정, 소개 입력 후 제출
                          →  [WF-16] NocoDB에 status=pending으로 저장
                          →  관리자에게 텔레그램 알림
                             "새 강의 검수 요청이 왔어요!"

4. 관리자가 강의 승인 (status → active)
                          →  목록 페이지에 강의 노출 시작!
```

## 흐름 3: 수수료 정산 과정

실제 수수료 정산은 **메이크샵 결제 → PRESSCO21 계좌 수령 → 파트너 계좌 직접 송금** 방식입니다.

```
수강생           메이크샵(PG사)         PRESSCO21(운영자)          파트너
───────────────────────────────────────────────────────────────────
1. 강의 결제
  (카드/계좌이체)
      ↓ 전액
              메이크샵이 보관
              (월 1~2회 정산)
                   ↓
                        PRESSCO21 계좌로 일괄 입금
                               ↓
                        [WF-05] 주문 자동 감지
                        NocoDB에 정산 기록 생성
                        (결제금액 + 수수료율 계산)
                               ↓
                        관리자가 정산 내역 확인 후
                        수수료 제한 금액을 파트너에게 직접 송금
                                                    ↓
                                               파트너 계좌 수령
```

**수수료 구조 (PRESSCO21이 가져가는 비율):**

| 파트너 등급 | PRESSCO21 수수료 | 파트너에게 송금 |
|-----------|----------------|--------------|
| SILVER | 20% | 결제금액의 80% |
| GOLD | 25% | 결제금액의 75% |
| PLATINUM | 30% | 결제금액의 70% |

**NocoDB tbl_Settlements가 하는 역할:**
- 누가 어느 강의를 결제했는지, 금액이 얼마인지 자동 기록
- 수수료 계산 내역 보관 (관리자가 직접 확인 가능)
- 관리자가 송금 처리 후 status를 COMPLETED로 변경

> **적립금(포인트)은 별도 이벤트 전용** — 수강 후기 작성 시 1,000원 적립 등
> 이벤트성으로만 운영하며, 파트너 수수료 정산 송금과는 완전히 별개입니다.

### 파트너 대금 자동 송금 로드맵

| 단계 | 방식 | 시점 |
|------|------|------|
| **1단계 (현재)** | 관리자가 NocoDB 정산 내역 확인 후 파트너 계좌에 직접 송금 | 즉시 운영 가능 |
| **2단계 (예정)** | 세틀뱅크 지급대행 API 연동 — n8n WF-05에 자동 계좌이체 추가 | 서비스 규모 성장 후 |

**1단계 — 현재 운영 방식 (수동 직접 납입):**

```
1. 메이크샵이 매월 1~2회 PRESSCO21 계좌로 결제 대금 일괄 입금
2. 관리자가 NocoDB tbl_Settlements에서 파트너별 정산 내역 확인
   (WF-05가 주문 감지 + 수수료 계산을 자동으로 기록해 둠)
3. 수수료(SILVER 20%, GOLD 25%, PLATINUM 30%)를 제한 후 파트너 계좌로 직접 이체
4. 이체 완료 후 NocoDB에서 해당 레코드의 status → COMPLETED 변경
```

**2단계 — 예정: 세틀뱅크 B2B 지급대행 API 자동화:**

- 세틀뱅크 지급대행 서비스(Settlebank Payout API) 가입 후 API 키 발급
- n8n WF-05의 D+3 정산 실행 배치에 "계좌이체 자동 발송" 노드 추가
- 수동 이체 없이 정산 기간(D+3) 도래 시 파트너 등록 계좌로 자동 송금
- **도입 시점 기준**: 파트너 10명 이상, 월 정산 건수 50건 이상으로 수동 처리가 번거로워지는 시점

> **결정사항 (2026-02-26)**: 초기는 수동 직접 납입으로 운영하고, 서비스가 성장하면 세틀뱅크 자동화를 도입한다.

### 수동 작업 4: 메이크샵 편집기 재저장

### 수동 작업 5: 자동 백업 crontab 설정

**왜 필요한가?** 서버 데이터를 매일 자동으로 백업하려면 서버에서 스케줄을 등록해야 합니다.

```bash
# 서버 접속
ssh -i ~/.ssh/oracle-n8n.key ubuntu@158.180.77.201

# 스크립트 폴더 생성
mkdir -p /home/ubuntu/scripts

# backup.sh 파일 생성 (아래 내용 붙여넣기)
nano /home/ubuntu/scripts/backup.sh
```

```bash
#!/bin/bash
TIMESTAMP=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/home/ubuntu/backups/${TIMESTAMP}"
mkdir -p "${BACKUP_DIR}"
cp /home/ubuntu/nocodb/nocodb_data/noco.db "${BACKUP_DIR}/noco_${TIMESTAMP}.db"
cp /home/ubuntu/docker-compose.yml "${BACKUP_DIR}/docker-compose_${TIMESTAMP}.bak"
echo "[${TIMESTAMP}] 백업 완료: ${BACKUP_DIR}"
```

```bash
# 실행 권한 부여
chmod +x /home/ubuntu/scripts/backup.sh

# 스케줄 등록 (매일 새벽 3시)
crontab -e
# 아래 두 줄 추가:
0 3 * * * /home/ubuntu/scripts/backup.sh >> /home/ubuntu/backups/backup.log 2>&1
30 3 * * * find /home/ubuntu/backups -maxdepth 1 -type d -mtime +7 -exec rm -rf {} \;
```

## 문제 발생 시 빠른 대처법

### 페이지가 안 열린다
```bash
ssh -i ~/.ssh/oracle-n8n.key ubuntu@158.180.77.201
docker ps   # 컨테이너 상태 확인
docker start n8n      # n8n이 꺼져 있으면
docker start nocodb   # NocoDB가 꺼져 있으면
```

### 데이터가 안 나온다
```
n8n.pressco21.com 접속 → 해당 워크플로우 클릭
→ 최근 실행 기록(Executions) 확인
→ 빨간 X 표시면 오류 내용 확인
```

### NocoDB에서 nocodb가 n8n 네트워크와 끊어졌다
```bash
docker network connect n8n_n8n-network nocodb
```
