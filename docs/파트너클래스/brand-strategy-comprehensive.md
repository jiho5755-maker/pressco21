## 1. 파트너 프로그램 브랜드 네이밍

### 1-1. 프로그램 공식 명칭

| 항목 | A안 (채택 권고) | B안 (대안) |
|------|----------------|-----------|
| **프로그램명** | **PRESSCO21 블룸 파트너** | PRESSCO21 크리에이터 파트너 |
| **영문** | PRESSCO21 Bloom Partner | PRESSCO21 Creator Partner |
| **약칭** | 블룸 파트너 | 크리에이터 파트너 |

**A안 채택 근거:**
- "블룸(Bloom)"은 꽃이 피어나는 것을 의미하며, PRESSCO21의 핵심 정체성인 "꽃"과 직결됩니다.
- "파트너가 피어나다", "파트너와 함께 꽃이 피다"라는 이중 의미를 담습니다.
- 기존 시스템 프롬프트의 Bloom/Garden/Atelier 등급 체계와 자연스럽게 연결됩니다.
- 브랜드 슬로건 "Forever and ever and Blooming"의 Blooming과 어원을 공유합니다.

**B안 설명:**
- "크리에이터"는 보편적이고 이해하기 쉽지만, 꽃 공예 브랜드만의 차별성이 약합니다.
- 유튜브/인스타 크리에이터와 혼동될 여지가 있습니다.

### 1-2. 파트너 호칭

| 항목 | A안 (채택 권고) | B안 (대안) |
|------|----------------|-----------|
| **호칭** | **꽃 선생님** | 플로리스트 파트너 |
| **공식 문서용** | 파트너 선생님 | 파트너 강사님 |
| **수강생 대상** | [이름] 선생님 | [이름] 강사님 |

**A안 채택 근거:**
- "꽃 선생님"은 따뜻하고, 초보 수강생에게 친근감을 줍니다.
- 공식 문서에서는 "파트너 선생님"으로, 수강생 대면에서는 "[이름] 선생님"으로 사용합니다.
- "강사"보다 "선생님"이 PRESSCO21 톤에 더 부합합니다 (교육적 권위보다 따뜻한 안내자).

### 1-3. 등급명 검토 및 개선 제안

#### 현행 등급 (SILVER / GOLD / PLATINUM)

현행 등급명은 보편적이고 직관적이라는 장점이 있지만, 압화/꽃 공예 브랜드만의 색채가 부족합니다.

#### A안: 자연 메타포 등급 (권고)

| 등급 | 현행 | 제안 | 아이콘 | 브랜드 메시지 |
|------|------|------|--------|-------------|
| 입문 | SILVER | **Bloom (블룸)** | 꽃봉오리 | "첫 번째 꽃이 피어납니다" |
| 성장 | GOLD | **Garden (가든)** | 활짝 핀 꽃 | "아름다운 정원이 자라고 있어요" |
| 마스터 | PLATINUM | **Atelier (아틀리에)** | 꽃다발 | "당신만의 아틀리에가 완성되었습니다" |

**장점:**
- 꽃 공예 브랜드에 완벽히 부합하는 세계관
- "Bloom -> Garden -> Atelier"로 성장 서사가 자연스러움 (씨앗 -> 정원 -> 공방)
- PRESSCO21만의 고유한 브랜드 자산이 됨

**단점:**
- 기존 GAS/UI 코드에서 SILVER/GOLD/PLATINUM으로 구현 완료 상태
- 수강생 대상 인지도가 SILVER < GOLD < PLATINUM보다 낮을 수 있음

#### B안: 하이브리드 (현실적 절충)

| 용도 | 등급명 사용 |
|------|-----------|
| **수강생 대상 (UI/배지)** | SILVER / GOLD / PLATINUM (현행 유지) |
| **파트너 내부 커뮤니케이션** | Bloom / Garden / Atelier |
| **파트너 승급 축하 이메일** | "Garden 파트너로 성장하셨습니다" |
| **파트너 대시보드** | Bloom / Garden / Atelier |

**채택 권고: B안 (하이브리드)**
- 수강생 대상: 보편적 SILVER/GOLD/PLATINUM 유지 (신뢰감, 즉각적 이해)
- 파트너 대상: Bloom/Garden/Atelier 사용 (소속감, 브랜드 정체성)
- 기존 코드 변경 최소화하면서 브랜드 차별화 달성

### 1-4. 프로그램 슬로건 후보

| 순위 | 슬로건 | 톤 |
|------|--------|------|
| 1 (채택) | **"꽃의 아름다움을, 함께 전해요"** | 따뜻 + 포용 |
| 2 | "당신의 수업에 날개를 달아드려요" | 실용 + 따뜻 |
| 3 | "30년 노하우, 함께 나누는 꽃 이야기" | 신뢰 + 따뜻 |

## 2. 파트너 가입 여정 설계

### 2-1. 전체 여정 맵

```
[ STEP 1 ]              [ STEP 2 ]              [ STEP 3 ]              [ STEP 4 ]              [ STEP 5 ]
 가입 신청               관리자 검토              승인/거절                필수 교육                활동 시작
 (Google Forms)          (1~3 영업일)            (이메일 안내)            (22분 영상+퀴즈)         (SILVER 활성)
     |                       |                       |                       |                       |
     v                       v                       v                       v                       v
[신청 접수 확인]        [검토 진행 중]          [승인: 코드 발급]       [합격: 인증서]          [첫 클래스 등록]
 이메일 자동 발송        (내부 프로세스)          [거절: 사유+재신청]      [불합격: 재응시]        환영 메시지
 텔레그램 관리자 알림                             이메일 발송              이메일 발송
```

### 2-2. 단계별 핵심 메시지

#### STEP 1: 가입 신청

**파트너에게 전달할 메시지:**

> 관심을 가져주셔서 감사합니다.
> 작성해주신 정보를 바탕으로 1~3 영업일 이내에 검토 결과를 안내해드릴게요.
> 궁금하신 점이 있으시면 언제든 편하게 연락주세요.

**핵심 감정:** 환영 + 안심 (기다리는 동안 불안하지 않도록)

#### STEP 2: 관리자 검토

**내부 프로세스 -- 파트너에게는 별도 커뮤니케이션 없음**
- 신청 접수 확인 이메일에 "1~3 영업일"이라는 기대치를 이미 설정
- 3 영업일 초과 시 지연 안내 이메일 발송 (아래 템플릿 포함)

#### STEP 3: 승인

**파트너에게 전달할 메시지:**

> 환영합니다! PRESSCO21 블룸 파트너가 되셨습니다.
> 파트너 코드를 발급해드렸어요. 이제 교육을 이수하시면 첫 클래스를 등록하실 수 있어요.
> 3단계만 더 남았어요!

**핵심 감정:** 축하 + 기대 + 다음 행동 명확히

#### STEP 3 (거절): 거절 시 커뮤니케이션

**원칙:**
- 거절 사유를 구체적으로 안내
- 재신청 가능함을 명시
- 개선 포인트를 건설적으로 제안
- 절대 "부적합", "미달", "거절"이라는 단어 사용 금지

**거절 이메일 카피:**

> **제목:** [PRESSCO21] 파트너 신청 검토 결과를 안내해드려요
>
> [신청자명]님, 안녕하세요.
> PRESSCO21 파트너 프로그램에 관심을 가져주셔서 감사합니다.
>
> 신중하게 검토한 결과, 현재 단계에서는 몇 가지 보완이 필요해요.
>
> **보완이 필요한 부분:**
> - [구체적 사유 1: 예) 포트폴리오에 수업 진행 사례가 포함되어 있지 않습니다]
> - [구체적 사유 2: 예) 활동 지역의 수업 공간 정보가 확인되지 않습니다]
>
> **다음 단계:**
> 위 사항을 보완하신 후 언제든 다시 신청해주세요.
> 파트너 신청에는 횟수 제한이 없으며, 보완 후 재신청 시 우선 검토해드립니다.
>
> 궁금하신 점이 있으시면 pressco21@foreverlove.co.kr로 편하게 연락주세요.
>
> PRESSCO21 드림

**톤 분석:**
- "거절"이라는 단어 없이 "보완이 필요한 부분"으로 표현
- "현재 단계에서는"으로 영구 거절이 아님을 암시
- 구체적 개선 포인트 + 재신청 가이드 제공
- "언제든", "우선 검토" 등 문턱을 낮추는 표현

#### STEP 4: 필수 교육

**파트너에게 전달할 메시지:**

> 영상 3개(22분)만 보시고 퀴즈를 풀면 돼요.
> 부담 없이 편하게 시청해주세요. 70점 이상이면 합격이에요.
> 합격하시면 바로 SILVER 파트너로 활성화됩니다!

**핵심 감정:** 부담 없음 + 달성 가능 + 빠른 보상

#### STEP 5: 활동 시작

**파트너에게 전달할 메시지:**

> 축하합니다! Bloom(블룸) 파트너로 활동을 시작하셨어요.
> 첫 클래스를 등록해볼까요? 좋은 사진과 상세한 소개글은
> 수강생 예약률을 높이는 데 큰 도움이 됩니다.

**핵심 감정:** 성취감 + 실용적 가이드 + 첫 행동 유도

## 4. 텔레그램 알림 메시지 템플릿 (4종)

> 관리자 텔레그램 채널에 자동 발송되는 알림 메시지입니다.
> n8n 워크플로우에서 트리거됩니다.
> 톤: 간결 + 정보 중심 + 이모지 최소화 (관리자 업무용이므로 브랜드 톤보다 실용성 우선)

### 4-1. 새 파트너 신청 알림

**트리거**: Google Forms 파트너 신청 접수 시
**채널**: 관리자 텔레그램

```
[파트너 신청] 새 신청이 접수되었습니다

신청자: {applicantName}
전문 분야: {specialtyField}
활동 지역: {activityRegion}
이메일: {applicantEmail}
접수 시각: {timestamp}
접수 번호: {applicationNumber}

포트폴리오: {portfolioLink}

-> 관리자 승인 페이지에서 검토해주세요.
{adminReviewUrl}
```

**설계 의도:**
- 첫 줄의 `[파트너 신청]` 태그로 알림 유형 즉시 식별
- 검토에 필요한 최소 정보만 포함
- 포트폴리오 링크로 즉시 검토 가능
- 승인 페이지 URL로 바로 행동 가능

### 4-2. 새 클래스 예약 알림

**트리거**: 메이크샵 주문 감지 후 GAS 정산 처리 시
**채널**: 관리자 텔레그램

```
[새 예약] 클래스 예약이 접수되었습니다

클래스: {className}
파트너: {partnerName} ({partnerCode})
수강생: {studentName} ({personnel}명)
수업 일시: {classDate} {classTime}
결제 금액: {totalAmount}원
주문번호: {orderNumber}

수수료: {commissionAmount}원 ({commissionRate}%)
파트너 수령: {partnerAmount}원

-> 정산 시트에 자동 기록 완료
```

**설계 의도:**
- `[새 예약]` 태그로 유형 구분
- 금액 관련 정보(결제/수수료/파트너 수령)를 한눈에
- "자동 기록 완료"로 별도 행동 불필요함을 명시

### 4-3. 정산 완료 알림

**트리거**: D+3 정산 배치 처리 완료 시
**채널**: 관리자 텔레그램

```
[정산 완료] 적립금 지급이 완료되었습니다

대상: {partnerName} ({partnerCode})
주문번호: {orderNumber}
클래스: {className}
결제 금액: {totalAmount}원
수수료: {commissionAmount}원
적립금 지급: {reserveAmount}원

지급 결과: 성공
적립금 API 응답: {apiResponse}

-> 정산 시트 상태: "완료"로 업데이트됨
```

**설계 의도:**
- 정산 완료를 관리자가 추적할 수 있도록 상세 정보 포함
- API 응답 코드를 포함하여 문제 발생 시 디버깅 용이
- 시트 상태 업데이트 확인으로 데이터 정합성 안내

### 4-4. 정산 실패 긴급 알림

**트리거**: 적립금 API 호출 실패 또는 정산 오류 발생 시
**채널**: 관리자 텔레그램

```
[정산 실패] 긴급 확인이 필요합니다

대상: {partnerName} ({partnerCode})
주문번호: {orderNumber}
클래스: {className}
결제 금액: {totalAmount}원
예정 적립금: {reserveAmount}원

실패 사유: {errorMessage}
실패 시각: {failureTimestamp}
재시도 횟수: {retryCount}/5

조치 방법:
1. GAS 편집기에서 retryFailedSettlements() 수동 실행
2. 메이크샵 관리자에서 적립금 API 상태 확인
3. 해결 후 정산 시트 상태를 "완료"로 수동 변경

정산 시트: {settlementSheetUrl}
```

**설계 의도:**
- `[정산 실패]`로 긴급성 즉시 전달
- 실패 사유와 재시도 횟수로 상황 파악
- 3단계 조치 방법을 구체적으로 제시 (관리자가 즉시 대응 가능)
- 정산 시트 직접 링크 제공

### 4-5. 텔레그램 메시지 공통 규칙

| 규칙 | 설명 |
|------|------|
| 태그 형식 | `[유형]`으로 첫 줄 시작 |
| 개인정보 | 수강생 정보는 마스킹하지 않음 (관리자 채널이므로) |
| 시간 형식 | `2026-03-15 14:00 KST` |
| 금액 형식 | 천 단위 쉼표 + "원" |
| URL | 클릭 가능한 전체 URL 제공 |
| 긴급도 | 일반: `[파트너 신청]`, `[새 예약]`, `[정산 완료]` / 긴급: `[정산 실패]` |

**섹션별 헤더:**

| 섹션 | 헤더 | 부제 |
|------|------|------|
| 클래스 소개 | 이 클래스에서 만나는 경험 | -- |
| 커리큘럼 | 수업 진행 순서 | 각 단계를 클릭하면 상세 내용을 볼 수 있어요 |
| 강사 소개 | 함께할 선생님 | -- |
| 일정 선택 | 수업 일정 선택 | 원하시는 날짜와 시간을 선택해주세요 |
| 수강 후기 | 수강생 후기 | {reviewCount}개의 후기 |
| 재료 안내 | 이 수업에 필요한 재료 | -- |
| 관련 영상 | 미리 만나보는 수업 영상 | -- |

**남은 자리 상세 표시:**

```
// 여유 (5석 이상)

// 임박 (3석 이하)

```

**예약 CTA 버튼 변형:**
