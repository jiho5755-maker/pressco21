## 1. 수수료/전환율 구조 검토 및 개선안

### 1-1. 현재 구조 요약

| 등급 | 수수료율 | 적립금 전환율 | 파트너 현금 | 파트너 적립금 | PRESSCO21 현금 |
|------|---------|-------------|-----------|-------------|--------------|
| SILVER | 10% | 100% | 90% | 10% | **0%** |
| GOLD | 12% | 80% | 88% | 9.6% | **2.4%** |
| PLATINUM | 15% | 60% | 85% | 9% | **6%** |

GAS 코드(`class-platform-gas.gs` L57~60)에 확정 구현됨:
```javascript
var COMMISSION_RATES = {
  'SILVER':   { commissionRate: 0.10, reserveRate: 1.00 },
  'GOLD':     { commissionRate: 0.12, reserveRate: 0.80 },
  'PLATINUM': { commissionRate: 0.15, reserveRate: 0.60 }
};
```

### 1-2. 현재 구조 평가

#### 강점

1. **SILVER 100% 전환은 매우 강력한 모집 훅(Hook)**
   - "수수료 전액 환급"이라는 메시지는 타 플랫폼 대비 압도적 차별점
   - 프립(15~20%), 클래스101(30%), 탈잉(20%) 대비 사실상 수수료 0%
   - 초기 파트너 모집 시 가장 강력한 무기

2. **단순 3단계 구조**
   - 파트너가 이해하기 쉬움
   - GAS 코드 관리 용이

3. **선순환 의도가 명확**
   - 적립금으로 재료를 구매하게 함으로써 PRESSCO21 매출 증대

#### 약점 (개선 필요)

1. **SILVER에서 PRESSCO21 현금 수익 0원 -- 초기 생존 리스크**
   - Phase 2 초기(3개월)에는 대부분 SILVER 파트너
   - 20명 파트너 x 평균 월 10만원 매출 = 200만원 총매출
   - PRESSCO21 직접 현금 수익: 0원
   - 간접 수익(적립금 재료구매): 20만원 x 50% 전환율 = 약 10만원 마진
   - **GAS 운영비, 이메일 비용 등을 커버하기 어려울 수 있음**

2. **전환율 "역감소"의 비즈니스 로직 혼선**
   - SILVER(100%) -> GOLD(80%) -> PLATINUM(60%)
   - 파트너 입장: "승급하면 적립금이 줄어든다?"
   - 의도: "잘 되는 파트너에게서 PRESSCO21이 더 가져가는 구조"
   - **문제**: 파트너가 승급을 기피할 동기 발생
   - GOLD로 승급 시 적립금이 10% -> 9.6%로 줄고, 현금도 90% -> 88%로 줌

3. **등급 간 혜택 체감 부족**
   - SILVER에서 GOLD로 올라갈 때:
     - 현금: 90% -> 88% (감소)
     - 적립금: 10% -> 9.6% (감소)
     - 추가 혜택: 재료 5% 할인, 우선 노출
   - 파트너 관점에서 순수 금전적으로는 **승급이 손해**

### 1-3. 개선안 A (권장): "수수료 고정 + 전환율 역증가" 모델

**핵심 변경**: 수수료율을 전 등급 동일하게 고정하고, 전환율을 승급할수록 높이는 방향으로 전환

| 등급 | 수수료율 | 적립금 전환율 | 파트너 현금 | 파트너 적립금 | PRESSCO21 현금 |
|------|---------|-------------|-----------|-------------|--------------|
| SILVER | 15% | 50% | 85% | 7.5% | **7.5%** |
| GOLD | 15% | 70% | 85% | 10.5% | **4.5%** |
| PLATINUM | 15% | 90% | 85% | 13.5% | **1.5%** |

**장점**:
- 파트너 현금 수취가 전 등급 동일(85%)하여 수수료 부담 예측 용이
- 승급할수록 적립금이 증가하므로 파트너 승급 동기 명확
- PRESSCO21이 초기(SILVER 다수)부터 7.5% 현금 수익 확보
- Phase 2 시나리오: 200만원 매출 x 7.5% = 15만원/월 현금 수익 (vs 현재 0원)

**단점**:
- SILVER 파트너의 적립금이 줄어 초기 모집력이 약화될 수 있음
- "수수료 전액 환급" 마케팅 메시지를 사용할 수 없음

**수치 비교 (SILVER 파트너, 월 매출 27만원 기준)**:

| 항목 | 현재 모델 | 개선안 A |
|------|----------|---------|
| 수수료 | 27,000원 | 40,500원 |
| 파트너 현금 | 243,000원 | 229,500원 |
| 파트너 적립금 | 27,000원 | 20,250원 |
| PRESSCO21 현금 | 0원 | 20,250원 |

### 1-4. 개선안 B: "현행 유지 + 마이너 보정" 모델 (차선)

현재 구조의 핵심 강점("수수료 전액 환급")을 유지하면서 약점만 보정

| 등급 | 수수료율 | 적립금 전환율 | 파트너 현금 | 파트너 적립금 | PRESSCO21 현금 |
|------|---------|-------------|-----------|-------------|--------------|
| SILVER | 10% | 100% | 90% | 10% | **0%** |
| GOLD | 12% | 100% | 88% | 12% | **0%** |
| PLATINUM | 15% | 80% | 85% | 12% | **3%** |

**변경 핵심**:
- GOLD의 전환율을 80% -> 100%로 상향 (승급 동기 강화)
- GOLD 파트너 적립금: 9.6% -> 12%로 실질 증가 (승급 인센티브)
- PLATINUM만 PRESSCO21 현금 수익 3% 확보
- **마케팅**: "SILVER, GOLD 모두 수수료 전액 환급!"

**장점**:
- 현재 코드 변경 최소 (GOLD 전환율만 변경)
- "수수료 전액 환급" 마케팅이 SILVER+GOLD에 적용 가능
- 승급 시 적립금이 오히려 증가하므로 동기 부여

**단점**:
- PRESSCO21 현금 수익은 PLATINUM 파트너가 나올 때까지 0원
- 초기 6개월 이상 수익 없이 운영해야 할 수 있음

### 1-5. 개선안 C: "낮은 고정 수수료 + 고전환" 하이브리드 (장기 추천)

n8n+Airtable 전환 후 운영비가 안정화되면 적용할 수 있는 장기 모델

| 등급 | 수수료율 | 적립금 전환율 | 파트너 현금 | 파트너 적립금 | PRESSCO21 현금 |
|------|---------|-------------|-----------|-------------|--------------|
| SILVER | 12% | 80% | 88% | 9.6% | **2.4%** |
| GOLD | 12% | 90% | 88% | 10.8% | **1.2%** |
| PLATINUM | 12% | 100% | 88% | 12% | **0%** |

**핵심 메시지**: "PLATINUM이 되면 수수료 전액 환급!"
- 승급 인센티브 극대화: 높을수록 전환율 높음
- 전 등급 현금 수취 동일(88%)
- PRESSCO21 초기 수익 확보(2.4%)
- PLATINUM의 "수수료 전액 환급"을 목표로 하는 게이미피케이션

### 1-6. 최종 권고

**Phase 2 오픈 시점**: 개선안 B (현행 유지 + 마이너 보정) 적용 권장

이유:
1. 코드 변경이 최소(GOLD reserveRate만 0.80 -> 1.00 변경)
2. "수수료 전액 환급" 마케팅 훅 유지
3. 승급 동기 문제 해결(GOLD 적립금 12% > SILVER 10%)
4. PRESSCO21 수익은 적립금 재료 구매 간접 수익으로 충당
5. PLATINUM 출현 시점(6~12개월)에 3% 직접 수익 확보

**Phase 3 또는 파트너 30명 달성 시점**: 개선안 C로 전환 검토

## 2. 정산 타이밍 설계

### 2-1. 현재 구현 상태

GAS 코드 분석 결과, 현재 정산 타이밍은 다음과 같다:

```
주문 감지(pollOrders, 10분 간격)
  -> 즉시 정산 시도(processOrderInternal_)
    -> 적립금 API 호출(processReservePayment_)
      -> 성공 시 COMPLETED / 실패 시 FAILED
```

즉, **주문 감지 즉시 정산**이 현재 구현이다. partner-registration-guide.md의 FAQ에는 "수업 완료 후 D+3"으로 안내되어 있으나, 코드와 불일치가 있다.

### 2-2. 세 가지 정산 타이밍 비교

#### Option 1: 즉시 정산 (현재 구현)

```
결제 확인 -> 10분 내 적립금 지급
```

| 장점 | 단점 |
|------|------|
| 파트너 즉각 보상 -> 만족도 높음 | 환불/취소 시 적립금 회수 필요 |
| 구현 단순 (현재 코드 그대로) | 노쇼/미수강에도 적립금 선지급 |
| 정산 투명성 (결제 = 정산) | 적립금 회수 실패 리스크 |

**환불 시 처리**: 적립금 차감 API(`process_reserve` minus) 호출 필요
- 이미 사용한 적립금이 있으면 잔액 부족 가능
- 마이너스 잔액 허용 or 현금 청구 필요

#### Option 2: 수강일 +1~3일 정산 (추천)

```
결제 확인 -> 예약 기록 -> 수강일 +3일 -> 적립금 지급
```

| 장점 | 단점 |
|------|------|
| 환불 기간(D-3 무료취소) 경과 후 지급 | 파트너 적립금 수취까지 대기 시간 |
| 실제 수강 확인 후 정산 -> 분쟁 최소화 | GAS 트리거 추가 구현 필요 |
| 노쇼/취소 건 자동 제외 | 정산 상태 관리 복잡도 증가 |

**구현 방법**:
- `processOrderInternal_`에서 즉시 COMPLETED 대신 PENDING으로 기록
- 새로운 일일 트리거(`triggerSettlePendingOrders`)에서 class_date +3일 경과 건만 정산
- 환불/취소 건은 PENDING 상태에서 CANCELLED로 변경

#### Option 3: 월말 일괄 정산

```
결제 확인 -> 월 누적 -> 매월 말일 일괄 적립금 지급
```

| 장점 | 단점 |
|------|------|
| 운영 단순 (월 1회 처리) | 파트너 자금 회전 느림 |
| 환불/취소 모두 반영 후 정산 | 소규모 파트너 불만 가능 |
| 정합성 검증 용이 | 매월 대량 API 호출 집중 |

### 2-3. 정산 타이밍 최종 권고: Option 2 (수강일 +3일)

**이유**:
1. **partner-registration-guide.md FAQ와 일치**: "수업 완료 후 D+3 자동 적립금 지급"으로 이미 안내됨
2. **환불 분쟁 최소화**: D-3 무료취소 정책과 맞물려 환불 처리가 깔끔
3. **파트너 신뢰도**: "수업이 실제로 진행된 후 정산"이라는 원칙이 공정
4. **적립금 회수 리스크 제거**: 지급 전 취소/환불 반영

**필요 코드 변경** (현재 즉시 정산 -> D+3 정산):

1. `processOrderInternal_`에서 적립금 API 호출 부분을 분리
2. 정산 내역에 `class_date` 기반 `settlement_due_date = class_date + 3` 컬럼 추가
3. 새 트리거 `triggerSettlePendingOrders()` 추가 (일 1회 실행)
4. 해당 트리거에서 `settlement_due_date <= today AND status == 'PENDING'` 건만 정산

```
[현재]
pollOrders -> processOrderInternal_ -> processReservePayment_ -> COMPLETED

[권고]
pollOrders -> processOrderInternal_ -> status='PENDING_SETTLEMENT'
(매일 1회) triggerSettlePendingOrders -> due_date 확인 -> processReservePayment_ -> COMPLETED
```

### 2-4. 환불/취소 처리 방법

```
취소 요청 시나리오별 처리:

[Case 1] 수업 7일 전 ~ D-3 취소 (무료 취소)
  - 정산 상태: PENDING_SETTLEMENT -> CANCELLED
  - 적립금 영향: 없음 (아직 미지급)
  - 환불: 100%

[Case 2] D-3 ~ D-1 취소 (부분 환불)
  - 정산 상태: PENDING_SETTLEMENT -> PARTIAL_CANCELLED
  - 적립금: 위약금 기반 일부 정산
  - 환불: 80% (D-3) / 50% (D-1~D-2)

[Case 3] 수강 완료 후 취소 (정산 완료 건)
  - 정산 상태: COMPLETED -> REFUND_REQUIRED
  - 적립금: 차감 API 호출 (process_reserve, minus)
  - 잔액 부족 시: 다음 정산에서 차감 (마이너스 이월)
  - 환불: 관리자 개별 판단

[Case 4] 노쇼 (수강생 불참)
  - 정산 상태: PENDING_SETTLEMENT -> COMPLETED (정상 정산)
  - 이유: 파트너는 준비를 완료했으므로 정산 대상
  - 노쇼 수강생: 환불 불가 (취소 정책 명시)
```

## 3. 적립금 전환율 전략

### 3-1. 적립금 사용 유도 설계

적립금이 PRESSCO21 매출로 전환되려면, 파트너가 실제로 적립금을 사용해야 한다. 미사용 적립금은 단순 부채(발행한 포인트)에 불과하다.

**적립금 사용 촉진 메커니즘 3가지**:

#### A. 유효기간 설정 (권장: 6개월)

| 항목 | 설계 |
|------|------|
| 유효기간 | 적립일로부터 **6개월** |
| 만료 알림 | D-30, D-7 이메일 알림 |
| 만료 처리 | 자동 소멸 (Sheets에서 expired로 표기) |
| 유예 | 첫 1회에 한해 관리자 수동 연장 가능 (요청 시) |

**6개월 근거**:
- 공방 운영자의 재료 구매 주기: 보통 1~3개월
- 계절 클래스 전환 주기: 3~6개월
- 너무 짧으면(3개월) 파트너 불만, 너무 길면(12개월) 전환 촉진 효과 약화

**구현**: 정산 시트에 `reserve_expire_date` 컬럼 추가, 월 1회 만료 배치 실행

#### B. 사용처별 인센티브 차등

| 사용처 | 적립금 가치 | 설명 |
|--------|-----------|------|
| 재료 구매 (도매가) | **1원 = 1.2원** (20% 보너스) | 재료 구매 시 적립금 가치 1.2배 |
| 광고 부스트 | 1원 = 1원 | 기본 가치 |
| 신상품 샘플 | 1원 = 1원 | 기본 가치 |
| 기타 사용 | 1원 = 1원 | 기본 가치 |

재료 구매에 20% 보너스를 부여하면 적립금 27,000원이 재료 구매 시 32,400원 가치가 된다. 파트너 입장에서 재료 구매가 가장 유리한 선택지가 되어 자연스럽게 전환율이 높아진다.

### 3-3. PRESSCO21 관점 적립금 수익 시뮬레이션

```
Phase 2 목표 (월 50건, 평균 4만원, 200만원 매출)

[현재 모델 - 개선안 B 적용]
수수료 총액: 200만원 x 10%(대부분 SILVER) = 20만원
적립금 전환: 20만원 x 100% = 20만원 적립금 발행
적립금 재료 구매: 20만원 x 50% 전환율 = 10만원 재료 주문
재료 마진: 10만원 x 40%(재료 마진율) = 4만원 순이익

PRESSCO21 직접 현금 수익: 0원
PRESSCO21 간접 수익(재료 마진): ~4만원/월
PRESSCO21 총 수익: ~4만원/월

[참고: 개선안 A 적용 시]
수수료 총액: 200만원 x 15% = 30만원
PRESSCO21 직접 현금: 30만원 x 50%(현금 비율) = 15만원
적립금 발행: 15만원
적립금 재료 구매: 15만원 x 50% = 7.5만원
재료 마진: 7.5만원 x 40% = 3만원

PRESSCO21 총 수익: 15만원 + 3만원 = ~18만원/월
```

**핵심 시사점**: 개선안 B(현행 유지)는 초기 마케팅에 유리하지만 직접 수익이 0이다. 서버 비용, 인건비 등을 고려하면 파트너 20명 이상 확보 후 수수료 구조 전환이 필요할 수 있다.

## 5. 파트너 등급 업그레이드 기준 최적화

### 5-1. 현재 기준 분석

GAS 코드(`class-platform-gas.gs` L4172~4175)에 구현된 현재 기준:

```
SILVER: 기본 (가입 + 교육 이수)
GOLD: 수강 완료 10건 이상 AND 평균 평점 4.0 이상
PLATINUM: 수강 완료 50건 이상 AND 평균 평점 4.5 이상
```

Task 223 파일에는 다른 기준도 언급:
```
GOLD: 월 매출 3회 이상 또는 누적 수강생 20명
PLATINUM: 월 매출 5회 이상 또는 누적 수강생 50명
```

**문제 1: 기준 불일치**
- GAS 코드: "완료 건수 + 평점" (AND 조건)
- Task 223 초기 설계: "월 매출 횟수 또는 누적 수강생" (OR 조건)
- 실제 구현은 GAS 코드 기준이므로 이를 기준으로 분석

**문제 2: SILVER -> GOLD 구간이 너무 넓음**
- 10건 수강 완료는 월 2~3건 클래스 운영 시 3~5개월 소요
- 초기 파트너의 승급 체감이 너무 느림
- 중간 단계 없이 장기간 SILVER에 머무르면 동기 저하

**문제 3: GOLD -> PLATINUM 구간도 넓음 (10건 -> 50건)**
- 5배 차이는 상당한 격차
- GOLD에서 PLATINUM까지 추가 40건 필요 (월 5건 기준 8개월)

**문제 4: 평점 기준의 불공정성**
- 수강 완료 10건에 평점 4.0 이상은 합리적
- 하지만 수강 완료 50건에 평점 4.5 이상은 높은 기준
- 50건 중 몇 건의 저평점이 평균을 크게 낮출 수 있음
- 신규 파트너의 첫 1~2건 저평점이 장기간 영향

**문제 5: 강등 없음 정책**
- GAS 코드에서 `newOrder > currentOrder` 조건으로 강등 방지
- partner-registration-guide.md에는 강등 프로세스(경고->유예->강등) 정의
- 코드와 문서 불일치 (코드가 강등 미구현)

### 5-2. 개선 기준 제안

#### Phase 2 (즉시 적용 가능)

| 등급 | 현재 기준 | 개선 기준 | 변경 이유 |
|------|----------|----------|----------|
| SILVER | 기본 | 기본 (변경 없음) | - |
| GOLD | 완료 10건 + 평점 4.0 | 완료 **8건** + 평점 **3.8** | 승급 체감 빠르게, 진입 장벽 낮춤 |
| PLATINUM | 완료 50건 + 평점 4.5 | 완료 **30건** + 평점 **4.2** | 도달 가능성 높이고, 평점 기준 현실화 |

**수치 근거**:

```
GOLD 8건 기준:
- 월 2건 클래스: 4개월 (시즌 1~2 내 달성)
- 월 4건 클래스: 2개월 (적극적 파트너)
- 파트너의 50%가 6개월 내 GOLD 도달 목표

PLATINUM 30건 기준:
- GOLD 후 추가 22건 (8건 -> 30건)
- 월 4건: 약 5.5개월 추가
- 월 6건: 약 3.7개월 추가
- 오픈 1년 내 상위 10% 파트너가 PLATINUM 도달 목표

평점 기준 완화:
- 압화/공예 클래스 특성: 소수 인원(4~8명) 밀착 수업
- 한 명의 3점 리뷰가 평균에 큰 영향
- 4.2점은 "매우 만족"과 "만족" 사이로 현실적
```

#### Phase 3 (장기 추가 고려)

**중간 등급 추가**: SILVER -> GOLD 사이에 "BRONZE PLUS" 또는 "SILVER PLUS" 삽입

| 등급 | 기준 | 수수료 | 전환율 |
|------|------|--------|--------|
| SILVER | 기본 | 10% | 100% |
| SILVER+ | 완료 3건 + 평점 3.5 | 10% | 100% |
| GOLD | 완료 8건 + 평점 3.8 | 12% | 100% |
| PLATINUM | 완료 30건 + 평점 4.2 | 15% | 80% |

SILVER+는 수수료/전환율 변동 없이 "배지"와 소소한 혜택(노출 +5 가중)만 부여하여, 초기 파트너에게 성장 체감을 제공한다.

등급 외에 파트너 동기부여를 위한 추가 장치:

## 6. 리스크 및 방어 전략

### 6-1. 플랫폼 우회 리스크

**시나리오**: 파트너가 PRESSCO21을 통해 수강생을 모집한 후, 다음부터는 직접 예약을 받아 수수료를 회피

**발생 가능성**: 중간~높음 (특히 수수료 부담이 큰 GOLD/PLATINUM)

**대응 전략 (3단계)**:

#### 1단계: 예방 (구조적)

| 전략 | 설명 | 효과 |
|------|------|------|
| **적립금 선순환** | 적립금이 재료 도매가와 연결되어 플랫폼 이탈 시 손실 | 높음 |
| **반복 예약 할인** | 같은 파트너 2회 이상 수강 시 5% 할인 (플랫폼 통해서만) | 중간 |
| **후기 자산** | 수강 후기가 플랫폼에 축적되어 신규 수강생 유치에 기여 | 높음 |
| **노출 가중치** | 후기/예약 많을수록 상위 노출 -> 이탈 시 노출 감소 | 중간 |

#### 2단계: 감지

| 지표 | 이상 징후 | 모니터링 |
|------|----------|---------|
| 예약 급감 | 월 5건 -> 갑자기 0건 (but 클래스 활성 상태) | Sheets 월간 리포트 |
| 후기 감소 | 후기 요청 미응답률 급증 | GAS 이메일 트래킹 |
| 대시보드 미접속 | 30일+ 미접속 (but 파트너맵에서 활동 중) | 로그인 로그 |

#### 3단계: 대응

- **소프트 대응**: "요즘 예약이 없으시네요. 도움이 필요하시면 연락주세요" 이메일
- **대화 시도**: 관리자가 1:1 연락하여 불만 사항 청취
- **가치 재인식**: 적립금 보너스 이벤트, 특별 노출 기회 제공
- **계약 해지**: 극단적 경우에만 (비독점이므로 강제할 수 없음)

**핵심 원칙**: 우회를 "처벌"하기보다 **플랫폼에 남을 이유를 강화**하는 방향. 적립금 도매가 혜택, 후기 자산, 노출 알고리즘이 핵심 방어선이다.

### 6-2. 클래스 품질 관리

**리스크**: 저품질 클래스가 전체 플랫폼 신뢰도를 하락시킴

**방어 전략**:

| 단계 | 시점 | 방법 |
|------|------|------|
| **사전 검수** | 등록 시 | 관리자 검수 (사진, 소개글, 커리큘럼) |
| **파트너 교육** | 가입 시 | 필수 3모듈 이수 (아카데미) |
| **초기 모니터링** | 첫 3건 | 첫 3건 수강 후기 집중 모니터링 |
| **자동 경고** | 운영 중 | 평점 3.5 미만 3건 연속 시 관리자 알림 |
| **개선 프로그램** | 경고 후 | 1:1 피드백 + 교육 콘텐츠 제공 |
| **일시 중단** | 심각 시 | 평점 3.0 미만 5건 연속 시 노출 중단 + 개선 요청 |

**자동화 구현** (GAS 추가 함수):

```
triggerQualityCheck() (주 1회)
  - 최근 30일 후기에서 평점 3.5 미만 파트너 감지
  - 3건 연속 3.5 미만 시 관리자에게 알림 이메일
  - 해당 파트너에게 "수강생 피드백 요약" 이메일 발송
```

### 6-3. 환불 시 파트너 수수료 회수

**D+3 정산 적용 시**:

| 환불 시점 | 적립금 상태 | 처리 |
|----------|-----------|------|
| 수강 전 (D+3 이전) | 미지급 | CANCELLED로 변경, 회수 불필요 |
| 수강 후 D+3 이내 | 미지급 | CANCELLED로 변경, 회수 불필요 |
| 수강 후 D+3 이후 | 지급 완료 | 적립금 차감 API 호출 |

**적립금 차감 실패 시**:

```
차감 시도 -> 실패 (잔액 부족)
  -> 정산 시트에 "REFUND_PENDING" 기록
  -> 파트너에게 "적립금 잔액 부족, 다음 정산에서 차감 예정" 이메일
  -> 다음 정산 시 미회수 금액 자동 차감
  -> 3회 연속 차감 불가 시 관리자 알림
```

### 6-4. 기타 리스크

| 리스크 | 발생 가능성 | 영향 | 대응 |
|--------|-----------|------|------|
