# 상세 UX 고도화 가이드

작성일: 2026-03-10

S1-2의 목적은 상세 페이지를 단순 정보 화면이 아니라 `신뢰를 만들고 예약으로 밀어주는 화면`으로 바꾸는 것이다.

## 이번 단계에서 추가된 핵심

### 1. Trust Summary Bar

- 상단 고정 요약 바에 아래 3개 수치를 노출한다.
  - `수강 N명`
  - `평점 ★X.X`
  - `후기 N건`
- 스크롤 중에도 핵심 신뢰 지표를 계속 보이게 해 예약 전 불안을 줄인다.

### 2. "이 가격에 포함된 것" 섹션

- 상세 상단에 `강의 / 재료키트 / 수료증` 3개 카드를 추가했다.
- 각 카드는 단순 제목이 아니라 상태와 설명을 같이 보여준다.
  - 예: `포함`, `선택 구매`, `클래스별 상이`
- 현재 데이터가 부족한 수료증은 거짓으로 `제공`이라고 단정하지 않고, 클래스별 상이로 처리한다.

### 3. CTA 계층

- 데스크탑 예약 패널:
  - 주버튼: `예약하기`
  - 보조버튼: `선물하기`
  - 링크: `키트만 구매`
- 모바일 하단 고정 바:
  - 주버튼: `예약하기`
  - 보조버튼: `선물하기`
  - 링크: `키트만 구매`
- 키트 링크는 별도 상품 상세로 보내는 대신, 현재 단계에서는 재료 섹션으로 스크롤해 개별 구매 또는 한번에 담기를 유도한다.

## 구현 메모

### 신뢰 요약 바

- `detailTrustBar`, `detailTrustBarStats`를 추가했다.
- `booked_count`, `avg_rating`, `class_count`를 읽어 상단 요약 바를 렌더링한다.

### 포함 내역

- `detailIncludes`, `detailIncludesGrid`를 추가했다.
- 상태값 판단 기준:
  - `강의`: 항상 포함
  - `재료키트`: `materials_included=포함`이면 포함, 키트 링크가 있으면 선택 구매, 없으면 별도 안내
  - `수료증`: 명시 데이터가 없으면 `클래스별 상이`

### 모바일 CTA

- 모바일 고정 바를 2줄 구조로 변경했다.
  - 윗줄: 총 결제 예상 + 예약하기
  - 아랫줄: 선물하기 + 키트만 구매
- 수량 변경 시 모바일 금액도 함께 바뀐다.
- 날짜/시간 선택 여부에 따라 데스크탑과 모바일 버튼 상태를 같이 동기화한다.

## 남은 주의사항

- 현재는 저장소 기준 구현만 완료된 상태다.
- 실제 라이브 확인이 필요하면 메이크샵 디자인편집기 배포 후 다시 Playwright로 검증해야 한다.
- 카카오 SDK `integrity` 불일치는 이번 태스크 범위 밖이며, 로컬 미리보기 콘솔에만 계속 남는다.
