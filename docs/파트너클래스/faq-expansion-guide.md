# S1-6 FAQ 확장 가이드

작성일: 2026-03-11

## 목적

상세 페이지 FAQ 탭을 `5개 고정 질문`에서 `15개 탐색형 FAQ`로 확장했다. 수강생이 예약 전에 가장 많이 확인하는 가격, 일정, 키트 배송, 환불, 파트너 신뢰 질문을 한 탭 안에서 바로 찾게 만드는 것이 목적이다.

## 반영 범위

- `파트너클래스/상세/Index.html`
- `파트너클래스/상세/css.css`
- `파트너클래스/상세/js.js`

## UI 구조

- 상단 설명 문구
- 검색 입력
- 카테고리 칩
  - `전체 / 수강 / 키트·배송 / 파트너 / 정산 / 기타`
- 결과 요약 문구
- FAQ 아코디언 목록
- 검색 결과 없음 상태
- 문의 연락처 블록

## 데이터 우선순위

1. `WF-01 class-api`가 `faq_items`, `faqItems` 계열 필드를 주면 먼저 사용
2. 값이 없거나 비정상이면 상세 페이지 기본 FAQ 15개를 fallback
3. 질문 중복은 질문 텍스트 기준으로 제거
4. 최종 노출 개수는 최대 15개

## 기본 FAQ 구성

- 수강 4개
- 키트·배송 5개
- 파트너 2개
- 정산 3개
- 기타 1개

질문/답변은 현재 클래스 데이터(`난이도`, `가격`, `장소`, `정원`, `재료키트`, `파트너명`)를 일부 반영해 클래스 맥락에 맞게 문구가 바뀐다.

## 동작 규칙

- 카테고리 칩은 전체 개수 기준 카운트를 표시
- 검색은 질문/답변/카테고리 텍스트를 실시간으로 함께 필터링
- FAQ 아코디언은 한 번에 1개만 열리도록 처리
- 결과가 없으면 빈 상태 문구와 문의 유도 문구를 노출

## 구조화 데이터

- `FAQPage` JSON-LD는 기존 커리큘럼 변환 방식 대신 실제 FAQ 데이터 기준으로 생성
- FAQ가 바뀌면 SEO용 구조화 데이터도 같은 내용으로 따라간다

## 검증

- `node --check 파트너클래스/상세/js.js`
- `python3 ~/.codex/skills/makeshop-d4-dev/scripts/check_makeshop_d4.py ...`
  - `http://` 경고는 기존 SVG namespace 기준 false positive
- Playwright 로컬 목업 검증
  - FAQ 15개 렌더링
  - 카테고리 `정산` 필터 3개 확인
  - 검색어 `환불` 결과 1개 확인
  - `정산 + 배송` 조합 빈 상태 확인
