# PRESSCO21 공용자료 파일명 규칙

작성일: 2026-03-19

이 문서는 `company-hub` 아래에 넣는 회사 공용자료의 파일명을 어떻게 정할지 정리한다.

핵심 원칙은 네 가지다.

1. 파일명만 보고도 무엇인지 알 수 있게 한다.
2. 날짜가 필요한 문서는 앞에 날짜를 붙인다.
3. 최종본과 수정본을 구분할 수 있게 버전을 붙인다.
4. 웹에 공개할 파일만 영어 파일명을 쓴다.

## 1. 가장 기본 규칙

기본 형식:

- `YYYYMMDD_문서이름_v01`
- `YYYYMM_문서이름_v01`
- `문서이름_v01`

예시:

- `20260319_회사소개서_v01.pdf`
- `202603_월간매출정리_v02.xlsx`
- `브랜드가이드_v03.pdf`
- `서비스소개원페이저_v01.pptx`

## 2. 권장 파일명 구성 순서

가능하면 아래 순서를 지킨다.

1. 날짜
2. 문서명
3. 필요하면 대상/클라이언트명
4. 버전
5. 확장자

예시:

- `20260319_클레어스_제안서_v01.pptx`
- `20260319_주간회의록_v01.docx`
- `202603_운영체크리스트_v03.xlsx`

## 3. 버전 표기 규칙

버전은 아래처럼 통일한다.

- 초안: `v01`
- 1차 수정: `v02`
- 2차 수정: `v03`

최종 확정본은 두 방식 중 하나만 쓴다.

- `v05_최종`
- `v05_final`

권장:

- 내부 공용자료는 `최종`
- 외부 전달/웹 공개 자료는 `final`

예시:

- `20260319_회사소개서_v05_최종.pdf`
- `pressco21-brand-guide-v03-final.pdf`

## 4. 날짜를 붙여야 하는 자료

날짜를 붙이는 것이 좋은 자료:

- 회의록
- 제안서
- 계약 관련 문서
- 월간/주간 보고서
- 체크리스트
- 재무/행정 문서

예시:

- `20260319_주간회의록_v01.docx`
- `20260319_고객제안서_클레어스_v02.pptx`
- `202603_세무정리_v01.xlsx`

## 5. 날짜를 꼭 안 붙여도 되는 자료

아래 자료는 날짜 없이 관리해도 된다.

- 로고 원본
- 브랜드 가이드 최신본
- 템플릿
- 고정 자산 문서

예시:

- `브랜드가이드_v03.pdf`
- `회사소개서_기본템플릿_v02.pptx`
- `로고_가로형_국문.svg`

## 7. 금지하거나 피해야 하는 파일명

아래 방식은 피한다.

- `최종최종진짜최종.pdf`
- `수정본(2).pptx`
- `새폴더용.docx`
- `무제.pdf`
- `IMG_001.pdf`

이유:

- 검색이 어렵다.
- 누가 봐도 내용이 안 보인다.
- 버전 추적이 어렵다.

## 8. 파일명에 쓰지 않는 것이 좋은 문자

가급적 아래 문자는 피한다.

- `/`
- `\\`
- `:`
- `*`
- `?`
- `\"`
- `<`
- `>`
- `|`

그리고 웹 공개용 파일은 아래도 피한다.

- 공백
- 한글
- 괄호

## 9. 웹 공개용 파일명 규칙

`design/publish` 나 향후 MinIO 이미지호스팅으로 나갈 파일은 영어만 쓴다.

형식:

- `brand-name_asset-purpose_yyyymmdd_v01.ext`
- 또는 `brand-name-asset-purpose-v01.ext`

예시:

- `pressco21-logo-primary-v01.svg`
- `pressco21-thumbnail-spring-campaign-20260319-v01.jpg`
- `foreverlove-product-detail-rose-oil-v02.jpg`

이유:

- URL 이 안전하다.
- MinIO object key 로 쓰기 좋다.
- 메이크샵/웹 연결 시 덜 꼬인다.

## 10. 아주 쉽게 기억하는 요약

회사 공용자료는 이렇게 생각하면 된다.

- 내부 문서: `날짜 + 문서명 + 버전`
- 브랜드 자산: `자산이름 + 용도 + 버전`
- 템플릿: `템플릿명 + 버전`
- 세일즈 자료: `날짜 + 고객명/자료명 + 버전`
- 웹 공개용: `영문 파일명`

## 11. 추천 운영 습관

1. 새 파일은 올릴 때부터 파일명을 바로 정한다.
2. 애매하면 `v01` 부터 시작한다.
3. 수정본을 덮어쓰기보다 `v02`, `v03` 으로 올린다.
4. 최종 확정 후에만 `_최종` 또는 `_final` 을 붙인다.
5. 웹 공개용은 반드시 영어 파일명으로 바꾼 뒤 `publish` 에 넣는다.
