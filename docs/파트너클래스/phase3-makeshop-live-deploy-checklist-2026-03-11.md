## 2. 신규 page id 확정 대상

| 상태 | 페이지 | 소스 폴더 | 메모 |
|---|---|---|---|
| 신규 생성 필요 | 콘텐츠 허브 | `파트너클래스/콘텐츠허브/` | 아직 실배포 page id 없음 |
| 신규 생성 필요 | 협회 제안서 | `파트너클래스/협회제안서/` | 어드민 URL 생성기에 넣을 실배포 page id 필요 |
| 별도 메인 편집기 반영 | 메인 허브 블록 | `메인페이지/파트너클래스-홈개편/` | 개별 페이지가 아니라 메인 디자인 반영 작업 |

## 3. 권장 배포 순서

1. `2606`, `2607`, `2608`, `2609`를 먼저 저장한다.
2. `8009`, `8010`, `8011`, `2610`을 이어서 저장한다.
3. 신규 자산이 필요하면 `콘텐츠허브`, `협회제안서` page id를 새로 만들고 기록한다.
4. 메인페이지 허브 블록은 별도 승인 후 메인 디자인에 반영한다.
5. 저장 직후 public 페이지 캐시를 비우거나 강력 새로고침 후 라이브 스모크를 돌린다.

## 4. 저장 직후 검증

### 4-1. 공개/비로그인 기준

```bash
NODE_PATH=/Users/jangjiho/workspace/codex/node_modules \
node /Users/jangjiho/workspace/pressco21/scripts/partnerclass-live-smoke.js
```

- 파트너 계정 환경변수가 없으면 게스트 + 관리자 읽기 전용 시나리오만 실행된다.

node /Users/jangjiho/workspace/pressco21/scripts/partnerclass-live-smoke.js

## 6. 현재 blocker

- 실관리자 계정 자격증명은 저장소에 없다. `8011` 양성 브라우저 검증은 별도 계정이 있어야 한다.
- 파트너 로그인 계정도 저장소에 없다. 로그인 시나리오 전체 PASS 판정은 계정 제공 후 가능하다.
- `콘텐츠허브`, `협회제안서`는 실배포 page id가 아직 정해지지 않았다.
- 현재 세션의 Playwright MCP가 불안정하면 `docs/playwright-mcp-setup.md` 절차대로 새 세션에서 재시도한다.
