# Phase 3 비전공자 테스트 가이드

작성일: 2026-03-10

## 이 문서는 왜 필요한가

Phase 3는 기능 이름이 많고 기술 용어도 많아서, 비전공자 입장에서는 "결국 내가 뭘 확인해야 하는지"가 흐려지기 쉽다.

이 문서는 그 문제를 줄이기 위한 가이드다.

- 개발 언어 대신 화면 기준으로 설명한다.
- "왜 바뀌는지"보다 "사용자가 무엇을 느껴야 하는지"를 먼저 본다.
- 테스트할 때 합격인지 불합격인지 빠르게 판단할 수 있게 정리한다.

## 한 줄로 이해하기

파트너클래스는 단순히 클래스를 모아두는 페이지가 아니라 아래 3가지를 한 번에 하는 서비스가 되어야 한다.

1. 수강생에게는 믿고 예약할 수 있는 클래스 허브
2. 파트너에게는 강의 외 운영을 덜어주는 지원 서비스
3. 협회에게는 회원 혜택과 공동 홍보를 제공하는 제휴 플랫폼

즉, 밖에서 보이는 화면은 "예약이 잘 되는 서비스"여야 하고,
안에서 돌아가는 구조는 "운영이 편한 서비스"여야 하며,
사업적으로는 "자사몰과 브랜드를 더 오래 쓰게 만드는 구조"여야 한다.

## 내가 테스트할 때 가장 먼저 봐야 할 3가지

### 1. 수강생 관점

- 이 클래스가 믿을 만한지 빠르게 판단되는가
- 돈 값을 할지 불안이 줄어드는가
- 예약, 후기, 재구매까지 흐름이 자연스러운가

### 2. 파트너 관점

- 강의만 잘하면 되겠다는 느낌이 드는가
- 재료 준비, 정산, 일정 관리, 홍보 부담이 줄어든다고 느껴지는가
- 대시보드에 매일 들어올 이유가 보이는가

### 3. 협회 관점

- 우리와 제휴할 이유가 분명한가
- 협회 일정과 세미나를 홍보할 수 있어 보이는가
- 협회원 전용 혜택과 전용 상품 운영이 자연스럽게 연결되는가

## Phase 3 전체를 쉬운 말로 풀면

### Phase 3-0: 바닥 공사

겉으로 크게 티가 안 나도 가장 먼저 필요한 단계다.

이 단계의 목적은 "뒤에서 터지지 않게 만드는 것"이다.

- 데이터가 뒤죽박죽이면 화면도 이상하게 보인다.
- 워크플로우 구조가 복잡하면 작은 수정도 위험해진다.
- 백업이 없으면 운영 중 사고가 나도 복구가 어렵다.

#### 여기서 내가 확인할 포인트

- 상태값과 용어가 화면마다 다르게 보이지 않는가
- 관리자 화면이 갑자기 죽지 않는가
- 문제 생겼을 때 복구할 준비가 되어 있는가

#### 현재 완료된 것

- `S0-1 NocoDB 일일 자동 백업`은 완료됐다.

이건 화면 기능이라기보다 운영 안전장치다.

비전공자 기준으로는 아래처럼 이해하면 된다.

- 매일 자동으로 백업이 남는가
- 오래된 백업은 자동으로 정리되는가
- 백업이 실패하면 관리자에게 알림이 가는가

즉, "망가져도 다시 살릴 준비가 돼 있는가"를 확인한 것이다.

### Phase 3-1: 수익 엔진 점화

이 단계부터는 수강생 화면과 파트너 운영 화면이 눈에 띄게 좋아져야 한다.

핵심은 4가지다.

1. 수강생이 더 믿고 예약하게 만들기
2. 키트와 재료 판매를 자연스럽게 연결하기
3. 파트너가 운영 부담을 덜 느끼게 만들기
4. 정산과 CS를 덜 피곤하게 만들기

### Phase 3-2: 성장 가속

이 단계는 "좋은 서비스"를 넘어서 "파트너와 협회를 설득할 수 있는 서비스"로 가는 구간이다.

핵심은 4가지다.

1. 파트너 신청 페이지를 세일즈 페이지로 바꾸기
2. 협회 제휴 가치를 제품 안에서 보여주기
3. 클래스, 협회, 혜택을 같은 허브 안에서 더 똑똑하게 나누기
4. 콘텐츠와 자동화로 리텐션을 만들기

### Phase 3-3: 스케일업

이 단계는 구조를 크게 키우는 구간이다.

핵심은 4가지다.

1. 협회 전용 데이터 구조 만들기
2. 파트너 등급별 차별화 체감 만들기
3. 키트 구독처럼 반복 매출 구조 시험하기
4. 파트너가 많아져도 서버와 운영이 버티는지 확인하기

## 앞으로의 테스트는 이렇게 보면 된다

### A. 수강생용 테스트

수강생 화면은 결국 "불안이 줄어드는가"가 핵심이다.

#### 1. 상세 페이지 신뢰감

봐야 할 것:

- 수강 인원, 평점, 후기 수 같은 신뢰 정보가 바로 보이는가
- "이 가격에 뭐가 포함되는지"가 명확한가
- 예약하기, 선물하기, 키트만 구매하기가 헷갈리지 않는가

합격 기준:

- 처음 들어가도 대충 어떤 클래스인지 바로 이해된다
- 비싸 보이는지보다 먼저 믿을 수 있는지가 보인다
- 모바일에서도 예약 버튼을 놓치지 않는다

불합격 신호:

- 정보가 너무 늦게 나와서 끝까지 읽어야 안다
- 포함 항목이 모호해서 추가 비용이 불안하다
- 버튼이 많아 뭘 눌러야 할지 헷갈린다

#### 2. 목록 페이지 탐색감

봐야 할 것:

- 인기, 신규, 마감임박, 키트포함 같은 배지가 이해되게 붙는가
- 지역, 난이도, 가격대 필터가 자연스럽게 작동하는가
- 내가 고른 필터가 새로고침 후에도 유지되는가

합격 기준:

- 한눈에 비교가 된다
- 내가 찾는 클래스 범위를 빨리 좁힐 수 있다
- 목록이 "상품 진열"이 아니라 "믿을 만한 탐색"처럼 느껴진다

불합격 신호:

- 배지가 많아도 왜 붙었는지 모르겠다
- 필터가 동작해도 결과가 이상하다
- 용어가 페이지마다 달라 헷갈린다

#### 3. 수강 후 재구매 흐름

봐야 할 것:

- 수강 완료 뒤 후기 작성 유도가 자연스러운가
- 같은 강사의 다른 클래스 추천이 이어지는가
- 키트 재구매 링크가 어색하지 않은가

합격 기준:

- 수강 후 관계가 끊기지 않는다
- 다시 사거나 다시 예약할 이유가 보인다

불합격 신호:

- 수강이 끝나면 아무 것도 이어지지 않는다
- 재구매가 억지 판매처럼 느껴진다

### B. 파트너용 테스트

파트너 화면은 결국 "운영 피로가 줄어드는가"가 핵심이다.

#### 1. 파트너 신청 페이지

봐야 할 것:

- 단순 신청 폼이 아니라 입점 제안 페이지처럼 보이는가
- "강의만 잘하시면 됩니다. 나머지는 저희가 합니다."가 실제 구조로 느껴지는가
- 직접 운영과 PRESSCO21 이용의 차이가 명확한가

합격 기준:

- 왜 제휴해야 하는지 읽자마자 이해된다
- 신청하기 전에 불안이 줄어든다
- 협회나 강사가 다른 사람에게도 소개할 만하다

불합격 신호:

- 결국 로그인하라는 페이지처럼만 느껴진다
- 서비스 장점이 감성 카피만 있고 실체가 없다

#### 2. 온보딩과 대시보드

봐야 할 것:

- 첫 로그인 시 무엇부터 해야 하는지 바로 보이는가
- 오늘 수업, 준비할 키트, 미답변 후기처럼 오늘 할 일이 한눈에 보이는가
- 모든 기능을 다 보지 않아도 당장 필요한 액션으로 갈 수 있는가

합격 기준:

- 파트너가 매일 들어올 이유가 생긴다
- 대시보드가 메뉴판이 아니라 작업판처럼 느껴진다
- 초보 파트너도 막막하지 않다

불합격 신호:

- 기능은 많아도 우선순위가 안 보인다
- 무엇을 먼저 해야 할지 감이 안 온다

#### 3. 키트/정산 흐름

봐야 할 것:

- 파트너가 자사몰 상품을 키트로 쉽게 연결할 수 있는가
- 수강생은 키트를 한 번에 담을 수 있는가
- 파트너는 정산 내역을 이해하기 쉬운가

합격 기준:

- 운영자가 수작업으로 설명할 일이 줄어든다
- 정산이 "내가 대충 계산해야 하는 것"이 아니라 "보여주는 것"이 된다

불합격 신호:

- 링크는 있어도 실제 주문 흐름이 끊긴다
- 정산 정보가 숫자만 나와서 믿기 어렵다

### C. 협회용 테스트

협회 화면과 제안 구조는 결국 "우리와 제휴할 이유가 보이는가"가 핵심이다.

#### 1. 협회/세미나 레이어

봐야 할 것:

- 협회 일정, 세미나, 이벤트가 클래스와 구분되어 보이는가
- 그렇다고 완전히 다른 사이트처럼 찢어져 보이지는 않는가
- 협회 콘텐츠가 브랜드 신뢰를 높여주는가

합격 기준:

- 같은 허브 안에서 자연스럽게 구분된다
- 수강생은 커뮤니티 허브처럼 느끼고, 협회는 제휴 채널처럼 느낀다

불합격 신호:

- 클래스인지 협회 공지인지 구분이 안 된다
- 아예 별도 서비스처럼 느껴져 연결감이 없다

#### 2. 협회원 혜택과 전용 상품

봐야 할 것:

- 협회원 전용 제품과 시그니처 상품이 자연스럽게 노출되는가
- "협회원만 받을 수 있는 혜택"이 분명한가
- 자사몰 가입과 구매 유도가 거부감 없이 이어지는가

합격 기준:

- 협회원이면 더 자주 들어오고 싶어진다
- 협회도 회원 혜택을 설명하기 쉬워진다

불합격 신호:

- 그냥 광고 배너처럼 보인다
- 협회와의 관계보다 판매 의도만 강하게 느껴진다

## 단계별로 쉽게 이해하는 체크포인트

### 지금 바로 보이는 체크포인트

- 백업 같은 운영 안전장치가 마련되고 있는가
- 용어와 상태값이 뒤섞이지 않게 정리되고 있는가
- 목록, 상세, 신청, 대시보드가 같은 서비스처럼 느껴지는가

### 다음에 보이기 시작할 체크포인트

- 상세 페이지가 더 믿음직스러워지는가
- 키트 판매가 어색한 추가 판매가 아니라 자연스러운 연장선으로 보이는가
- 파트너 대시보드가 실제 운영 도구처럼 변하는가

### 그 다음에 봐야 할 체크포인트

- 협회 제휴가 말만 있는 것이 아니라 제품 안에서 드러나는가
- 협회원 혜택 구조가 자사몰 락인으로 이어지는가
- 콘텐츠와 이벤트가 반복 방문 이유를 만들어주는가

## 테스트할 때 자주 던지면 좋은 질문

### 공통 질문

- 이 화면은 누구를 위한 화면인가
- 들어오자마자 주 목적이 명확한가
- 내가 다음 행동을 쉽게 고를 수 있는가

### 수강생 화면 질문

- 이 클래스가 믿을 만한지 빨리 판단되는가
- 가격보다 먼저 신뢰가 보이는가
- 예약 후에도 다시 돌아올 이유가 생기는가

### 파트너 화면 질문

- 강의 외 잡일이 줄어들 것 같은가
- 오늘 해야 할 일이 잘 보이는가
- 내가 이 서비스에 계속 머물 이유가 있는가

### 협회 화면 질문

- 협회가 우리와 손잡을 이유가 충분한가
- 회원에게 줄 수 있는 혜택이 분명한가
- 자사몰과 협회가 자연스럽게 연결되는가

## 이 문서로 테스트 결과를 정리하는 법

테스트 후에는 기술 용어보다 아래 형식으로 적는 것이 좋다.

1. 어디를 봤는가
2. 무엇이 좋아졌는가
3. 아직 왜 불안한가
4. 합격인지 보류인지

예시:

- 상세 페이지 상단은 전보다 믿음이 생겼다.
- 다만 가격 포함 항목이 아직 모호해서 예약 직전 불안은 남아 있다.
- 현재 평가는 "부분 합격"이다.

## 가장 중요한 최종 기준

Phase 3가 잘 되고 있는지는 결국 아래 3문장으로 판단하면 된다.

1. 수강생이 "여기서 예약해도 되겠다"라고 느끼는가
2. 파트너가 "여기 있으면 운영이 편해지겠다"라고 느끼는가
3. 협회가 "우리 회원에게 이걸 붙이면 이득이 있겠다"라고 느끼는가

이 3개가 동시에 맞아가면 방향이 맞다.
