# Telegram 토픽 분리 운영 가이드

## 적용 기준

- 그룹: `PRESSCO21 매출 공유`
- Supergroup chat ID: `-1003917399249`
- 매출보고 topic ID: `3`
- 운영알림 topic ID: `4`

## 텔레그램 UI 제약

- `All`은 텔레그램 클라이언트의 전체 토픽 보기라서 봇이나 n8n 설정으로 제거할 수 없다.
- `General`은 포럼 그룹의 기본 토픽이다. 일반 토픽처럼 완전 삭제되지는 않으며, 앱 버전에 따라 숨김/닫기/이름 변경만 가능하다.
- 자동화는 `General`에 발송하지 않고, 아래 두 토픽으로만 보낸다.

## 자동화 발송 규칙

- F24 일일 매출 리치 리포트: `매출보고` topic ID `3`
- F23 운영 알림: `운영알림` topic ID `4`
- F23B 쿠팡 pending backfill 알림: `운영알림` topic ID `4`
- F22 Telegram Router: 그룹/포럼 서비스 메시지는 무응답 처리해서 `General` 오염을 막는다.

## 운영자가 텔레그램에서 할 일

1. 좌측 토픽 목록에서 `General`을 길게 누른다.
2. 앱에서 제공하는 경우 `Hide` 또는 `Close`를 선택한다.
3. `General` 완전 삭제는 불가하므로, 숨김이 불가능한 앱에서는 이름을 `기타/사용안함`처럼 바꾸고 사용하지 않는다.
4. 팀원에게 `매출보고`와 `운영알림` 두 토픽만 확인하도록 안내한다.
5. `All`은 전체 보기이므로 필요할 때만 사용한다.

## 재설정 시 주의

토픽을 삭제 후 다시 만들면 topic ID가 바뀐다. 그 경우 n8n 워크플로우의 `message_thread_id`를 새 값으로 다시 반영해야 한다.
