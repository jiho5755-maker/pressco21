# PRESSCO21 Offline CRM v2 1차 개발 완료 기록

- 작성일: 2026-04-15
- 상태: 1차 개발 완료, 운영 사용 단계
- 범위: `offline-crm-v2`
- 운영 URL: `https://crm.pressco21.com`
- 배포: `bash deploy/deploy.sh`
- 백엔드: NocoDB, n8n `/crm-proxy` 경유

## 완료 선언

Offline CRM v2는 1차 개발을 완료하고 실사용 운영 단계로 전환한다.

당분간 큰 구조 변경보다는 실제 업무에서 발견되는 불편, 누락, 데이터 보정 필요 사항을 작게 정리해 반영한다. 운영 데이터는 실제 장부로 취급하며, 금액/미수/지급/정산 로직은 변경 전에 사례와 기대 결과를 먼저 확인한다.

## 1차 완료 범위

- 고객/거래처 관리: 목록, 검색, 상세, 기본 정보 수정, 등급/메모 관리
- 거래명세표: 발행, 수정, 복사, 삭제, 인쇄, 고객 선택, 품목 자동완성, 금액 역산
- 출력 문서: 거래명세표, 견적서, 납품서, 청구서 출력 흐름 분리
- 고객 상세: 기본정보, 명세표, 거래내역, 미수금 탭
- 거래내역: 대량 기존 거래 데이터 조회, 검색, 필터, 페이지네이션
- 제품/공급처 관리: 제품 CRUD, 공급처 CRUD, 단가/과세/거래처 연계 관리
- 수금/미수 관리: 미수 목록, 지급/예치금 흐름, 자동입금 검토, 초과 입금 보정
- 캘린더/리포트: 월간 캘린더, 기간 매출 리포트, KPI, 차트
- 설정: 공급자 정보, 인쇄 설정, 로고/도장, 입금계좌, 시스템 설정
- 인증/연동: 세션 기반 CRM 인증, 자동화 헤더, n8n 프록시 연동
- 내부 관리 메모: 거래명세표 출력 비고와 분리된 CRM 내부 메모
- 납부 예정 관리: 미수 거래별 납부 예정일/예정 금액 기록
- 운영실 알림: 납부 예정일 당일 1회 Telegram `PRESSCO21 운영실` 알림

## 운영 기준

- 거래명세표의 `비고`는 외부 출력 문서에 표시되는 문구로 유지한다.
- 내부 확인용 내용은 `내부 관리 메모`에 기록한다.
- 납부 예정일 알림은 한 번만 발송한다. 반복 알림이나 재촉성 알림은 기본값으로 두지 않는다.
- 미수 관리 화면과 캘린더는 납부 예정 정보를 빠르게 확인하는 운영 보조 화면으로 사용한다.
- 금액 계산, 잔액 재계산, 자동입금 반영 로직은 작은 UI 수정과 다르게 취급한다. 운영 사례를 확인한 뒤 수정한다.
- n8n workflow, 인증 헤더, 환경 변수, 텔레그램 알림 경로가 바뀌면 문서와 `AI_SYNC.md`에 함께 기록한다.
- 배포 전 기본 확인은 `npm run build`를 기준으로 한다. E2E는 관련 기능을 건드렸을 때 범위를 좁혀 먼저 실행하고, 필요 시 전체 실행한다.

## 알려진 관리 포인트

- `npm run lint`는 프로젝트 전역의 기존 lint debt가 남아 있어 1차 완료 기준에서 제외했다. 기능 변경 검증은 현재 `npm run build`와 관련 Playwright 테스트를 우선 기준으로 삼는다.
- 캘린더 E2E 일부 케이스는 오늘 운영 데이터 건수에 의존하는 테스트 기대값 이슈가 있었다. 운영 기능 문제와 분리해서 테스트 안정화 항목으로 관리한다.
- 운영 데이터가 계속 누적되므로, 테스트 데이터는 `TEST-` 계열 prefix를 사용하고 기존 운영 데이터는 읽기 전용 원칙을 유지한다.
- 내부 메모/납부 예정 정보는 출력 문서에 노출되지 않아야 한다. 이 분리는 이후 수정에서도 유지한다.

## 앞으로의 변경 방식

1. 실사용 중 발견한 요청은 작은 작업 단위로 기록한다.
2. 금액/잔액/정산에 영향이 있는 요청은 재현 사례와 기대 결과를 먼저 정리한다.
3. UI 편의 개선은 업무 빈도가 높은 화면부터 반영한다.
4. 기능 변경 후에는 빌드, 관련 E2E, 필요 시 운영 화면 확인까지 기록한다.
5. 배포한 변경은 `AI_SYNC.md`의 Last Changes에 남긴다.
