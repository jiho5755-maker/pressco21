# PRD v2: 초단순 실무형 회계프로그램

> Version: 2.0  
> Date: 2026-03-11  
> Scope: offline-crm-v2  
> Principle: 기존 기능은 유지하고, 더 쉽게 쓰게 만든다.

## 1. 문서 목적

이 문서는 기존 CRM 고도화 PRD를 `직원이 바로 쓰는 실무형 회계프로그램` 방향으로 다시 정리한 기준 문서다.

핵심 원칙은 아래 3개다.

1. 직원이 어렵지 않게 써야 한다.
2. 회사가 돈 흐름에서 손해 보지 않게 기록돼야 한다.
3. 세무사/세무서 제출용 자료가 바로 나와야 한다.

## 2. 전제 조건

이번 고도화는 재구축이 아니다.

- 지금까지 만든 기능은 유지한다.
- 기존 고객/거래/명세표/제품/공급처/캘린더/설정 기능은 그대로 둔다.
- 새 기능은 기존 화면 위에 더 쉬운 흐름과 정산 기능을 얹는 방식으로 구현한다.

## 3. 유지해야 하는 현재 기능

반드시 유지:

- 고객 관리
- 고객 상세
- 명세표 작성/조회/인쇄
- 제품 관리
- 공급처 관리
- 거래/명세표 조회
- 수금 관리
- 지급 관리
- 기존 장부 원본 조회
- 기존 장부 수금/지급 이력
- 작업 계정 로그
- 거래내역 다운로드
- 고객 제출용 거래내역 인쇄
- 송장 자동 다운로드
- 회기 기준 미수/미지급 집계
- 기존 장부 백업 복원 데이터

## 4. 최종 제품 목표

이 프로그램은 복잡한 ERP가 아니라 아래 목표를 가진다.

- `쉬운 입력`
- `정확한 돈 관리`
- `실수 방지`
- `제출 가능한 장부 출력`

## 5. 핵심 화면 구조

직원이 자주 쓰는 핵심 화면은 4개로 본다.

### A. 고객 관리

- 고객 검색
- 고객 상세
- 현재 받을 돈/줄 돈/예치금/환불대기 확인
- 거래원장 이동

### B. 수금 관리

- 받을 돈 확인
- 입금 확인
- 초과 입금 처리
- 예치금 확인

### C. 지급 관리

- 줄 돈 확인
- 지급 확인
- 환불대기 확인
- 지급/환불 처리

### D. 거래원장

- 출고
- 입금
- 지급
- 예치금 적립/사용
- 환불
- 취소

를 시간순으로 조회/출력

## 6. 직원이 이해해야 하는 돈 개념

직원에게는 아래 4개만 보여준다.

- `받을 돈`
- `줄 돈`
- `예치금`
- `환불대기`

필요하면 작은 글씨로 회계 용어를 병기한다.

- 받을 돈 `(미수금)`
- 줄 돈 `(미지급금)`

## 7. 업무 규칙

### 7.1 받을 돈

- 고객이 아직 우리에게 줘야 하는 돈
- 수금 처리 시 차감

### 7.2 줄 돈

- 우리가 실제로 보내야 하는 돈
- 지급 처리 시 차감

### 7.3 예치금

- 고객이 미리 더 낸 돈
- 다음 거래에서 차감할 수 있는 돈
- 정부지원금 선입금, 실수 초과입금도 여기에 포함 가능

### 7.4 환불대기

- 돌려주기로 확정됐지만 아직 송금하지 않은 돈
- 환불 완료 시 차감

## 8. 실무 처리 원칙

### A. 입금

- 정상 입금이면 받을 돈만 차감
- 초과 입금이면 처리 방식을 반드시 선택

선택지:

- 예치금으로 보관
- 환불대기로 등록
- 기존 받을 돈에 우선 차감

### B. 지급

- 줄 돈 또는 환불대기만 대상으로 처리
- 지급 후 거래원장에 `지급` 또는 `환불` 이력 생성

### C. 취소

- 삭제하지 않는다
- 취소 이력을 남긴다
- 원래 금액이 다시 복원돼야 한다

## 9. 최소 거래 유형

직원이 다뤄야 하는 거래 유형은 아래만 쓴다.

- 출고
- 입금
- 지급
- 예치금 적립
- 예치금 사용
- 환불
- 반입
- 취소

## 10. 화면 원칙

### 수금 관리

- 상단 카드
  - 총 받을 돈
  - 기존 장부 받을 돈
  - 새 입력 받을 돈
  - 총 줄 돈
- 주요 액션
  - 입금 확인
  - 초과 입금 처리
  - 고객 상세 이동

### 지급 관리

- 상단 카드
  - 총 줄 돈
  - 기존 장부 받을 돈
  - 새 입력 받을 돈
  - 총 줄 돈
- 주요 액션
  - 지급 확인
  - 환불 처리
  - 고객 상세 이동

### 고객 상세

- 정산 카드
  - 받을 돈
  - 줄 돈
  - 예치금
  - 환불대기
- 빠른 이동
  - 수금 관리
  - 지급 관리
  - 거래원장
  - 명세표 작성

## 11. 실수 방지 원칙

- 받을 돈보다 많이 입금 처리 못 함
- 줄 돈보다 많이 지급 처리 못 함
- 초과 입금은 반드시 분기 처리
- 현재 작업 계정 항상 표시
- 처리 전/후 금액 항상 표시
- 잘못 처리한 건 즉시 취소 가능

## 12. 출력/제출 자료

복잡한 분개장보다 바로 제출 가능한 실무 자료를 우선한다.

- 기간별 거래원장
- 고객별 받을 돈 현황표
- 공급처별 줄 돈 현황표
- 고객별 예치금 현황표
- 환불대기 현황표
- 월별 매출/입금/지급 요약표

형식:

- 엑셀
- 인쇄용 문서

## 13. 데이터 원칙

- 모든 돈 이동은 로그로 남긴다.
- 수금/지급/환불/예치금/취소는 이벤트 단위로 기록한다.
- 덮어쓰기보다 이력 누적을 우선한다.

## 14. UX 원칙

- 설명보다 행동 흐름을 먼저 보여준다.
- 문서형 도움말보다 단계형 가이드를 우선한다.
- 버튼을 따라가며 업무를 수행할 수 있어야 한다.
- 회계 용어보다 쉬운 말이 먼저 보여야 한다.

## 15. 우선순위

### P1

- 초과 입금 처리
- 예치금
- 환불대기
- 명세표 예치금 사용
- 거래원장 유형 확장

### P2

- 단계형 가이드 투어
- 고객 상세 정산 카드 정리
- 보고서 1차 완성

### P3

- 농협 입금 자동 매칭
- 승인 큐
- 자동 수금 반영

## 16. 성공 기준

- 직원이 핵심 화면 4개만으로 업무 가능
- 초과 입금이 예치금/환불대기로 정확히 나뉨
- 거래원장과 잔액이 맞음
- 취소 이력이 남음
- 제출 자료를 바로 출력 가능
- 기존 기능은 유지되고 연결은 더 좋아짐
