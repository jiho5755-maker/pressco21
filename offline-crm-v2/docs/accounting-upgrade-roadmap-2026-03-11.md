# 로드맵: PRESSCO21 회계 운영 프로그램 고도화

> Date: 2026-03-11  
> Scope: offline-crm-v2  
> Reference PRD: [accounting-upgrade-prd-2026-03-11.md](/Users/jangjiho/workspace/pressco21/offline-crm-v2/docs/accounting-upgrade-prd-2026-03-11.md)

## 1. 이번 작업의 출발점

현재 상태는 아래 기준으로 복구 가능 포인트를 고정했다.

- Git tag: `crm-backup-before-accounting-upgrade-20260311`
- 당시 기준 커밋: `3eee628`
- 목적: 회계 고도화 중 문제가 생기면 언제든 이 시점으로 복귀

## 2. 최종 목표

1. 회계 운영 프로그램 고도화
2. 직원 사용 가이드 투어
3. 농협 계좌 입금 자동 반영

## 3. 실행 원칙

- 한 번에 다 바꾸지 않는다.
- `처리 엔진`을 먼저 만들고 `UX`를 올린다.
- 자동화는 마지막에 붙인다.
- 모든 단계마다 롤백 가능 상태를 유지한다.

## 4. 단계별 로드맵

### Phase 0. 안전장치와 기준 고정
기간: 즉시

#### 목표
- 현재 상태를 복구 가능한 기준점으로 고정
- 회계 고도화 동안 데이터 훼손을 막을 준비

#### 작업
- Git tag 생성 및 원격 푸시
- 데이터 진실 원본 경로 문서화
- 운영 DB 백업 정책 점검
- 회계 용어 표준안 확정

#### 완료 기준
- 문제가 생기면 코드/데이터를 즉시 되돌릴 수 있음

### Phase 1. 회계 처리 엔진 분리
기간: 1주차

#### 목표
- `받을 돈 관리`와 `줄 돈 관리`를 대칭 구조로 분리

#### 작업
- 수금 처리 공통 모델 정리
- 지급 처리 공통 모델 추가
- 처리 로그 스키마 통일
- 거래원장에 `지급`, `취소`, `조정` 타입 추가
- 고객/공급처 상세에 정산 카드 구조 통일

#### 산출물
- 수금 처리
- 지급 처리
- 취소 처리
- 거래원장 반영

#### 리스크
- 기존 미수금 계산과 충돌 가능
- 지급 처리 분류 기준 미정 시 UI 혼선 가능

### Phase 2. 화면 구조 재설계
기간: 2주차

#### 목표
- 비전공자 기준으로 업무 흐름이 바로 이해되게 함

#### 작업
- 메뉴 구조 개편
  - `수금 관리`
  - `지급 관리`
  - `거래원장`
- 화면 상단 KPI 정리
- 쉬운 용어 전면 반영
- 상태 배지 추가
  - 대기
  - 부분 처리
  - 완료
  - 확인 필요

#### 산출물
- 신규/개편 화면 디자인
- 통일된 용어표
- 상태 배지 정책

### Phase 3. 실수 방지 UX 고도화
기간: 3주차

#### 목표
- 잘못 입력하더라도 복구 가능하고, 헷갈리지 않게 함

#### 작업
- 처리 전 영향 안내
- 초과 금액 방지
- 현재 작업 계정 상시 표시
- 취소 시 이력 유지
- 처리 후 바로가기
  - 고객 상세
  - 거래원장
  - 정산서

#### 산출물
- 수금/지급 다이얼로그 개선
- 로그 시각화
- 빠른 액션 버튼

### Phase 4. 직원 가이드 투어
기간: 4주차

#### 목표
- 직원이 프로그램을 설명 없이도 따라 쓸 수 있게 함

#### 작업
- 첫 진입 온보딩
- 화면별 핵심 버튼 안내
- 다시 보기 도움말
- 용어 설명
- 역할별 가이드
  - 일반 직원
  - 관리자

#### 산출물
- 인앱 투어
- 화면별 도움말 패널
- 기본 사용 시나리오

#### 측정
- 신규 직원 셋업 시간
- 반복 문의 감소

### Phase 5. 정산서/보고서 체계
기간: 5주차

#### 목표
- 실제 제출/보고용 서류를 프로그램에서 바로 생성

#### 작업
- 고객별 받을 돈 정산서
- 공급처별 줄 돈 정산서
- 기간별 거래원장 엑셀
- 회기별 잔액 보고서
- 담당자별 처리 로그 보고서

#### 산출물
- 인쇄 양식
- 엑셀 내보내기
- 관리자 요약 리포트

### Phase 6. 농협 입금 자동화 1차
기간: 6~7주차

#### 목표
- 계좌 입금 내역을 받아 자동 후보 매칭까지 구현

#### 작업
- 농협 계좌 입금 데이터 수집 경로 확정
- 입금 내역 저장 테이블 설계
- 고객 미수와 매칭 규칙 구현
- 자동 후보 큐 구현

#### 산출물
- 입금 수집 배치
- 자동 후보 리스트
- 검토 승인 화면

#### 의사결정 필요
- 농협 오픈 API 사용 여부
- 계좌 조회 권한/보안 방식
- 입금 주기

### Phase 7. 농협 입금 자동화 2차
기간: 8주차

#### 목표
- 명확한 입금은 자동 반영

#### 작업
- 자동 승인 규칙 적용
- 수금 처리 자동 생성
- 거래원장 입금 행 자동 생성
- 예외/실패 로그 수집

#### 산출물
- 자동 수금 파이프라인
- 예외 검토 대기열
- 자동화 운영 로그

## 5. 우선순위 백로그

### P1. 꼭 필요한 것
- 지급 확인 기능
- 지급 취소 기능
- 공급처별 줄 돈 원장
- 거래원장에 지급 반영
- 쉬운 용어 전환
- 현재 작업 계정 표시 강화

### P2. 빠르게 가치가 큰 것
- 직원 온보딩 투어
- 처리 후 바로가기
- 정산서 출력
- 담당자별 로그 조회
- 상태 배지/예정일 관리

### P3. 자동화 단계
- 농협 입금 수집
- 자동 후보 매칭
- 자동 수금 반영
- 예외 승인 큐

### M2. 비전공자 친화 UX 완성

### M4. 자동 수금 운영 시작
- 명확 건 자동 반영
- 예외 건만 검토
