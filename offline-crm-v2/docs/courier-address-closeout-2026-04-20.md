# 송장 엑셀 선택 배송지 복구 마감 기록 (2026-04-20)

## 사용자 확인 이슈

CRM 명세표에서 배송지를 `배송지 2` 또는 `배송지 3`으로 선택해 저장한 뒤 `송장 엑셀`을 다운로드해도 택배 업로드용 엑셀의 `받는분총주소`가 기본주소로 나오는 회귀가 있었다.

## 이전 이력 확인

관련 복구 이력은 아래 커밋에서 확인했다.

- `9dcfd5d` — 명세표 주소 출력 경로 정합성 수정
- `a8f82a5` — 명세표 선택 주소 메타 저장 반영
- `25f4571` — 명세표 배송주소 셀렉트 반영 수정
- `9989fc3` — CRM 수금 반영 및 송장 주소 우선순위 수정

## 원인

1. 고객 주소 유틸이 `address1`, `address2`, `extra_addresses` 중심으로 동작하면서 `address3`~`address10` 계열 주소를 충분히 복원하지 못할 수 있었다.
2. 송장 엑셀 생성 경로에서 명세표에 저장된 기본주소 스냅샷(`customer_address`)이 남아 있으면, 비기본 배송지 키(`customer_address_key` 또는 명세표 메모 메타의 `customerAddressKey`)보다 먼저 사용되는 케이스가 있었다.
3. 명세표 목록 캐시에는 일부 주소 필드만 들어오므로, 선택 배송지 키가 캐시에 없으면 기본주소 fallback으로 떨어질 여지가 있었다.

## 적용한 수정

### `src/lib/api.ts`

- `getCustomerAddressEntries()`가 `address1`~`address10` 전체를 순회하도록 복구했다.
- 기존 `extra_addresses` JSON 주소도 계속 병합한다.
- 값이 같은 주소는 중복 제거한다.

### `src/components/InvoiceDialog.tsx`

- 명세표 수정/복사로 열 때 선택 주소 키 복원 우선순위를 고정했다.
  1. 명세표 메모 메타 `customerAddressKey`
  2. 명세표 컬럼 `customer_address_key`
  3. `address1`
- 저장 payload에 `customer_address`, `customer_address_key`를 명시적으로 포함한다.
- 고객 스냅샷 생성 시 잘못된 키가 들어와도 실제 고객 주소 목록 기준으로 유효한 키로 정규화한다.

### `src/pages/Invoices.tsx`

- 송장 엑셀 주소 해석에서 비기본 주소 키가 있으면 저장 스냅샷보다 현재 고객의 선택 주소 값을 우선한다.
- 목록 캐시가 선택 주소 키를 갖고 있지 않으면 해당 고객을 전체 조회해 주소를 재해석한다.
- 기본주소 키(`address1`)는 기존처럼 저장 스냅샷 우선 동작을 유지한다.

## 운영 직접 검증

운영 CRM에서 실제 브라우저 + 실제 다운로드 파일 검사로 확인했다.

검증 흐름:

1. 운영 API로 임시 고객 생성
   - 기본주소
   - 배송지 2
   - 배송지 3 (`extra_addresses:0`)
2. 운영 `/invoices`에서 새 명세표 작성
3. 배송 주소 셀렉트에서 `배송지 3` 선택
4. 명세표 저장
5. 명세표 목록에서 테스트 거래처로 필터링
6. `송장 엑셀` 다운로드
7. 다운로드된 `전자송장(3.9)_2026-04-20_2026-04-20.xlsx`를 열어 `받는분총주소` 검사

## 회귀 방지 장치

- `tests/02-invoices.spec.ts`에 `T2-15: 송장 엑셀은 선택한 배송지를 기본주소보다 우선 출력` 테스트를 추가했다.
- 이 테스트는 고객의 기본주소/배송지2/배송지3을 만든 뒤, 배송지3을 선택해 저장하고, 실제 다운로드된 송장 엑셀의 `받는분총주소`가 배송지3인지 검사한다.

## 앞으로 수정 시 지켜야 할 규칙

- 비기본 `customer_address_key`가 있으면 송장/인쇄 주소 해석에서 기본주소 스냅샷보다 선택 주소 키를 우선한다.
- 고객 주소 유틸은 `address1`~`address10`과 `extra_addresses`를 모두 유지해야 한다.
- `송장 엑셀` 관련 변경 시 최소 `T2-15`를 실행하고, 운영 반영 전 선택 배송지 저장/다운로드를 직접 확인한다.
