- 조회/명세표/미수금/캘린더/제품/공급처의 주요 화면은 실제 데이터로 동작한다.
- 레거시 거래 `97,086건`은 적재 완료, 고객 마스터는 `legacy_id` 기준 대체로 연결되었다.
- 하지만 레거시 백업 기준 누락/오염/잔액 불일치가 남아 있어 `100% 복원` 판정은 불가하다.
- 캘린더/미수금의 기준일 계산은 현재 운영 스키마 제약으로 프론트 보정 비중이 높다.
- 기존 Playwright 기준선 `28건`도 현재 UI와 불일치해 자동 회귀 신뢰도가 깨져 있다.

- 중복 고객 정책 확정 및 대표 레코드 기준 적용 완료

### B. 업무 정확도

- 명세표 생성/수정/복사/삭제 동작 안정화
- 고객 상세에서 레거시 거래와 CRM 명세표가 의도대로 연결
- 미수금 기준일 계산 규칙 문서화
- 캘린더 기간 리포트와 미수금 수치가 샘플 검산과 일치

### C. UX 완성도

- 중복 고객/오염 필드가 사용자에게 혼동 없이 표시
- 신규 명세표 작성 흐름이 키보드 기준으로 끊기지 않음
- 배송정보/연락처 누락을 저장 전에 감지
- 접근성 경고(Dialog description 누락 등) 제거

### D. QA 기준선

- 기존 28건 테스트 전부 통과
- 신규 핵심 흐름 테스트 추가 후 전체 기준선 확립
- 화면명/메뉴명/헤더 구조 변경 시 테스트도 함께 갱신되는 체계 확보
