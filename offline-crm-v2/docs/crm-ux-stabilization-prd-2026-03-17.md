# PRD: PRESSCO21 CRM UX 안정화 및 회계 운영 가이드 고도화

## 1. 목적

현재 CRM은 실무 기능은 충분히 갖췄지만, 비전공자 직원이 회계 흐름을 빠르게 이해하고 실수 없이 처리하기에는 다음 문제가 남아 있다.

- 화면별 기능 가이드가 하단 플로팅 버튼 중심이라 데이터 표를 가린다.
- 고객 상세, 수금/지급, 입금 수집함에서 회계 판단 근거와 실제 행동이 분리되어 있다.
- 저장/탭 전환 직후 중간 계산값이 잠깐 보이거나, 로딩 지연 때문에 클릭이 반영되지 않은 것처럼 느껴진다.
- 직원이 한 작업의 결과와 변경 이력을 다시 확인하는 흐름이 약하다.

이 문서는 `비전공자 회계 담당자가 매일 써도 헷갈리지 않는 CRM`을 목표로 UX 안정화 범위를 정의한다.

## 2. 목표

### 2.1 핵심 목표

1. 직원이 `지금 무엇을 해야 하는지`를 화면에서 바로 이해할 수 있어야 한다.
2. 기능 가이드는 업무 흐름을 설명하되 실제 데이터 표를 가리지 않아야 한다.
3. 수치와 상태는 가능한 한 첫 화면부터 최종값에 가깝게 보여야 한다.
4. 수정/삭제/반영 실수는 되돌리기와 로그로 추적 가능해야 한다.

### 2.2 비목표

- 이번 범위에서 회계 규칙 자체를 전면 변경하지 않는다.
- 외부 은행 API 도입이나 NH 메일 수집 구조 변경은 포함하지 않는다.
- 신규 백엔드 서비스 분리 없이 현재 React + n8n + NocoDB 구조 안에서 안정화를 우선한다.

## 3. 사용자

### 3.1 1차 사용자

- 사내 비전공자 회계 담당자
- 고객 응대와 회계 보조를 같이 담당하는 운영 직원

### 3.2 2차 사용자

- 관리자
- 고객 제출용 자료를 출력하는 영업/관리 담당자

## 4. 현재 문제 정의

### 4.1 가이드 UX 문제

- 하단 플로팅 가이드 버튼이 표와 카드 하단을 가린다.
- 현재 화면에 맞는 핵심 행동보다 단계 설명이 먼저 보인다.
- 고객 상세 같은 복합 화면 전용 가이드가 부족하다.

### 4.2 회계 판단 UX 문제

- `받을 돈`, `줄 돈`, `최종 정산`은 좋아졌지만, 다음 행동까지는 충분히 연결되지 않는다.
- 고객 제출용 자료와 내부 회계용 데이터가 일부 화면에서 여전히 섞여 보인다.
- 미수/미지급/이관/조정 흐름이 내부 용어 중심으로 노출된다.

### 4.3 속도 체감 문제

- 탭 전환 직후 빈 상태 또는 임시 계산값이 먼저 보일 수 있다.
- 무거운 다이얼로그와 목록 데이터가 동시에 로드되면 첫 진입 체감이 늦다.
- 저장 직후 결과 수치가 즉시 보이지 않으면 직원이 같은 작업을 다시 시도할 위험이 있다.

### 4.4 검증/감사 문제

- 되돌리기는 개선됐지만, 작업 결과를 다시 확인하는 흐름은 아직 약하다.
- 활동 로그는 있으나 화면별 감사 관점으로 모아 보는 센터가 없다.

## 5. 제품 원칙

1. 행동 중심
   - 버튼과 가이드는 `입금 반영`, `지급 처리`, `다른 고객으로 이관`처럼 행동을 먼저 말한다.
2. 결과 우선
   - 계산 재료보다 `최종 얼마를 받아야/줘야 하는지`를 먼저 보여준다.
3. 단계적 공개
   - 초보자는 핵심 요약만, 숙련자는 세부 근거까지 펼쳐 볼 수 있어야 한다.
4. 되돌릴 수 있음
   - 중요한 조작은 즉시 undo 또는 로그 추적이 가능해야 한다.
5. 고객 제출과 내부 운영 분리
   - 고객용 문서는 전문적이고 간결해야 하며 내부 조정 정보는 제외한다.

## 6. 범위

## 6.1 Phase 1: 가이드 네비게이션 개편

- 하단 플로팅 가이드 버튼 제거
- 상단 우측 `업무 가이드` 진입 버튼 도입
- 화면별 가이드를 사이드 패널 또는 상단 패널로 재구성
- 고객 상세 전용 가이드 추가
- 빠른 이동, 용어 설명, 오늘의 작업 순서 포함

## 6.2 Phase 2: 회계 흐름 가독성 강화

- 고객 상세 상단 정산 카드와 문장을 더 요약형으로 정리
- 거래/수금/지급 화면에 `지금 해야 할 일` 블록 도입
- 고객 제출 기준 거래와 내부 회계 거래 분리 토글 검토

## 6.3 Phase 3: 작업 결과 및 감사 로그 가시화

- 최근 작업 요약 패널 강화
- 회계 활동 로그 센터 설계
- 수정/삭제/이관/입금 반영 결과를 사건 단위로 추적

## 6.4 Phase 4: 성능 및 반응성 최적화

- 고객 상세, 수금/지급, 거래원장 주요 수치를 optimistic 하게 보여주기
- 상세 다이얼로그, 인쇄, 차트 영역 추가 분리 로드
- 자주 쓰는 화면 prefetch 확대

## 7. 기능 요구사항

### 7.1 업무 가이드 네비게이션

- `업무 가이드`는 데이터 화면을 가리지 않는 상단 진입점이어야 한다.
- 현재 경로에 맞는 가이드가 자동 선택되어야 한다.
- 가이드는 아래를 포함해야 한다.
  - 이 화면에서 하는 일
  - 가장 먼저 볼 숫자
  - 처리 순서 3~5단계
  - 관련 화면 빠른 이동
  - 핵심 용어 설명
- 기존 `이 화면 숨기기` 상태와 `가이드 다시보기 초기화`는 유지한다.

### 7.2 고객 상세 전용 가이드

- `받을 돈 / 줄 돈 / 최종 정산` 읽는 법
- 새 장부 미수와 기존 장부 미지급 차이
- 고객 제출 자료와 내부 거래내역 차이
- 수금/지급/이관으로 이어지는 처리 순서

### 7.3 속도/상태 UX

- 로딩 중에도 최종 계산에 가까운 수치를 우선 노출한다.
- 중간 계산값이 잠깐 잘못 보이는 현상을 최소화한다.
- 반영 중 상태는 전역에서 일관되게 보인다.

### 7.4 검증 기준

- `npm run build` 통과
- 주요 화면 headless 브라우저 스모크 테스트 통과
- 고객 상세에서 상단 정산 카드 값이 초기 0원으로 튀지 않아야 함
- 업무 가이드 패널이 표/카드를 가리지 않아야 함

## 8. 성공 지표

- 하단 가이드 버튼 제거 후 데이터 가림 불만 0건
- 주요 화면 첫 판단 시간 감소
  - 고객 상세/수금/지급 화면에서 “무엇을 해야 하는지” 판단까지 5초 이내
- 동일 작업 재클릭 빈도 감소
- 되돌리기 및 로그 활용률 증가

## 9. 출시 전략

1. Phase 1 문서 고정 후 구현
2. 빌드 및 주요 화면 스모크 테스트
3. 운영 배포
4. 실사용 피드백 수집 후 Phase 2 진행
