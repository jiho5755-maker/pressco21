# ROADMAP: PRESSCO21 CRM UX 안정화 실행 계획

> Reference PRD: [crm-ux-stabilization-prd-2026-03-17.md](/Users/jangjiho/workspace/pressco21/offline-crm-v2/docs/crm-ux-stabilization-prd-2026-03-17.md)

## 1. 원칙

- 기능 추가보다 `실무자가 바로 이해하는 UX`를 우선한다.
- 각 단계는 `구현 → 빌드 → 브라우저 검증 → 운영 배포`까지 완료 후 다음 단계로 넘어간다.
- 회계 숫자와 관련된 변경은 실제 고객 데이터 화면에서 최소 1건 이상 확인한다.

## 2. 진행 단계

### Phase 1. 가이드 네비게이션 전환

- [ ] PRD/ROADMAP 고정
- [ ] 하단 플로팅 가이드 버튼 제거
- [ ] 상단 `업무 가이드` 진입점 도입
- [ ] 현재 화면 기반 가이드 패널 구현
- [ ] 고객 상세 전용 가이드 추가
- [ ] Settings의 `가이드 다시보기 초기화`와 연동 확인
- [ ] 빌드/스모크 테스트/운영 배포

### Phase 2. 회계 흐름 읽기 쉬운 화면 정리

- [ ] 고객 상세 `정산 요약` 문구를 더 간결하고 공식적으로 정리
- [ ] 수금/지급 화면에 `지금 해야 할 일` 블록 추가
- [ ] 거래/명세표 조회의 내부용/고객제출용 구분을 더 명확히 정리
- [ ] 고객 제출용 문서 문구와 컬럼 최종 정리
- [ ] 빌드/스모크 테스트/운영 배포

### Phase 3. 작업 결과와 감사 흐름 강화

- [ ] 최근 작업 요약 패널 고도화
- [ ] 활동 로그 센터 1차 설계
- [ ] 주요 회계 액션 로그를 사건 단위로 재정리
- [ ] 되돌리기 가능한 액션 범위 점검
- [ ] 빌드/검증/운영 배포

### Phase 4. 반응 속도와 중간 상태 최소화

- [ ] 고객 상세 상단 카드의 중간 상태 제거 추가 보정
- [ ] 수금/지급/입금수집함의 optimistic 반영 범위 확대
- [ ] 차트/인쇄/상세 다이얼로그 추가 분리 로드
- [ ] 자주 쓰는 고객 상세 prefetch 강화
- [ ] 빌드/스모크 테스트/운영 배포

## 3. 이번 세션 목표

이번 세션에서는 Phase 1을 완료한다.

### 완료 조건

- 하단 고정 가이드 버튼이 사라진다.
- 상단에서 `업무 가이드`를 열 수 있다.
- 현재 화면에 맞는 가이드와 빠른 이동이 동작한다.
- 고객 상세 전용 가이드가 제공된다.
- 주요 데이터 표를 가리지 않는다.
- 빌드와 브라우저 스모크 테스트가 통과한다.

## 4. 검증 시나리오

1. 고객 상세 진입
   - 상단 요약 카드 정상 표시
   - 업무 가이드 버튼 노출
   - 고객 상세 전용 가이드 내용 표시
2. 수금 관리 진입
   - 표와 하단 영역이 버튼에 가려지지 않음
   - 가이드 빠른 이동으로 다른 화면 전환 가능
3. 입금 수집함 진입
   - 현재 화면 요약과 처리 순서 표시
4. 설정 진입
   - `화면 가이드 다시보기 초기화` 동작

## 5. 리스크

- 가이드 패널이 상단 상태 위젯과 충돌할 수 있음
- 고객 상세 전용 selector가 없으면 하이라이트가 불안정할 수 있음
- 패널 도입 후 모바일/작은 해상도에서 폭이 과할 수 있음

## 6. 메모

- 현재 하단 `AppGuideWidget` 구현을 전면 폐기하기보다, 기존 가이드 데이터와 하이라이트 로직을 재사용하는 것이 안전하다.
