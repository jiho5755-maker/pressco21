# RPA 자동송금 은행 메뉴 체크리스트

## 목적

- 특정 은행이 RPA 기반 직접 송금에 적합한지 실무 기준으로 빠르게 판단한다.
- 직원 또는 대표가 인터넷뱅킹 화면을 직접 보며 체크할 수 있게 한다.

## 우선 확인 은행

1. 신한
2. KB
3. 우리

## 공통 체크 항목

### 1. 로그인/인증

- 인터넷뱅킹 또는 기업뱅킹 로그인 가능
- 매번 비정상적으로 많은 보안 팝업이 뜨지 않음
- OTP 또는 인증서 입력 흐름이 반복 가능
- 브라우저가 고정 가능
- 실행 중 세션 만료가 과도하지 않음

판단:

- 쉬움
- 보통
- 어려움

### 2. 송금 메뉴

아래 중 최소 1개는 필요:

- 예약이체
- 다건이체
- 파일이체 업로드
- 즉시이체

우선순위:

1. 예약이체
2. 다건이체
3. 파일이체
4. 즉시이체

### 3. 입력 화면 안정성

- 계좌번호 입력란이 명확함
- 예금주 확인 결과가 화면에 표시됨
- 금액 입력란이 명확함
- 메모/적요 입력란이 있음
- 완료 결과 화면이 안정적으로 보임

### 4. 결과 확인

- 송금 성공/실패 문구가 명확함
- 이체 결과 조회 메뉴가 있음
- 최근 거래내역/이체내역 조회가 쉬움

### 5. 대량 처리 적합성

- 여러 건을 한 번에 처리하는 메뉴가 있음
- 업로드 파일 형식이 제공됨
- 예약 등록 후 조회/취소가 가능함

## 은행별 실사 항목

### 신한

- 기업뱅킹 또는 인터넷뱅킹에서 예약이체 메뉴 존재 여부
- 다건이체 또는 급여이체 메뉴 존재 여부
- 파일 업로드형 이체 지원 여부
- 이체 결과 조회 메뉴 존재 여부
- 개인사업자 사업자계좌 기준 사용 가능 여부

### KB

- 대량이체 메뉴 존재 여부
- 예약이체 메뉴 존재 여부
- StarCMS 또는 유사 자금관리 메뉴 존재 여부
- 업로드 파일 기능 존재 여부
- 개인사업자 이용 가능 여부

### 우리

- 웹펌뱅킹 또는 지급서비스 메뉴 존재 여부
- 예약/다건 이체 기능 존재 여부
- 개인사업자 계약 가능 여부
- 수동이 아닌 자동/반자동 연계 가능 구조 여부

## 체크 방식

각 항목은 아래 중 하나로 기록:

- 가능
- 불명확
- 불가

## 최종 판단 기준

### 파일럿 적합

- 예약이체 또는 다건이체 가능
- 로그인/인증 흐름이 고정적
- 결과 조회 가능
- 직원이 메뉴를 이해하기 쉬움

### 보류

- 메뉴는 있으나 인증 흐름이 너무 복잡함
- 예금주/결과 확인 구조가 불명확함
- UI 요소가 동적으로 크게 바뀜

### 탈락

- 예약이체/다건이체/파일이체 모두 없음
- 로그인 보안 절차가 매번 불규칙함
- 결과 조회가 어려움
- 반복 자동화에 부적합

## 실사 후 정리해야 할 정보

3. 사용 메뉴명
4. 필요 인증 수단
5. 추천 자동화 방식
   - 예약이체
   - 다건이체
   - 파일이체
   - 즉시이체
6. 파일럿 적합 여부
