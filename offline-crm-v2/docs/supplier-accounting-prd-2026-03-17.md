# 공급처 회계 고도화 PRD

## 목적

- 공급처 지급을 `지급 관리` 안에서 실무자가 바로 처리할 수 있는 수준으로 고도화한다.
- 비전공자 직원이 `누구에게`, `얼마를`, `언제까지`, `무슨 근거로` 지급해야 하는지 한 번에 이해하도록 만든다.
- 기능 추가보다 `우선순위`, `상태`, `감사 추적`, `제출 문서`를 중심으로 마감 가능한 구조를 만든다.

## 현재 상태

현재 시스템에서 가능한 범위:

- 공급처 계좌 등록/수정
- 지급 예정 금액 등록
- 지급 완료/취소
- 활동 로그 기록
- 기간 리포트/문서 출력
- 공급처 관리 화면에서 예정 금액, 최근 지급 확인

현재 부족한 점:

- 지급 우선순위가 한눈에 보이지 않음
- `오늘 처리해야 할 건`, `기한 지난 건`, `계좌 빠진 건`이 섞여 보임
- 공급처 목록과 지급 관리가 연결되어 있지만, 상태가 직관적이지 않음
- 운영자는 숫자는 볼 수 있지만 행동 우선순위를 바로 판단하기 어려움

## 사용자

1. 회계 관리자
- 지급 우선순위를 정하고 실제 지급 처리
- 월별 지급 내역 검토
- 공급처 확인서 발행

2. 비전공자 운영 직원
- 계좌 누락 공급처 확인
- 지급 예정 금액 등록
- 지급 완료 후 상태 확인

3. 대표/관리자
- 지급 규모와 위험 항목을 요약으로 확인

## 핵심 원칙

1. `수금 관리`와 `지급 관리`는 계속 분리한다.
- 수금: 들어오는 돈
- 지급: 나가는 돈

2. 공급처 지급은 `지급 관리` 안에서 별도 성격으로 유지한다.
- 고객 지급/환불과는 문구와 상태를 다르게 쓴다.

3. 화면은 `데이터 구조`보다 `행동 우선순위`를 먼저 보여준다.
- 기한 초과
- 오늘 지급
- 계좌 확인 필요
- 예정만 등록

4. 제출 문서와 내부 운영 기록을 분리한다.
- 내부: 우선순위, 메모, 활동 로그
- 외부: 공식 지급 내역서/예정서

## 1차 목표

### A. 지급 상태 체계

공급처별 지급 상태를 아래로 구분한다.

- `계좌 확인 필요`
- `지급 기한 초과`
- `오늘 지급`
- `지급 예정`
- `최근 지급 완료`

### B. 우선순위 요약

지급 관리 상단에서 바로 보여준다.

- 오늘 지급할 공급처 수 / 금액
- 기한 지난 공급처 수 / 금액
- 계좌 누락 공급처 수
- 예정만 등록된 공급처 수

### C. 공급처 목록 연동

공급처 관리 화면에서도 상태가 보인다.

- 지급 예정 금액
- 예정일
- 상태 배지
- 지급 관리 바로 이동

## 2차 목표

- 공급처별 부분 지급 누적
- 지급 승인 상태
- 공급처 정산 주기 관리
- 월 마감 검토 카드

## 3차 목표

- 매입/제품 흐름과 지급 예정 자동 연결
- 은행 출금 이벤트와 공급처 지급 대조
- 공급처별 연간 지급 리포트

## 성공 기준

- 회계 담당자가 `지급 관리` 화면만 보고 오늘 처리 대상을 바로 판별할 수 있다.
- 공급처 관리 화면에서 계좌 누락과 지급 예정 상태를 즉시 알 수 있다.
- 월별 지급 검토에 별도 수기 정리가 거의 필요하지 않다.

## 이번 턴 구현 범위

1. 공급처 회계 PRD/ROADMAP 문서 작성
2. 지급 상태/우선순위 헬퍼 추가
3. 지급 관리 `공급처 지급(준비)` 우선순위 요약 추가
4. 공급처 관리 상태 배지 및 예정일 표시 추가
5. 빌드/브라우저 검증/운영 배포
