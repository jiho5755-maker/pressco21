# 공급처 지급 로드맵

## 목적

- `지급 관리`를 고객 환불/고객 지급뿐 아니라 `공급처 지급`까지 포함하는 출금 관리 화면으로 확장한다.
- 비전공자 직원이 `누구에게 얼마를 줘야 하는지`, `지금 실제로 처리할 수 있는지`, `계좌 정보가 준비됐는지`를 한눈에 이해할 수 있게 만든다.

## 1차 범위

- 지급 관리에 `공급처 지급(준비)` 탭 추가
- 공급처 계좌 등록 상태 집계
- 계좌 미등록 공급처 목록 노출
- 운영자가 공급처 지급 준비 상태를 화면에서 바로 확인 가능

## 2차 범위

- 공급처별 지급 예정 금액 수동 등록
- 지급 예정 사유 분류
  - 매입 대금
  - 위탁 정산
  - 외주/작업비
  - 운송/포장비
- 지급 완료 / 취소 / 되돌리기
- 활동 로그 연동

## 3차 범위

- 공급처 지급과 제품/매입 흐름 연결
- 공급처별 미지급 잔액 자동 계산
- 지급 예정 리포트 / 월말 정산 리포트
- 향후 은행 출금 자동 매칭 연결

## UX 원칙

1. `수금 관리`와 `지급 관리`는 유지한다.
   - 수금: 들어오는 돈
   - 지급: 나가는 돈

2. `지급 관리` 안에서 성격을 다시 나눈다.
   - 고객 지급 예정
   - 환불 예정
   - 공급처 지급(준비)

3. 실제 처리 가능 여부를 먼저 보여준다.
   - 계좌 등록 완료
   - 계좌 미등록
   - 지급 예정 금액 등록 여부

4. 공급처 지급은 고객 지급과 문구를 다르게 쓴다.
   - 고객: 환불, 지급
   - 공급처: 정산, 대금 지급

## 데이터 준비 체크

- 공급처명
- 대표자
- 은행명
- 계좌번호
- 메모
- 향후 필요 필드
  - 지급 예정 금액
  - 지급 예정일
  - 정산 구분
  - 마지막 지급일
  - 담당자
