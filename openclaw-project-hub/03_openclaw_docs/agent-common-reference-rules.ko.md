# 개발 에이전트 공통 참조 규칙 v1

_대상: 플로라, Claude Code, Codex, 향후 개발 에이전트_

## 1. 목적

이 문서는 PRESSCO21 개발/설계 요청을 받을 때 어떤 상위 문서를 먼저 읽고, 어떤 순서로 판단할지 정하는 공통 규칙이다.

---

## 2. 최우선 참조 원칙

공통 설계, 데이터 구조, 플로라 운영 방식, 시스템 통합 방향과 관련된 요청은
기본적으로 `openclaw-project-hub`의 공통 문서를 우선 참조한다.

---

## 3. 상위 문서 우선순위

### 1순위: 회사 공통 설계 문서
- `company-integrated-os-architecture.ko.md`
- `company-db-decision-guide.ko.md`
- 향후 추가될 원천데이터/상태코드/자동화 정책 문서

### 2순위: 라우팅/운영 정책 문서
- 플로라 specialist routing policy
- owner/frontdoor 정책 문서
- 에이전트 공통 운영 규칙

### 3순위: 프로젝트별 문서
- 프로젝트 `README.md`
- 프로젝트 `AGENTS.md`
- 프로젝트 roadmap / PRD / 기능 문서

### 4순위: 구현 코드 / 로컬 맥락
- 실제 코드 구조
- 최근 변경 흔적
- 테스트/배포/스크립트

---

## 4. 요청 유형별 읽기 순서

## 4.1 DB / 데이터 구조 / 통합 아키텍처 질문
1. `company-integrated-os-architecture.ko.md`
2. `company-db-decision-guide.ko.md`
3. 관련 프로젝트 문서
4. 구현 코드

## 4.2 플로라 운영 / 역할 / 라우팅 질문
1. canonical 정책 문서
2. specialist routing 문서
3. 관련 skill 문서
4. 실제 설치/검증 스크립트

## 4.3 프로젝트 개발 질문
1. 공통 상위 문서와 충돌 여부 먼저 판단
2. 프로젝트 `AGENTS.md` / `README.md`
3. 세부 PRD / roadmap
4. 코드/테스트/스크립트

## 4.4 신규 시스템 제안 질문
1. 통합 아키텍처 문서
2. DB 의사결정 문서
3. 원천 엔터티/상태코드 문서(향후 추가)
4. 관련 프로젝트 문서

---

## 5. 반드시 답해야 할 공통 질문

에이전트는 설계/제안 전에 아래를 먼저 묻거나 내부적으로 답한다.

1. 이 요청은 회사 공통 원칙에 영향을 주는가?
2. 원천데이터를 새로 만들거나 바꾸는가?
3. 기존 프로젝트 문서보다 상위 문서가 먼저 필요한가?
4. 이 결과가 다른 시스템에도 재사용될 가능성이 큰가?
5. 허브 canonical 문서에 반영해야 하는 변화인가?

---

## 6. 금지 규칙

- 상위 공통 문서를 확인하지 않고 프로젝트 로컬 판단만으로 구조를 바꾸지 않는다.
- 프로젝트 편의 때문에 공통 정책과 충돌하는 답을 하지 않는다.
- 구현 디테일만 보고 회사 전체 구조를 뒤집는 제안을 하지 않는다.

---

## 7. 최종 운영 문장

> 공통 설계 질문은 허브의 상위 문서를 먼저 본다.  
> 프로젝트 문서는 그 다음에 본다.  
> 공통 원칙과 프로젝트 구현이 충돌하면 우선 공통 원칙을 기준으로 정리한다.
