# PRESSCO21 원천데이터 정교화 질문지 v1

_대상: 지호님, 대표님, 실무자, 향후 인터뷰 응답자_

## 사용 목적
이 질문지는 PRESSCO21의 원천데이터 체계를 최종 확정하기 위해 사용한다.
응답은 CRM, 파트너클래스, NocoDB, 자동화, 플로라 OS 설계의 기준이 된다.

응답 방식:
- 모르면 "모름" 또는 "추후 확인"이라고 적어도 된다.
- 반드시 완벽한 표현일 필요 없다.
- 실제 운영 기준으로 답해주면 된다.

---

# 1. 고객 / 거래처 / 조직 구분

## 1-1. 고객
1. 우리 회사에서 "고객"이라고 부르는 대상은 정확히 누구인가?
2. 개인 고객과 사업자 고객을 어떻게 구분하고 있나?
3. 한 사람이 개인 고객이면서 동시에 공방 운영자/강사일 수 있나?
4. 고객을 구분할 때 반드시 알아야 하는 정보는 무엇인가?
5. 고객 관리에서 가장 자주 헷갈리는 경우는 무엇인가?

## 1-2. 거래처 / 기관 / 협회
6. 거래처와 고객은 실제로 어떻게 다르게 취급하나?
7. 학교, 협회, 기업, 공방은 모두 같은 "조직"으로 봐도 되나?
8. 조직 단위에서 반드시 저장해야 하는 정보는 무엇인가?
9. 조직 거래에서 개인 담당자는 어떻게 연결되고 있나?
10. 거래처별 정산 방식이나 문서 방식이 다르면 어떤 식으로 다른가?

---

# 2. 파트너 / 강사 / 공방 구조

11. 파트너와 강사는 같은 개념인가, 다른 개념인가?
12. 파트너 1곳에 강사가 여러 명일 수 있나?
13. 강사 1명이 여러 파트너/공방과 연결될 수 있나?
14. 파트너 승인 시 꼭 확인해야 하는 기준은 무엇인가?
15. 파트너 등급은 왜 필요하고, 등급에 따라 무엇이 달라지나?
16. 파트너의 비활성/휴면/탈퇴는 어떤 상태로 봐야 하나?
17. 공방 위치/지역 정보는 어느 정도까지 중요하게 관리해야 하나?

# 3. 상품 / 클래스 / 프로그램 관계

18. 상품과 클래스는 현재 운영상 어떻게 연결되나?
19. 클래스에 필요한 재료는 항상 별도 상품으로 판매되나?
20. 키트 상품과 클래스 상품은 같은가 다른가?
21. 기관/협회 제안용 프로그램은 일반 클래스와 어떻게 다른가?
22. 프로그램/패키지 단위로 별도 관리해야 하는 정보가 있나?
23. 클래스 운영에서 꼭 관리해야 하는 핵심 정보 5개를 적어달라.

---

# 4. 신청 / 주문 / 견적 / 문서 흐름

24. 주문과 신청은 실제 운영에서 어떻게 다르게 취급하나?
25. 신청 유형에는 무엇이 있나? (예: 수강 신청, 파트너 신청, 제휴 신청 등)
26. 견적서 발행 이후 실제 거래명세서/세금/입금으로 이어지는 대표 흐름을 적어달라.
27. 가장 자주 생기는 예외 상황은 무엇인가?
28. 입금 확인이 늦어지거나 헷갈리는 대표 사례는 무엇인가?
29. 재발행/수정발행/부분입금 같은 예외는 얼마나 자주 생기나?
30. 어떤 문서는 반드시 사람이 확인해야 하나?

# 5. 결제 / 입금 / 정산

32. 입금 확인 시 가장 필요한 정보는 무엇인가?
33. 미수금은 어떤 기준으로 관리하고 있나?
34. 정산 대상은 누구까지 포함되나? (강사, 파트너, 거래처, 채널 등)
35. 정산 금액 계산 시 꼭 들어가야 하는 기준은 무엇인가?
36. 정산에서 자주 분쟁/혼선이 생기는 포인트는 무엇인가?
