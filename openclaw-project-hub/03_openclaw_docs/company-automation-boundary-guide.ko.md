# PRESSCO21 자동화 경계 가이드 v0.2 (현실반영 초안)

_상태: 현실반영 초안 / 지호님 추가 검증 필요 / 실무 적용 전_

_대상: 지호님, 플로라, Claude Code, Codex, 향후 개발 에이전트_

## 1. 문서 목적

이 문서는 PRESSCO21에서 **어디까지 자동화하고, 어디서 사람 확인을 거쳐야 하며, 어디는 자동화하면 안 되는지**를 구분하기 위한 현실반영 초안이다.

이 문서의 목적은 아래와 같다.

1. 자동화 가능한 영역과 금지 영역을 구분한다.
2. 플로라가 제안만 할지, 실행까지 할지 기준을 만든다.
3. 실무 편의와 안전성 사이 균형을 잡는다.
4. n8n / OpenClaw / CRM / 파트너클래스 자동화 설계의 기준으로 삼는다.
5. 지호님의 실질 결정권과 직원 UX 현실을 반영한다.

---

## 2. 이 문서는 아직 확정본이 아니다

이 문서는 지호님 피드백을 반영한 현실반영 초안이지만,
아직 아래 이유로 최종 확정본은 아니다.

- 대표님 기준과 직원 실사용 반응이 추가 확인되지 않음
- 실제 승인 체계를 문서화한 상태가 아님
- 자동화 가능 범위는 넓게 보되, 권한 모델은 더 세밀하게 정리해야 함

즉 이 문서는 **실행 규칙 직전 단계의 현실반영 초안**이다.

---

## 3. 현재 반영된 핵심 현실 기준

### 3.1 실질 결정권
- 법적/직함상 대표는 이진선 대표님
- 실제 운영/시스템/자동화의 실질 결정권자는 지호님
- 따라서 자동화 경계는 아래 3층으로 보는 것이 현실적이다.

1. **지호님 확인 필수**
2. **대표 최종 승인 필수**
3. **실무자 단독 가능**

### 3.2 직원 UX 원칙
직원들은 AI 자체보다 아래 요소에 더 민감할 가능성이 높다.

- 입력 규칙이 많아서 귀찮음
- AI가 먼저 자꾸 끼어듦
- 실제 사용 흐름이 불편함

즉,
**자동화 내용보다 UX가 나쁘면 안 쓴다**
가 핵심 원칙이다.

### 3.3 상태 자동화 원칙
상태 변경 자동화는 보수적으로 막기보다,
**넓게 허용 가능**하다고 본다.

단, 조건이 있다.
- 아무나 수정하면 안 됨
- 권한은 강하게 제한해야 함
- 처리력이 떨어지는 직원에게 넓게 열어두면 시스템이 오히려 흐트러질 수 있음

즉,
**자동화 범위는 넓게, 수정 권한은 좁게**
가 현실적인 방향이다.

### 3.4 외부 발송 판단 기준
외부 고객/거래처/파트너 발송은 단순 민감정보 여부보다,
**대표 없이 보냈을 때 상대방이 불만/오해를 가질 수 있는가**가 중요한 판단 기준이다.

## 4. 자동화 분류 체계 (현실반영 초안)

PRESSCO21 자동화는 아래 4단계로 분류한다.

### A. 자동 실행 가능
사람 승인 없이 자동 실행 가능

### B. 자동 제안 + 지호님 확인 후 실행
플로라/시스템이 제안 또는 초안을 만들고, 지호님이 확인 후 실행

### C. 사람 요청 시만 실행
자동으로 먼저 움직이지 않고, 사람이 요청해야만 실행

### D. 자동화 금지 또는 현단계 보류
기술적/운영상 리스크가 높아서 현재는 자동화하지 않음

## 8. 권한 모델 초안 (중요)

이 문서에서 가장 중요한 현실 포인트는 권한 모델이다.

### 레벨 1. 지호님
- 시스템/자동화/상태 변경/내부 실행 전반 확인 가능
- 실질 운영 결정권자

### 레벨 2. 대표님
- 최종 대외/금액/승인 판단 축
- 대표 명의/대표 책임이 걸리는 영역 최종 승인

### 레벨 3. 실무자
- 자기 업무 범위 내 입력/처리
- 넓은 상태 수정 권한은 제한
- UX가 단순해야 사용 가능

### 핵심 원칙
**자동화 범위는 넓게 갈 수 있지만, 수정 권한은 좁게 가져간다.**

## 12. 최종 운영 문장 (현실반영 초안)

> PRESSCO21는 브리핑, 회의요약, 디자인/기획 정리, 내부 초안 생성, 내부 업무 생성은 적극 자동화한다.  
> 상태 자동화는 넓게 허용할 수 있으나 수정 권한은 지호님 중심으로 강하게 제한한다.  
> 외부 발송은 상대 불만/오해 가능성을 기준으로 지호님 또는 대표 확인 경계를 둔다.  
> 플로라는 구조화와 제안에는 적극 개입하되, 민감 실행의 최종 책임을 대체하지 않는다.
