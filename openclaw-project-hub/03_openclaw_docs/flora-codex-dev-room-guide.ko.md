# 플로라 코덱스 개발실 운영 가이드 v0.1

_상태: 실사용 시작용 초안 / 운영하면서 보정 예정_

## 1. 방 목적

`플로라 코덱스 개발실`은 **로컬 개발용 코덱스 CLI 작업 전용 방**이다.

이 방에서는 아래 작업을 우선 다룬다.

- 코드 수정
- 파일 생성 / 정리
- 실행 / 테스트
- 에러 확인
- 리팩토링
- 로컬 프로젝트 개발 진행

이 방은 **실행형 개발 창**이며,
회사 전체 운영/전략/브리핑/승인 판단은 통합비서 창에서 다룬다.

---

## 2. 기본 운영 원칙

### 원칙 1. 이 방은 코덱스 전용이다
이 방에서 개발 실행은 기본적으로 **코덱스 CLI** 기준으로 진행한다.

### 원칙 2. 통합비서 창과 역할을 분리한다
- 통합비서 창: 판단 / 문서 방향 / 운영 회의 / 우선순위
- 코덱스 개발실: 실제 코드 작업 / 실행 / 디버깅 / 수정

### 원칙 3. Claude Code와 병행 시 로컬 지침을 우선한다
같은 프로젝트에서 Claude Code와 함께 작업할 경우,
반드시 아래 순서로 먼저 확인한다.

1. 로컬 프로젝트 지침
2. `AGENTS.md`
3. `README.md`
4. 허브 문서
5. 관련 작업 메모 / 운영 규칙

### 원칙 4. 다른 에이전트 작업 범위를 침범하지 않는다
- Claude Code가 이미 맡은 범위를 함부로 덮어쓰지 않는다.
- 같은 파일을 건드릴 때는 범위 충돌 여부를 먼저 확인한다.
- 애매하면 먼저 지호님에게 확인한다.

### 원칙 5. 실사용 우선으로 움직인다
이 방은 이론 설계보다 **실제 작동하는 수정/실행/확인**을 우선한다.

## 3. 이 방에서 하는 것

### 포함
- "이 파일 고쳐줘"
- "이 기능 구현해줘"
- "실행해보고 에러 원인 찾아줘"
- "리팩토링해줘"
- "로그 보고 원인 분석해줘"
- "이 프로젝트 구조 정리해줘"

### 제외
- 회사 전체 브리핑
- 전략 회의
- 운영 정책 판단
- 대표 보고
- 정산/금액 확정
- 외부 발송 판단
- 장기 문서 설계 회의

이런 건 통합비서 창에서 진행한다.

---

## 4. 권한/사용자 전제

### 사용자
- 현재는 **지호님 혼자 사용**

### 권한 원칙
- 이 방에서 플로라는 지호님 컨텍스트를 따른다.
- 따라서 개발/수정/실행 관련 범위는 넓게 열 수 있다.
- 다만 외부 발송, 금액 확정, 운영 승인 판단은 이 방의 기본 범위를 벗어난다.

---

## 5. 추천 사용 흐름

### 흐름 A. 통합비서 → 개발실
1. 통합비서 창에서 방향 결정
2. 개발실에서 실제 수정/실행
3. 결과를 다시 통합비서 창에서 정리/판단

### 흐름 B. 개발실 직접 실행
1. 지호님이 개발실에서 바로 요청
2. 코덱스 기준으로 수정/실행
3. 필요 시 통합비서 창에서 상위 판단 요청

---

## 6. 방 설명 문구

코덱스 CLI 기반 로컬 개발 전용 방. 코드 수정, 실행, 에러 확인, 리팩토링 중심. 회사 운영/전략/브리핑은 통합비서 창에서 진행. Claude Code와 병행 시 로컬 지침을 먼저 확인하고 충돌 없이 작업한다.

---

## 7. 첫 고정 메시지

이 방은 **플로라 코덱스 개발실**입니다.

용도:
- 로컬 개발 작업
- 코드 수정
- 실행 / 에러 확인
- 리팩토링
- 프로젝트 구조 정리

원칙:
1. 이 방에서는 **코덱스만 사용**
2. Claude Code와 같이 개발하는 경우 **로컬 지침을 먼저 확인**
3. 기존 작업 범위와 충돌 가능성이 있으면 먼저 확인 후 진행
4. 회사 운영/전략/브리핑/판단은 **통합비서 창**에서 진행

예시 요청:
- 이 파일 수정해줘
- 이 에러 원인 찾아줘
- 이 기능 구현해줘
- 실행해보고 로그 봐줘
- 리팩토링해줘

## 8. 통합비서 ↔ 개발실 라우팅 기준

### 통합비서 창으로 갈 것
- 뭐부터 해야 하는지 결정
- 문서 방향성 논의
- 자동화 경계 / 승인 정책
- 회사 운영 판단
- 대표 보고 / 브리핑

### 코덱스 개발실로 갈 것
- 코드 작성
- 파일 수정
- 실행 / 테스트
- 에러 분석
- 리팩토링
- 로컬 프로젝트 작업

---

## 9. 운영 중 체크 포인트

운영 시작 후 아래를 보며 보정한다.

1. 통합비서와 개발실 역할이 실제로 안 섞이는지
2. Claude Code와의 충돌이 없는지
3. 개발실에서 문서 회의가 너무 길어지지 않는지
4. 지호님이 실제로 빠르게 쓰기 편한지

---

## 10. 최종 운영 문장

> 플로라 코덱스 개발실은 로컬 개발용 코덱스 CLI 전용 방으로 운영한다.  
> 이 방은 코드 수정, 실행, 에러 확인, 리팩토링에 집중하고, 회사 전체 판단과 브리핑은 통합비서 창에서 분리해 다룬다.  
> Claude Code와 병행 작업 시에는 로컬 지침을 먼저 확인하고 충돌 없이 작업하는 것을 기본 원칙으로 한다.
