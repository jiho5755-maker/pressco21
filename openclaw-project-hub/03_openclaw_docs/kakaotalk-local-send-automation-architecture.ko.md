# PRESSCO21 맥북 카카오톡 로컬 전송 자동화 설계 v1

_대상: 지호님, 플로라, Claude Code, Codex, 향후 자동화 에이전트_

## 1. 목적

이 문서는 지호님이 외부에 있을 때 텔레그램으로 플로라에게 명령하면,
맥북에 로그인된 카카오톡 데스크톱 앱을 통해 **내부 팀방에 파일과 메시지를 자동 전송**하는 구조를 설계하기 위한 문서다.

핵심 목표는 다음과 같다.

1. 텔레그램 명령으로 맥북 로컬 파일을 카카오톡에 보낼 수 있게 한다.
2. 카카오톡 공식 API가 아닌 **macOS UI 자동화 기반**으로 동작한다.
3. 내부 팀방 전송부터 시작해 점진적으로 안정화한다.
4. 플로라 종합 OS의 로컬 실행 브리지로 활용한다.

---

## 2. 최상위 비전

목표 구조:

**텔레그램 → 플로라 → 맥북 자동화 스크립트 → 카카오톡 PC → 내부 팀방 전송 → 결과 회신**

예시 명령:

> 디자인기획팀 단톡방에 로컬에 있는 동그라미숲 견적서를 올려주고  
> "해당 견적서 토대로 상세페이지 작업해주세요"라고 써줘

처리 결과:
- 플로라가 채팅방, 파일, 메시지를 해석한다.
- 맥북이 로컬 파일을 찾는다.
- 카카오톡 앱을 열고 해당 채팅방에 파일 첨부 + 메시지 입력 + 전송한다.
- 성공/실패 결과를 텔레그램으로 다시 알려준다.

---

## 3. 전제 조건

이 자동화는 아래 전제가 충족되어야 한다.

### 필수 전제
- 맥북 전원이 켜져 있음
- 맥북이 네트워크 연결 상태임
- 카카오톡 데스크톱 앱이 설치되어 있음
- 카카오톡이 로그인 유지 상태임
- macOS 접근성 권한 허용 가능
- macOS 자동화 권한 허용 가능
- 플로라/OpenClaw가 맥북에 명령을 전달할 수 있음

### 권장 전제
- 화면 잠금/절전 정책 최소화 또는 자동화 예외 설정
- 카카오톡 채팅방 이름이 표준화되어 있음
- 전송 대상이 우선 내부 팀방으로 제한되어 있음
- 전송용 파일 위치 규칙이 정리되어 있음

## 4. 기본 아키텍처

## 4.1 계층 구조

### A. 명령 해석 계층 (플로라)
역할:
- 텔레그램 자연어를 구조화된 전송 명령으로 변환
- 채팅방/파일/메시지/전송 정책 해석
- 승인 필요 여부 판단

예시 내부 구조:
- room_alias: 디자인기획팀
- file_query: 동그라미숲 견적서
- message_text: 해당 견적서 토대로 상세페이지 작업해주세요
- send_mode: internal_auto

### B. 로컬 브리지 계층 (맥북 실행기)
역할:
- 플로라 명령 수신
- 파일 탐색
- 카카오톡 UI 자동화 실행
- 전송 결과 반환

### C. 카카오톡 UI 자동화 계층
역할:
- 카카오톡 앱 활성화
- 채팅방 검색
- 파일 첨부
- 메시지 입력
- 전송

기술 후보:
- AppleScript / osascript
- Shortcuts
- Hammerspoon
- 필요 시 Keyboard Maestro

### D. 로깅/검증 계층
역할:
- 전송 로그 기록
- 실패 원인 기록
- 재시도 또는 사용자 알림

## 5. 1차 파일럿 범위

처음부터 범위를 넓히지 않는다.

### 허용 범위
- 내부 팀방만 대상
- 파일 1개 첨부
- 텍스트 메시지 1개
- 채팅방 alias 기반 지정
- 전송 성공/실패만 우선 반환

### 1차 대상 방 추천
- 디자인기획팀
- 물류팀
- 경영기획/내부 운영방

### 1차 제외 대상
- 외부 고객 개인톡
- 외부 거래처 단톡방
- 파트너/협회/기관 외부 채팅
- 대량 동시 전송

## 6. 명령 처리 흐름

## Step 1. 텔레그램 명령 수신
플로라가 자연어를 수신한다.

예:
- 디자인기획팀 단톡방에 동그라미숲 견적서 올리고 상세페이지 작업해주세요라고 써줘

## Step 2. 구조화
플로라가 아래 항목으로 해석한다.
- 대상 방 alias
- 파일 검색 키워드
- 전송 메시지
- 승인 필요 여부

## Step 3. 파일 검색
맥북 로컬에서 파일을 찾는다.

규칙:
- 정확 일치 우선
- 부분 일치 허용
- 여러 후보면 자동 전송 금지
- 후보 1개일 때만 전송
- 파일 없으면 실패 반환

## Step 4. 카카오톡 UI 자동화
- 카카오톡 활성화
- 채팅방 검색
- 대상 방 선택
- 파일 첨부
- 메시지 입력
- 전송

## Step 5. 결과 확인
반환 정보:
- 성공 여부
- 전송한 파일 경로
- 대상 채팅방
- 전송 시각
- 실패 시 오류 유형

## Step 6. 텔레그램 회신
플로라가 결과를 다시 알린다.

## 7. 채팅방 식별 규칙

자연어 그대로 검색하면 흔들릴 수 있으므로 alias 체계를 둔다.

예:
- 디자인기획팀 → 카카오톡 실제 방 이름 1개에 매핑
- 물류팀 → 실제 방 이름 매핑
- 경영기획팀 → 실제 방 이름 매핑

### 추천 파일
- `kakaotalk-room-aliases.json`

예시 구조:
```json
{
  "디자인기획팀": "디자인기획팀 단톡방",
  "물류팀": "물류팀 단톡방",
  "경영기획": "경영기획 운영방"
}
```

### 원칙
- alias는 짧고 명확하게
- 실제 방 이름 중복 금지 권장
- 팀방 이름을 표준화하는 것이 안정성에 유리

## 8. 파일 검색 규칙

파일 검색은 자연어 해석보다 규칙이 중요하다.

### 추천 방식
- 지정 루트에서 파일명 검색
- 최근 수정 파일 우선
- 허용 확장자: pdf, xlsx, docx, hwp, png, jpg, rtf, md

### 검색 우선순위
1. 정확 파일명 일치
2. 부분 일치
3. 최근 수정일 우선
4. 여러 개면 사용자 확인 필요

### 권장 전송 루트
초기에는 아래처럼 제한된 루트를 두는 것이 좋다.
- `~/Desktop/카톡전송용/`
- `~/workspace/pressco21/`
- 특정 문서 폴더

### 금지
- 시스템 전역 무차별 검색
- 개인 민감 파일 폴더 자동 탐색

## 9. 카카오톡 UI 자동화 방식

## 9.1 1차 권장 기술
**AppleScript + osascript**

이유:
- macOS 기본 탑재
- 별도 상용 툴 없이 시작 가능
- 파일 첨부/앱 전환/키 입력 자동화 가능

## 9.2 보강 후보
**Hammerspoon**

이유:
- 키보드/윈도우/UI 제어 세밀함
- 장기 운영 시 안정화에 유리

## 9.3 예상 동작
- 카카오톡 activate
- 검색 포커스 이동
- 채팅방 검색어 입력
- 채팅방 선택
- 파일 첨부 대화상자 열기
- 파일 경로 입력/선택
- 메시지 붙여넣기
- Enter 전송

## 10. 승인/보안 정책

## 10.1 기본 정책
1차 파일럿은 아래 정책으로 시작한다.

- 내부 팀방만 자동 전송 허용
- 외부 채팅은 자동 금지
- 파일 1개 + 메시지 1개만 허용
- 후보 파일 2개 이상이면 자동 중단
- 미확인 방 이름이면 자동 중단

## 10.2 승인 필요 대상
- 외부 고객
- 외부 파트너
- 거래처/기관/협회
- 견적/계약/정산 민감 문서
- 가격/환불/정책 변경 메시지

## 10.3 로그 필수 항목
- 요청 시각
- 요청자
- 대상 방
- 파일명
- 메시지 요약
- 성공/실패
- 실패 원인

## 11. 실패 케이스 정의

대표 실패 케이스는 아래와 같다.

### 파일 관련
- 파일 없음
- 파일 후보 여러 개
- 권한 없음
- 파일 경로 특수문자 문제

### 채팅방 관련
- alias 미등록
- 채팅방 검색 실패
- 잘못된 방 진입 위험

### 카카오톡 관련
- 앱 미실행
- 로그인 만료
- 팝업으로 포커스 꼬임
- UI 변경

### macOS 관련
- 접근성 권한 없음
- 자동화 권한 없음
- 화면 잠금/절전
- 입력 포커스 유실

### 처리 원칙
- 실패 시 절대 억지로 재시도 반복하지 않음
- 텔레그램에 실패 원인 보고
- 필요 시 수동 개입 요청

## 13. 1차 파일럿 명령 형식 제안

플로라 자연어 명령 예시:
- 디자인기획팀에 동그라미숲 견적서 보내고 해당 견적서 토대로 상세페이지 작업해주세요라고 써줘

플로라 내부 파싱 결과 예시:
- room_alias = 디자인기획팀
- file_query = 동그라미숲 견적서
- message_text = 해당 견적서 토대로 상세페이지 작업해주세요
- policy = internal_team_auto

CLI 인자 예시:
```bash
send-kakaotalk-file-and-message.sh \
  --room-alias "디자인기획팀" \
  --file-query "동그라미숲 견적서" \
  --message "해당 견적서 토대로 상세페이지 작업해주세요"
```

- 파일 검색 개선

## Phase 3. 플로라 통합
- 텔레그램 자연어 명령 처리
- 승인 규칙 연결
- 전송 결과 자동 회신

## Phase 4. 운영 확장
- 추가 팀방
- 메시지 템플릿
- 예약 전송
- 문서 유형별 정책 적용

## 15. 최종 운영 문장

> PRESSCO21의 카카오톡 로컬 전송 자동화는 맥북에 로그인된 카카오톡 데스크톱 앱을 UI 자동화로 제어하는 방식으로 구현한다.  
> 1차는 내부 팀방 대상 파일+메시지 전송에 한정한다.  
> 플로라는 텔레그램 명령을 구조화하고, 맥북은 파일 탐색과 전송을 수행하며, 결과를 다시 플로라에 반환한다.
