# PRESSCO21 OpenClaw 사용 안내

## 지금 무엇이 연결된 상태인가

- Telegram 봇 `@pressco21_openclaw_bot`
- `clawdbot` 게이트웨이
- PRESSCO21 OpenClaw 플레이북
- 회사 비서 에이전트(`executive-assistant`)

즉, 지금은 터미널에서 작업을 고르면 결과가 Telegram으로 오는 상태입니다.

## 가장 쉬운 실행 방법

### 방법 1. 데스크톱에서 바로 실행

`PRESSCO21 OpenClaw.command` 파일을 더블클릭합니다.

그러면 터미널이 열리고 메뉴가 뜹니다.

### 방법 2. 터미널에서 실행

터미널에서 아래 한 줄만 입력합니다.

```bash
pressco-openclaw
```

그러면 메뉴가 뜹니다.

```text
PRESSCO21 OpenClaw
1. 회의 브리프
2. 주간 액션 트래커
3. 핸드오프 브리프
4. 상세페이지 브리프
5. 영상 기획안
6. CS 트리아지
7. 운영 다이제스트
8. 지식 업데이트
9. 고객 브리프
10. 가격/마진 검토
11. 오픈마켓 운영 브리프
12. 입금 검토
q. 종료
```

원하는 번호를 고른 뒤 질문에 따라 내용을 입력하면 됩니다.
완료되면 결과는 Telegram으로 도착합니다.

## 각 메뉴가 하는 일

### 1. 회의 브리프

이럴 때 사용합니다.

- 회의 전에 논의할 내용을 정리하고 싶을 때
- 부서별 할 일과 결정사항을 한 번에 정리하고 싶을 때
- 메모가 흩어져 있는데 브리프 형태로 바꾸고 싶을 때

입력하는 것:

- 회의 목적
- 참여자
- 마감일
- 원문 메모
- 관련 문서 경로
- 승인자

결과:

- 회의 요약
- 결정 로그
- 아젠다
- 액션 아이템
- 후속 질문
- 담당자 추적표

### 2. 주간 액션 트래커

이럴 때 사용합니다.

- 이번 주 할 일을 담당자별로 정리하고 싶을 때
- 막힌 일과 승인 대기 일을 한 번에 보고 싶을 때
- OpenClaw 파일럿 운영 상황을 주간 단위로 관리하고 싶을 때

입력하는 것:

- 이번 주 초점
- 담당자
- 막힌 항목
- 승인 대기 항목
- 기준 마감일

결과:

- 주간 초점
- 담당자별 상태
- 막힌 항목
- 승인 대기 목록
- 에이전트 사용 로그
- 주간 요약

### 3. 핸드오프 브리프

이럴 때 사용합니다.

- 대표 메모를 부서 전달용 문서로 바꾸고 싶을 때
- 디자인팀, 영상팀, 운영팀에 넘길 전달사항을 정리하고 싶을 때
- 구두 지시를 실행 가능한 문장으로 정리하고 싶을 때

입력하는 것:

- 전달 목적
- 전달자
- 수신자
- 마감일
- 전달 메모
- 승인자

결과:

- 전달 요약
- 핸드오프 노트
- 액션 아이템
- 후속 질문
- 승인 체크리스트

### 4. 상세페이지 브리프

이럴 때 사용합니다.

- 신상품 상세페이지를 처음 기획할 때
- 디자이너에게 넘길 상세페이지 구조를 먼저 만들고 싶을 때
- 어떤 사진과 자료가 더 필요한지 알고 싶을 때

입력하는 것:

- 상품명
- 카테고리
- 상품 유형
- 핵심 고객
- 핵심 특징
- 보유 자산
- 판매 채널
- 꼭 들어가야 할 내용

결과:

- 상품 요약
- 섹션 구조안
- 카피 포인트
- 자료 체크리스트
- 채널 메모
- 승인 체크리스트

### 5. 영상 기획안

이럴 때 사용합니다.

- 이번 달 또는 다음 달에 무엇을 찍을지 막막할 때
- 숏츠와 릴스를 같이 운영할 계획이 필요할 때
- 기존 영상을 재활용할 우선순위를 정하고 싶을 때

입력하는 것:

- 계획할 주 수
- 시즌 맥락
- 촬영 후보 상품
- 채널 목표
- 반드시 포함할 내용
- 재활용 가능한 기존 영상

결과:

- 주차별 촬영 계획
- 재활용 우선순위
- 플랫폼별 메모
- 준비물 체크
- 승인 체크리스트

### 6. CS 트리아지

이럴 때 사용합니다.

- 전화, 카톡, 게시판 문의를 먼저 분류하고 싶을 때
- 바로 답하지 말고 안전한 초안을 먼저 만들고 싶을 때
- 누가 처리해야 할지 빠르게 정리하고 싶을 때

입력하는 것:

- 문의 채널
- 고객 문의 원문
- 주문번호
- 문의 상품명
- 고객 유형

결과:

- 문의 카테고리
- 위험도
- 1차 답변 초안
- 추가 질문
- 담당자 경로
- 승인 체크리스트

이럴 때 사용합니다.

- 이번 주 운영에서 무엇이 막혔는지 정리할 때
- 반복 질문과 실패 사례를 한 번에 보고 싶을 때
- 다음 개선 행동을 뽑고 싶을 때

입력하는 것:

- 대상 기간
- 운영 로그
- 반복 질문
- 사고/막힘 사례

결과:

- 운영 요약
- 병목
- KPI 신호

- 다음 액션
- 승인 체크리스트

### 8. 지식 업데이트

이럴 때 사용합니다.

- 결정된 내용을 FAQ나 SOP 후보로 남기고 싶을 때
- 반복 질문을 문서 갱신으로 연결하고 싶을 때
- OpenClaw가 쓸수록 회사 문서가 좋아지게 만들고 싶을 때

입력하는 것:

- 정리할 주제
- 확정된 결정
- 반복 질문
- 기준 문서 경로
- 변경 메모

결과:

- FAQ 후보
- SOP 변경 후보
- 문서 갱신 후보
- 미해결 질문
- 승인 체크리스트

### 9. 고객 브리프

이럴 때 사용합니다.

- 거래처나 강사 고객을 다시 파악하고 싶을 때
- 휴면 고객 재활성화 아이디어가 필요할 때
- 미수금과 관계 리스크를 같이 보고 싶을 때

입력하는 것:

- 고객명
- 고객 유형
- 이번 관계 목표
- 최근 거래
- 최근 주문 요약
- 고객 메모

결과:

- 고객 요약
- 기회 신호
- 관계 리스크
- 다음 액션
- 후속 연락 스크립트
- 승인 체크리스트

### 10. 가격/마진 검토

이럴 때 사용합니다.

- 이 상품을 얼마에 팔아야 할지 검토할 때
- 채널별 마진이 위험한지 보고 싶을 때
- 가격을 올릴지 묶음으로 팔지 판단하고 싶을 때

입력하는 것:

- 상품명
- SKU코드
- 메이크샵 branduid
- 판매 채널
- 현재 원가
- 원가 유형
- 현재 판매가
- 목표 마진
- 건당 비용 배분수량
- 개당 CBM 또는 가로/세로/높이
- 원산지증명 적용 여부

중요:

- 현재 원가와 현재 판매가를 비워두면 연계마스터 엑셀에서 자동 조회를 시도합니다.
- 후보가 여러 개 나오면 SKU코드나 branduid를 같이 넣는 것이 안전합니다.
- 자사몰이 아닌 채널에서 판매가를 비워두면 메이크샵 기준가를 임시 기준가로 사용합니다.
- 사람이 검토한 원가 원장이 있으면 `reviewed.json`을 연계마스터보다 먼저 봅니다.
- 환율은 입력하지 않으면 최신 게시 CNY/KRW 환율을 기본값으로 가져옵니다.
- 수입 부가세는 기본적으로 COGS에 넣지 않고 별도 관리합니다.
- 광고비는 COGS가 아니라 채널별 광고 예산율로 따로 봅니다.

결과:

- 마진 요약
- 최소선/목표선 체크
- 위험 플래그
- 가격 옵션
- 시나리오 비교
- 계산 가정
- 부족한 데이터
- 채널 메모
- 카탈로그 조회 요약
- 다음 액션
- 승인 체크리스트

원가 원장 운영:

- 원가 원장 엑셀: `output/spreadsheet/pressco21-product-cost-ledger.seed.xlsx`
- OpenClaw 우선 참조 JSON: `docs/reference/openclaw-pressco21-product-cost-profiles.reviewed.json`
- 운영 문서: `docs/openclaw-pressco21-cost-ledger-workflow.ko.md`
- 정책 문서: `docs/openclaw-pressco21-cost-policy.ko.md`


## 꼭 기억할 점

- 터미널 입력은 초안 작성을 위한 재료입니다.
- 최종 승인과 실행 판단은 사람이 합니다.
- 주문, 재고, 환불 같은 고위험 업무 자동 실행용으로 쓰는 구조는 아닙니다.
